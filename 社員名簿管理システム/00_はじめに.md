## 0.1 本書の目的

本書は、架空のマルチテナント型SaaS「社員名簿管理システム(Employee Roster Management System)」を対象に、要求定義からクエリ設計までを統合し、各設計成果物が相互にどのようにつながるかを示す統合設計書である。利用登録者が会社のオーナーとなり、会社情報を登録し、システム管理者を招待・有効化した後に社員名簿の運用を開始する一連の流れを扱う。統合する設計成果物は次のとおり。

1. 要求定義(システム化の目的・業務要求・非機能要求)
2. 機能要件(機能一覧・システムユースケース)
3. 画面設計
4. データベース設計
5. API設計
6. JOB設計
7. モジュール設計
8. クエリ設計

本書は各成果物を個別に記載するだけでなく、業務要求(BR)から機能(F)、システムユースケース(UC)、オンライン経路（画面(SCR)→API）またはバッチ経路（JOB）、モジュール(M)、SQL、データベースへ落とし込む関係を、第9章「要求・設計トレーサビリティ」と第12章「設計の落とし込み関係」で示す。画面・データベース・API・JOB・モジュール・クエリは実装判断に必要な詳細設計レベルまで定義する。

## 0.2 サンプル・技術基盤としての前提

本書に記載する会社・組織・社員・権限・業務フロー・画面・API・データはすべて架空である。

| 項目 | 前提 |
|---|---|
| 対象システム | 社員名簿管理システム(Employee Roster Management System) |
| 提供形態 | 複数企業が共同基盤を利用し、会社（テナント）単位で契約・設定・権限・データを分離するB2B SaaS |
| 主な利用者 | SaaS運営管理者（`SERVICE_ADMIN`）、オーナー（`OWNER`）、人事担当者（`HR`）、部門管理者（`DEPARTMENT_MANAGER`）、一般社員（`EMPLOYEE`）、システム管理者（`SYSTEM_ADMIN`） |
| 対象業務 | SaaS利用開始、会社情報、管理者招待・有効化、利用者・権限、社員の名簿情報の登録・検索・参照・更新・異動・退職・一括取込・出力・変更履歴・監査・解約 |
| 認証 | メールアドレス確認を伴うアカウント認証。オーナー・システム管理者は多要素認証を必須とする |
| バックエンド | Cloudflare Workers Paid。HTTP APIは`fetch`ハンドラー、定期処理はCron Triggersの`scheduled`ハンドラーからCloudflare Queuesへ投入し、`queue`ハンドラーで継続実行する |
| JOB固定値 | 日次JOBは1チャンク最大40社員、D1実行文予算900文（Paid上限1000文）、Queue consumerは`max_batch_size=1`・`max_concurrency=1`・`max_retries=3`、規定回数超過はDLQへ移送 |
| データベース | Cloudflare D1。SQLite互換SQLを使用し、Worker Binding `env.DB`はM-006だけが扱う |
| 対象外 | 給与計算、勤怠計算、採用選考、評価査定、社会保険手続、実決済、SSO・SCIM |
| 設計粒度 | 要求・機能に加え、画面・データベース・API・JOB・モジュール・SQLを詳細設計レベルで定義 |

## 0.3 設計確定範囲と実装時確定範囲

| 対象 | 本書で詳細レベルまで定義する内容 | 実装・環境設計で確定する内容 |
|---|---|---|
| 画面 | 全画面の項目、初期表示、イベント、入力チェック、状態・表示制御、遷移、メッセージ。D1/SQLを直接参照しない | ピクセル単位レイアウト、デザインシステム実装、アニメーション |
| DB | Cloudflare D1、SQLite互換SQL、全テーブルの物理カラム、制約、インデックス、`batch()`境界、M-006だけに許可するBindingアクセス境界 | D1データベースID、環境別容量、実行計画の継続チューニング |
| API | Cloudflare Workers上の全APIの物理入出力、バリデーション、単一のM-002公開IF（ログインだけM-003）を呼ぶ処理フロー・処理詳細、エラー、冪等・競合。`env.DB`/M-006/SQLを直接参照しない | Worker内のファイル・クラス配置、シリアライザ実装 |
| JOB | Cloudflare Cron TriggersからQueuesへ投入する初回経路、`queue`ハンドラーからM-002公開IFを1回呼ぶ40件単位の継続経路、処理フロー・処理詳細、冪等性、再配信、D1クエリ予算、DLQ、監視 | Wranglerの環境別Queue ID、通知先、DLQ再投入の運用主体 |
| モジュール | M-001〜M-008の公開処理、入出力、処理フロー、処理詳細、例外、およびM-006に限定した利用SQL | プログラミング言語固有のクラス名、DI設定、ソース配置 |
| クエリ | SQL-ID、入力、SQL本文、出力、期待件数、利用インデックス、競合制御 | 実データ統計に基づく実行計画の継続チューニング |
