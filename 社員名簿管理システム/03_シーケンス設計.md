### 3.1.3 Cloudflare D1共通再試行契約

D1書込みの即時再試行は、Cloudflareが再試行推奨として示す`D1 DB reset because its code was updated.`、`Internal error while starting up D1 DB storage caused object to be reset.`、`Network connection lost.`、`Internal error in D1 DB storage caused object to be reset.`、`Cannot resolve D1 DB due to transient issue on remote node.`だけを対象とする。M-002は初回に加えて最大2回、full jitterで0〜500ms、0〜1000ms待機してbatch全体を再試行する。

書込み送信後に成否が不明な場合は、再試行前にM-006経由で対象状態とversionを読み直す。意図した状態・versionならコミット済み成功、事前状態なら未反映として再試行、どちらでもなければ競合とする。version競合、制約違反、業務条件不成立は再試行しない。過負荷、タイムアウト、CPU、メモリ、実行文予算超過は即時再試行せず、オンラインAPIでは一時障害応答、JOBではretryable属性付き`DATA_ACCESS_ERROR`としてQueue再配信へ委ねる。

## 3.15 ログアウト

UC-013（状態パターン UC-013/SP-1〜SP-4）。共通ヘッダーの明示ログアウトまたは認証失効を契機に、M-001が進行中要求、ブラウザータブ内メモリのアクセストークン、検証済み戻り先、画面内個人情報を破棄し、SCR-010へ遷移する。新規API、JOB、M-002〜M-008、D1、SQLは呼び出さない。

```mermaid
sequenceDiagram
    autonumber
    actor U as 全利用者
    participant UI as 認証済み画面（SCR-001〜SCR-009・SCR-011）
    participant PRES as M-001 プレゼンテーション
    participant MEM as ブラウザータブ内メモリ
    participant LOGIN as SCR-010 ログイン画面

    U->>UI: 共通ヘッダーのログアウトを押下する
    UI->>PRES: M-001/IF-04 ログアウト処理を要求する
    PRES->>PRES: 新規操作を無効化し、進行中要求を取り消す
    PRES->>MEM: アクセストークンと検証済み戻り先を破棄する
    MEM-->>PRES: 未認証状態
    PRES->>UI: 入力値と表示中の個人情報を破棄する
    PRES->>LOGIN: 未認証状態で遷移する
    LOGIN-->>U: ログイン画面を表示する
```

**連携定義**

条件分岐

| 条件ID | 判定箇所 | 条件 | 成立時 | 不成立時 | 根拠 |
|---|---|---|---|---|---|
| COND-01 | M-001 | 明示ログアウト時に認証状態が残っている | 基本フローで全状態を破棄する | 冪等に破棄を再実行してSCR-010へ遷移する | UC-013/SP-1（不成立=UC-013/SP-2） |
| COND-02 | M-001 | 起動契機がUNAUTHENTICATEDまたは期限到来である | MSG-12通知後に同じ破棄処理を行う | 明示ログアウトとして処理する | UC-013/SP-3（不成立=UC-013/SP-1） |
| COND-03 | M-001 | 起動契機がタブ終了または再読込である | メモリを引き継がず次回SCR-010から開始する | 現在タブ内でSCR-010へ遷移する | UC-013/SP-4 |

データ参照・更新

| 対象 | 操作 | 目的 | 実行主体 |
|---|---|---|---|
| 進行中API要求 | 取消 | ログアウト後の応答反映を防ぐ | M-001 |
| アクセストークン | メモリ破棄 | 認証状態を終了する | M-001 |
| 検証済み戻り先 | メモリ破棄 | ログイン後の意図しない復帰を防ぐ | M-001 |

トランザクション境界

なし。クライアント内の冪等な状態破棄であり、API・JOB・D1トランザクションを使用しない。

補足事項

| 観点 | 内容 |
|---|---|
| API | ログアウトAPIは新設しない。M-001だけで完結し、M-002〜M-008を呼ばない |
| 冪等性 | 二重押下、認証失効後の明示操作、再読込をすべて同じ未認証状態へ収束させる |
