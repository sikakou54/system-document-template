# 機能要件テンプレート記載ルール

本書は、[機能要件テンプレート](02_機能要件.md)の各項目を同じ粒度で記載するための正本である。明示した要求を後続設計で変更してはならず、資料から確定できない業務判断は推測せずTBDとして管理する。

## 2. ID・命名規則

| 対象 | 形式 | 採番単位 | 参照方法 |
|---|---|---|---|
| 機能 | F-XXX | 文書全体 | F-001 |
| ユースケース | UC-XXX | 文書全体 | UC-001 |
| 基本フローStep | UC-XXX/BF/Step-n | UC内 | UC-001/BF/Step-5 |
| 代替フロー | UC-XXX/ALT-x | UC内 | UC-001/ALT-1 |
| 代替フローStep | UC-XXX/ALT-x/Step-n | 代替フロー内 | UC-001/ALT-1/Step-9 |
| 例外フロー | UC-XXX/EXC-x | UC内 | UC-001/EXC-1 |
| 例外フローStep | UC-XXX/EXC-x/Step-n | 例外フロー内 | UC-001/EXC-1/Step-1 |
| 業務ルール | RULE-XXX | 文書全体 | RULE-001 |
| 状態 | UC-XXX/STATE-XX | UC内の状態定義行順 | UC-001/STATE-01 |
| 状態パターン | UC-XXX/SP-x | UC内 | UC-001/SP-1 |
| 状態遷移 | UC-XXX/TR-XX | UC内 | UC-001/TR-01 |
| 入力 | UC-XXX/IN-XX | UC内 | UC-001/IN-01 |
| 出力 | UC-XXX/OUT-XX | UC内 | UC-001/OUT-01 |
| 未決事項 | TBD-XXX | 文書全体 | TBD-001 |

- 新規IDは同じ採番単位の最大値+1とする。欠番は許容し、欠番・廃止IDを別対象へ再利用しない。
- 廃止時は参照元を確認して廃止記録を残し、既存IDを詰めない。
- ID変更時は本文、状態パターン、フロー、トレーサビリティ、後続設計、試験の正引き・逆引きを同時に確認する。
- フロー図の`U`・`SCR`・`FUNC`は図内aliasであり要件IDではない。具体的な画面名・API・モジュール・データベースを機能要件へ持ち込まない。

## 3. 完了判定

- 全F・UC・SP・ALT・EXC・RULE・入力・出力・状態・TBDが正引き・逆引きできる。
- 事前条件とフロー内判定を混在させず、正常・代替・例外の条件・結果・終了状態を一意に説明できる。
- 補助シーケンス図の番号付きメッセージが内容表と一致し、`SCR->>FUNC`の要求に`FUNC-->>SCR`の戻りがある。
- TBD以外に「所定」「適切」「など」「有効な値」のような未定義表現が残らない。
- TBDの着手影響に「解消後」とある機能は、決定前に確定設計へ進めない。
