# 統合設計書 テンプレート（設計種別別）

要求定義から詳細レベルの画面・DB・API・JOB・モジュール・クエリ設計、トレーサビリティまでを設計種別ごとに記述するテンプレート集。画面設計はHTML原本とPNGモックアップを対で管理する書式を含む。各文書には、章または記入ブロックの直前に定義ルールコメント（定義内容／定義する条件／項目説明／定義ルール）を配置している。コピーして各文書を埋める。記入例は [../社員名簿管理システム/](../社員名簿管理システム/)。

## 技術基盤・アーキテクチャ前提

- 本番バックエンドはCloudflare Workers Paid、永続化DBはCloudflare D1、非同期基盤はCloudflare Queuesとする。
- `env.DB`、D1 Workers Binding API、SQL実行はM-006 D1データアクセスだけに限定し、APIはログイン以外をM-002、ログインをM-003へ、JOBはM-002へ委譲する。
- 技術基盤の確定前提とデータアクセス境界は[サンプル・技術基盤としての前提](00_はじめに.md#02-サンプル技術基盤としての前提)・[設計確定範囲と実装時確定範囲](00_はじめに.md#03-設計確定範囲と実装時確定範囲)、JOB固定値（チャンク上限・Statement内部予算/上限・Queue設定・再配信）は[JOB設計](06_JOB設計.md#6-job設計)、`?1`〜`?100`の連番bindとD1実行上限は[クエリ設計](08_クエリ設計.md#8-クエリ設計)を正本とする。本節は概観のみを示し、固定値は再記載しない。
