# 統合設計書（単一文書様式）

要求定義から機能設計（シーケンス・画面・API・DB・モジュール）までを **1つの文書に統合** して記述する様式。
アップロードされたサンプル（基本設計サンプル）の目次・フォーマット（0. はじめに 〜 11. 設計の落とし込み関係）に沿いつつ、記載粒度は本リポジトリ `template/` の現行水準（状態パターン SP-x・代替/例外フロー・エラー表・DB 制約・処理フロー等）まで引き上げている。

`docs/` と `template/` の粒度別・多ファイル方式（正本1箇所＋ID参照）とは **別系統の成果物** として提供する。小〜中規模のシステムで、1冊にまとまった設計書を短期に作りたい場合に使う。

## 収録ファイル

同じ内容を **単一文書版** と **設計種別別（分割）版** の2形態で収録する。

### 単一文書版（1ファイルに統合）

| ファイル | 役割 |
|---|---|
| [テンプレート_統合設計書.md](テンプレート_統合設計書.md) | **テンプレートの正本**。各セクション直上に定義ルールコメント（定義内容 / 定義する条件 / 項目説明 / 定義ルール）を持つ空欄雛形。新規プロジェクトはこれをコピーして埋める |
| [社員名簿管理システム_統合設計書.md](社員名簿管理システム_統合設計書.md) | 上記テンプレートに従って書いた **記入サンプル**（架空の社員名簿管理システム。全機能網羅＋現行粒度） |

### 設計種別別（分割）版

単一文書版を設計種別ごとの文書へ分割したもの（内容は同一）。章立ての各節（0. はじめに 〜 11. 落とし込み関係）が1文書に対応する。

| ディレクトリ | 役割 | 入口 |
|---|---|---|
| [テンプレート/](テンプレート/) | 設計種別別のテンプレート集（各文書に定義ルールコメントを保持） | [テンプレート/README.md](テンプレート/README.md) |
| [社員名簿管理システム/](社員名簿管理システム/) | 設計種別別の記入サンプル | [社員名簿管理システム/README.md](社員名簿管理システム/README.md) |

## 目次構成

| No | セクション | 主な内容 |
|---|---|---|
| 0 | はじめに | 本書の目的、前提、基本設計と詳細設計の境界 |
| 1 | 要求定義 | 背景・課題、目的(OBJ)、利用者と役割、業務要求(BR)、非機能要求(NFR)、制約、対象範囲 |
| 2 | 機能要件 | 機能一覧(F)、主要ユースケース(UC：状態パターン SP-x・基本/代替(ALT)/例外(EXC)フロー)、機能要求詳細、シーケンス要否 |
| 3 | シーケンス設計 | 論理構成要素、主要 UC の連携シーケンス（Mermaid） |
| 4 | 画面設計 | 画面一覧(SCR)、画面遷移、代表画面の項目・状態・操作・メッセージ |
| 5 | API設計 | 方針、API一覧、代表 API の入出力・処理フロー・エラー定義・バリデーション、更新競合方針 |
| 6 | データベース設計 | 論理データモデル(ER)、テーブル一覧・定義、制約/インデックス、トランザクション境界、共通カラム、命名規則 |
| 7 | モジュール設計 | 論理モジュール構成(M)、責務、依存ルール、連携、公開機能 |
| 8 | 要求・設計トレーサビリティ | 要求→機能→シーケンス→画面/API/DB/モジュール の対応マトリクス |
| 9 | 設計レビュー用チェックリスト | 成果物種別（要求・機能／シーケンス／画面／API／DB／モジュール）ごとの観点 |
| 10 | 詳細設計への引継ぎ事項 | 本書を入力に詳細設計で確定する事項 |
| 11 | 設計の落とし込み関係 | 要求から各設計への落とし込みの流れ図 |

## 使い方

1. [テンプレート_統合設計書.md](テンプレート_統合設計書.md) をコピーする。
2. 各セクション直上の定義ルールコメントを読み、上から順に埋める（0→11）。
3. 記入の具体例は [社員名簿管理システム_統合設計書.md](社員名簿管理システム_統合設計書.md) を参照する。

> 注: 本様式はセクション 0.3 で「基本設計／詳細設計の境界」を示す。これは工程分担の目安であり、`docs/`・`template/` 側の運用ルール（基本設計／詳細設計への分類を用いない粒度別方式）とは独立した、単一文書向けの割り切りである。
