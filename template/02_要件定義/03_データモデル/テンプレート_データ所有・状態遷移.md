# 1. 概要

{システム名} が扱うデータについて、業務上の所有者、正本(System of Record: SoR)、書込責任、保持期間、および許可する状態遷移を横断的に定義する。

- 属性・関連・CRUD・アクセス権限の正本は各MDL文書と概念ER図、本書はデータ所有・正本・保持・状態遷移の正本とする。
- 外部サービスの情報を自システムへ保持する場合は、外部サービスの正本を置き換えず、検索・表示・再処理に用いるローカル投影として扱う。
- 状態を持つデータは本書に記載した遷移だけを許可する。記載のない遷移と、終端状態から過去状態への巻き戻しは禁止する。
- 保持期間は本プロダクトの運用方針であり、組織の法務・会計・個人情報方針がより長い期間を求める場合はその方針を優先する。

# 2. 用語

| 用語 | 定義 |
|---|---|
| 業務オーナー | データの利用目的・品質・アクセス方針・保持期間を決定する部門 |
| SoR | 対象情報について最終的に正しいと判断する正本システムまたは記録 |
| 書込責任 | データの作成・更新・削除を実行し、業務ルールと状態遷移を保証する主体 |
| ローカル投影 | 外部SoRの情報を自システム内で参照・検索するために同期した複製。外部SoRと不一致の場合は照合・再同期する |
| 導出データ | 別のSoRから再生成できる集計・表示用データ |

# 4. 状態遷移の共通ルール

| No | ルール |
|---|---|
| 1 | 初期状態はデータ作成と同じ確定単位で設定する |
| 2 | 状態更新は、現在状態と更新契機を再確認した上で条件付きで行う |
| 3 | 同一要求・同一外部イベントの再処理では、同じ状態を再設定せず変更なしとして正常終了できるようにする |
| 4 | 外部イベントの発生日時と現在状態を比較し、遅延到着した過去イベントで状態を巻き戻さない |
| 5 | 業務状態の更新と、その根拠となる外部イベントの処理済み更新は同じ確定単位で扱う |
| 6 | 論理削除は業務状態とは別の終端条件とし、削除済みデータへの通常操作を許可しない |
