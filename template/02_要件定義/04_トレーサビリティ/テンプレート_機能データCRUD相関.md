# 1. 概要

(機能要件・共通機能要件と各データの CRUD 関係を俯瞰する目的を1〜3行で記載。詳細は各 FR/CFR・MDL 文書を参照)

<!--
【2. CRUD相関マトリクス】
定義内容: 機能(行)と各データ(列)の交点に、当該機能が当該データへ行う操作を C/R/U/D で記す。
定義する条件: FR/CFR・エンティティを新設・廃止したら同一作業内で本表を更新する。
項目説明:
- 行(機能): FR/CFR を ID と名称で記す。特定データへの CRUD を持たない横断的機能(入力検証・エラー表示等)は行に含めず、表下に注記する。
- 列(データ): エンティティを業務用語(論理名)で記す。対応する MDL-ID は概要または表直前の一文で対応づける。
- セル: C/R/U/D(複数は「/」区切り)。操作しない場合は空欄。
- システム行: 利用者操作を伴わない自動処理(定期集計・課金計上・状態更新・セッション失効・ログ記録など)の CRUD を「システム（自動処理）」の1行にまとめる。
定義ルール:
- 表下に凡例を記す(C=作成 / R=参照 / U=更新 / D=削除。D は論理削除を含む)。作成・更新は該当データの検証参照を含むものとする。
- データ列は業務用語(論理名)で表し、物理名(英語のテーブル名・カラム名)や自動処理の実現方式(JOB 等)の設計層 ID は記載しない(§10-13)。機能行は FR/CFR ID を用いる。
- 各機能×データの操作は、各データ定義書(MDL)の CRUD・関連機能および各 FR/CFR の業務ルール・ユースケースに整合させる。

# 2. CRUD相関マトリクス

(列の各データが対応する MDL-ID を1文で対応づける)

| 機能 ＼ データ | エンティティA | エンティティB | エンティティC |
|---|:-:|:-:|:-:|
| FR-XXX 機能名 | C/U/D | R |  |
| CFR-XXX 機能名 | R |  | R |
| システム（自動処理） |  | C/U |  |

凡例: C=作成 / R=参照 / U=更新 / D=削除（論理削除を含む）。空欄は当該機能が当該データを操作しないことを示す。

# 3. 更新ルール

| 更新契機 | 更新内容 |
|---|---|
| FR/CFR の新規作成・変更 | 該当機能の行を追加／更新し、各データへの CRUD を各 FR/CFR の業務ルール・ユースケースに合わせて記入する |
| エンティティ(MDL)の追加・変更・廃止 | 該当データの列を追加／更新し、各 MDL の CRUD・関連機能に整合させる |
| ID廃止 | 行・列は削除せず、該当機能・データを廃止として扱い本表から除外する |

- 本表の更新は関連文書(FR/CFR/MDL)の作成・更新と同一作業内で行う(正本: docs/CLAUDE.md §6・§10-7)。
