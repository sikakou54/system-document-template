# 3. 更新ルール

- 機能要件・ユースケース・主要操作・API・JOB・外部連携の追加または変更時に、作成要否を再評価する。
- 個別SEQ文書の作成・更新・廃止と同一作業内で該当行を更新する。
- 省略対象に分岐・非同期・外部連携・トランザクション上の論点が生じた場合は「作成」へ変更する。
