# 3. 文書種別ごとの追加観点

| 文書 | 観点 |
|---|---|
| FR | 受入条件が検証可能な形で書かれている／画面・API・DB の詳細が書かれていない |
| NFR | 達成基準と検証方法が分離され、数値・条件・対象期間などで判定できる |
| UC | 基本・代替・例外フローが分離されている／事前・事後条件がある |
| SCR | 初期表示・権限・0件時・失敗時の表示が定義されている |
| API | 認可(ロール別可否)・冪等性が定義されている／エラーがエラーコード参照になっている |
| MOD | トランザクション境界・排他制御が定義されている／TBL×CRUD が網羅されている |
| TBL | 共通カラムを再記載していない／FK の削除時挙動・想定レコード数がある |
| JOB | リトライ・冪等性・多重起動制御が定義されている |
