# 2. ゲート状態の語彙

<!--
【2. ゲート状態の語彙】
定義内容: 工程ゲートで用いる状態値の一覧(意味の要約)。詳細な遷移条件は CLAUDE.md §7 を正本とする。
定義ルール: 文書状態(未着手/作成中/レビュー中/確定/廃止)とは別に管理する。承認は「承認済」でのみ表し、承認者はユーザーまたは設計責任者に限る。
-->

| 状態 | 意味 |
|---|---|
| 未着手 | 当該 Step の成果物にまだ着手していない |
| 作業中 | 成果物を作成・修正中 |
| レビュー中 | 作成者が完了宣言し、レビュー・承認を依頼済み |
| 承認済 | ユーザーまたは設計責任者が出口条件を確認し承認した(次 Step を確定してよい) |
| 失効 | 上流変更により承認が無効化された(影響分析後に再レビューする) |
| 廃止 | 当該 Step を使用しない |

定義する条件: 全プロジェクトで必須。成果物の作成・更新と同時に該当 Step の状態・証跡・更新日を更新する。
