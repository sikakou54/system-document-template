# マルチエージェント編成

| Agent | 役割 | 主な責務 |

## ワークフロー

1. 統括Agent: docs/CLAUDE.md と本設計の前提文書・テンプレート・共通仕様を読み、作業計画を立てる
2. リサーチAgent: 上流・依存・共通仕様を分析し、作成対象・依存関係・ID採番・整合ギャップを報告する
3. 作業Agent: リサーチ結果を基に、文書・一覧・トレーサビリティを作成する
4. レビューAgent(エンジニア視点)と(非エンジニア視点): それぞれ独立に並行してレビューし、指摘一覧を出す
5. 統括Agent: 両レビューの指摘を統合し採否を判断して、作業Agentへ修正を指示する
6. 手順4〜5を、両レビュー観点が全てパスし「未定」「要確認」がゼロになるまで繰り返す
7. 統括Agent: 状態を「確定」にし、最終成果物(下記「出力」)をまとめる

各Agentは docs/CLAUDE.md の作成ルール(§8)・参照方向ルール(§5)・状態定義(§7)・禁止事項(§10)に従う。

# 各エージェントの作業指示

## リサーチAgent(エンティティ→テーブルの分析)
- MDL のデータ項目定義から、テーブルのカラム(論理名・物理名・型・NULL・PK・Default)を洗い出す。物理名は命名規則.md に従う(テーブル: M_/T_/H_/TP_ 接頭辞、大文字)
- 他エンティティを参照する項目を外部キー(FK)として特定し、参照先「テーブル名.物理名」と削除時挙動(RESTRICT/CASCADE/SET NULL)を整理する
- 検索・整合性のためのインデックス(IDX-x・一意性)と、複数カラムにまたがる制約(UNIQUE/CHECK)を洗い出す
- 区分値カラムが参照する DEF-XXX/CODE-XXX を特定する(値・表示名・説明は再記載しない)
- 共通カラム(CREATED_AT/UPDATED_AT/DELETED_AT)は共通カラム.md が正本のため本文書に含めない点を確認する。TBL-ID の採番を確認する

## 作業Agent(データベース設計作成)
- docs/03_機能設計/06_データベース設計/index.md に先に行を追加してから、テンプレート_テーブル.md をコピーして TBL-XXX_テーブル名.md を作成し、§1〜§5 を全て埋める
- §2 カラム定義は主キー ID を先頭に、業務カラムを物理名(大文字)・型・NULL・PK/FK・Default・補足で定義する。共通カラム(CREATED_AT/UPDATED_AT/DELETED_AT)は記載しない
- FK は参照先「テーブル名.物理名」を記し削除時挙動を補足に、区分値カラムは補足に DEF-XXX/CODE-XXX を記載する。§3 インデックス(IDX-x)・§4 制約・§5 共通定義参照を定義する
- 全テーブル作成後、ER図.md を更新する(エンティティ・関連・多重度。物理カラムは書かず概念/主要関連中心)
- 論理名は種別接尾辞(〜マスタ/〜トランザクション/〜履歴/〜一時テーブル)を付ける

## レビューAgent(エンジニア視点)
- TBL-ID・FK 参照先テーブル・DEF-XXX が実在するか。物理名(テーブル・カラム・インデックス)が命名規則.md に従い大文字か
- 共通カラム(CREATED_AT/UPDATED_AT/DELETED_AT)を再記載していないか(共通カラム.md 参照か)。区分値の値・表示名を再記載していないか(DEF 参照か)
- 主キー・外部キー・一意制約・インデックスが整合し、FK の削除時挙動が定義されているか
- ER図.md の関連・多重度が各 TBL のカラム定義(FK)と整合しているか(矛盾時は TBL 側を正とできるか)
- 逆参照(このテーブルを使う MOD 一覧など)を書いていないか(§5・§10-2)

## レビューAgent(非エンジニア視点)
- トレース元 MDL のデータ項目が、テーブルのカラムとして漏れなく物理化されているか(網羅性)
- 論理名・概要が業務のデータ概念を正しく表しているか(物理名だけでなく論理名で意味が追えるか)
- 区分値・制約が業務ルール(重複禁止・有効期間など)と対応しているか
- 論理削除(DELETED_AT)などのデータライフサイクルが MDL の CRUD 方針と一致しているか
- テーブルの粒度・関連が、業務の実体(会議室・予約・利用者など)と自然に対応しているか

# この設計の必須ルール

- データベース設計(06)は物理名(英語のテーブル名・カラム名)を記載してよい例外領域(§8-13)。物理名は命名規則.md に従い大文字で記載する
- 共通カラム(CREATED_AT/UPDATED_AT/DELETED_AT)は共通カラム.md が正本。各 TBL に再記載しない(主キー ID は §2 に記載する)
- 区分値カラムの値・表示名・説明は共通定義設計(DEF)が正本。本文書は DEF-XXX/CODE-XXX 参照とし再記載しない
- テーブル(TBL)は他TBL(FK)・DEF のみ参照する(逆参照禁止。§5)。トレース元として MDL を参照する場合は対応表で管理する
- 「一覧に行追加 → 文書作成」の順を守り、同一作業内で index.md・ER図.md を更新する(§8-5・§10-7)
- ユーザーが記法・セクション構成を指摘したら、テンプレート_テーブル.md の定義ルールコメントへ反映する(§8-10)

# 完了条件

- 「未定」「要確認」がゼロ。ER図.md が最新のテーブル・関連を反映している
- index.md・ER図.md の内容と各 TBL 文書の基本情報・カラム(FK)が整合している
