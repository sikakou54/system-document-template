# システム設計ドキュメント テンプレート

要件定義・機能設計のドキュメント一式を、ID採番・トレーサビリティ・レビュー観点まで含めて標準化するためのテンプレートプロジェクト。
本ディレクトリを新規プロジェクトの docs/ としてコピーし、各テンプレートを埋めていくことで、粒度と整合性のそろったドキュメント群を作成できる。

## 1. このテンプレートの使い方

| 手順 | 内容 |
|---|---|
| 1 | 本ディレクトリ一式を新規プロジェクトの docs/ としてコピーする |
| 2 | CLAUDE.md(作業ルール正本)と 99_その他/ID採番ルール.md(採番の正本)に目を通す |
| 3 | 各ディレクトリの テンプレート_XXX.md をコピーし「ID_名称.md」にリネームして記入する(先に該当 index.md へ行を追加) |
| 4 | 記入後、99_その他/プロンプト/ のレビュー・横断整合チェックで整合性を確認する |
| 5 | 使わない文書種別ディレクトリは削除してよい(ID採番・参照方向のルールは維持する) |

## 2. ディレクトリ構成

| パス | 責務 |
|---|---|
| README.md | プロジェクト概要とドキュメント全体の入口 |
| CLAUDE.md | 作業ルール正本(役割・ID規則・参照方向・状態・レビュー・禁止事項) |
| 01_要件定義/01_業務要件/ | 業務要件(BR) |
| 01_要件定義/02_機能要件/ | 機能要件(FR)・共通機能要件(CFR)。CFR-000 がシステム全体の要件・CFR一覧・共通業務ルールを統括 |
| 01_要件定義/03_非機能要件/ | 非機能要件(NFR)。性能・可用性・セキュリティ・データ保護・監査など |
| 01_要件定義/04_業務ユースケース/ | 業務ユースケース(UC) |
| 02_機能設計/01_概要設計/ | システム全体の構成・外部サービス連携 |
| 02_機能設計/02_画面設計/ | 画面(SCR)、項目、イベント、入力チェック、メッセージ |
| 02_機能設計/03_API設計/ | API(API)の入出力・認証認可・バリデーション・エラー |
| 02_機能設計/04_JOB設計/ | バッチ・定期実行・非同期処理(JOB) |
| 02_機能設計/05_モジュール設計/ | サービス・ドメインロジック・内部処理(MOD) |
| 02_機能設計/06_データベース設計/ | テーブル(TBL)、ER図、Enum、共通カラム、命名規則 |
| 02_機能設計/07_クエリ設計/ | クエリ(SQL) |
| 02_機能設計/08_シーケンス設計/ | シーケンス(SEQ) |
| 02_機能設計/09_権限設計/ | ロール・権限制御(PERM) |
| 03_トレーサビリティ/ | BR/FR/UC/設計の対応表 |
| 99_その他/ | ID採番ルール、用語集、一覧・レビュー観点の共通テンプレート、作成・レビュー・整合チェック用プロンプト |

## 3. 推奨読み順(記入順)

| 順 | 対象 | 目的 |
|---|---|---|
| 1 | CLAUDE.md / 99_その他/ID採番ルール.md | 作業ルールと採番・命名規則を把握する |
| 2 | 01_要件定義/01_業務要件/ | 業務上実現したい状態(BR)を定義する |
| 3 | 01_要件定義/02_機能要件/(CFR-000・CFR・FR) | 共通要件と提供する機能を定義する |
| 4 | 01_要件定義/03_非機能要件/ | 品質要求(NFR)を定義する |
| 5 | 01_要件定義/04_業務ユースケース/ | 利用者とシステムの業務フロー(UC)を定義する |
| 6 | 02_機能設計/01_概要設計/ 〜 09_権限設計/ | 実装対象の設計(画面・API・JOB・モジュール・DB・クエリ・シーケンス・権限)を定義する |
| 7 | 03_トレーサビリティ/ | 要件→設計の対応を辿れるようにする |
| 随時 | 99_その他/用語集.md | 用語を統一する |

## 4. ルール

作業ルールの正本は CLAUDE.md である(作業開始時に必ず読むこと)。
ID採番・命名の正本は 99_その他/ID採番ルール.md である。
各文書種別のテンプレートは対応する各ディレクトリの テンプレート_XXX.md、一覧(index.md)の雛形は 99_その他/テンプレート_一覧.md を用いる。
