# 1. 概要

MeetRoom のドキュメント全体で使用する用語を定義する。用語の正本は本ファイルのみとし、各文書は本ファイルの定義に従って用語を使用する。

# 2. 用語一覧

| 用語 | 読み | 意味 | 使用箇所 |
|---|---|---|---|
| 会議室 | かいぎしつ | 予約の対象となる社内の会議室。管理者が登録・編集する | 03_機能設計/06_データベース設計(TBL-002)、FR-001 |
| 設備 | せつび | プロジェクター等、会議室に備え付けられる設備。設備マスタで管理する | 03_機能設計/06_データベース設計(TBL-004)、FR-005 |
| 予約 | よやく | 会議室を特定の日時・時間帯で利用するための登録。状態は DEF-001/CODE-004 で管理する | 03_機能設計/06_データベース設計(TBL-003)、FR-002 |
| 二重予約(ダブルブッキング) | にじゅうよやく | 同一会議室で時間帯が重複する予約。登録できない(発生時は ERR-003) | FR-002、MOD-003 |
| リマインド | リマインド | 予約開始前に予約者へメールで通知すること | FR-004、JOB-001 |
| 利用実績 | りようじっせき | 会議室ごとの月次の利用実績。管理者が確認する | FR-006、03_機能設計/06_データベース設計(TBL-006) |
| 稼働率 | かどうりつ | システムが正常に稼働している時間の割合。平日8:00〜20:00で99.5%以上を求める | 02_要件定義/02_非機能要件/NFR-002_可用性.md |
| ロール | ロール | ユーザーの権限区分。種別は DEF-001/CODE-001 を参照 | 03_機能設計/10_共通定義設計(DEF-001/CODE-001) |
| 論理削除 | ろんりさくじょ | レコードを物理削除せず deleted_at に日時を設定して無効化する削除方式。NULL=有効 | 03_機能設計/06_データベース設計/共通カラム.md |
| 空き判定 | あきはんてい | 指定した時間帯に重複する予約がない会議室を判定する処理 | MOD-002、API-002 |
| 有料会議室 | ゆうりょうかいぎしつ | 利用単価が0より大きい会議室。利用時間に応じ従量課金される | FR-008、TBL-002 |
| 利用単価 | りようたんか | 会議室の1時間あたりの利用料金(円)。0=無料 | TBL-002、FR-005 |
| 従量課金 | じゅうりょうかきん | 有料会議室の利用時間に応じて課金する方式(metered billing)。月次で請求する | FR-008、MOD-007 |
| Cloudflare | クラウドフレア | 本システムの実行基盤。Pages/Workers/D1/KV/Queues/Cron Triggers を利用 | 03_機能設計/01_概要設計 |
| Cloudflare Workers | ワーカーズ | API(REST)を実行するサーバーレス基盤 | 03_機能設計/03_API設計 |
| Cloudflare Pages | ページズ | フロント(SPA)を配信する基盤 | 03_機能設計/02_画面設計 |
| Cloudflare D1 | ディーワン | SQLite互換のデータベース。業務データを格納 | 03_機能設計/06_データベース設計 |
| Cloudflare KV | ケーブイ | キー値ストア。セッション・一時データに利用 | 03_機能設計/01_概要設計 |
| Cloudflare Queues | キューズ | 非同期処理のメッセージキュー。JOBで利用 | 03_機能設計/04_JOB設計 |
| Cron Trigger | クロントリガー | 定期実行のトリガー。リマインド・利用量計上JOBを起動 | JOB-001、JOB-002 |
| Stripe | ストライプ | 決済・従量課金基盤。従量サブスク・Meter Event・請求・Webhookを利用 | MOD-007、03_機能設計/01_概要設計/02_外部サービス連携.md |
| Meter Event | メーターイベント | Stripeへ送る利用量の記録。利用時間を計上する | MOD-007、TBL-007 |
| Webhook | ウェブフック | Stripeからのイベント通知。署名検証の上で冪等に処理する | API-011、MOD-007 |
| Resend | リセンド | メール送信サービス。リマインド等の送信に利用 | MOD-006、JOB-001 |
