# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | CFR-007 |
| 要件名 | 監査ログ記録 |
| 概要 | 予約・課金・会議室管理などの重要操作の実行時に、システムが操作の記録(監査証跡)を監査ログとして残す |
| 目的 | 問題発生時に重要操作を遡って追跡・確認できるようにし、不正・誤操作の検知と説明責任を担保する |
| 適用範囲 | データを変更する重要操作(予約・変更・キャンセル・会議室/設備管理・課金など)を伴う機能 |
| 対象ロール | 全ユーザー(記録はシステムが自動で行う。監査ログの参照は監査担当の管理者) |

# 2. 共通機能要件

適用範囲の各機能に共通して課す振る舞いを定義する。

| 項目 | 内容 |
|---|---|
| 利用者の操作 | なし(利用者の明示操作を要さず、重要操作の実行に付随してシステムが記録する) |
| システムの処理 | 重要操作(予約・変更・キャンセル・会議室/設備管理・課金など)の実行時に、いつ・誰が・何を行ったかを監査ログとして記録する |

# 3. 適用条件

この共通機能が適用される対象と適用外を定義する。

| 区分 | 内容 |
|---|---|
| 適用対象 | データを変更する重要操作(予約・変更・キャンセル・会議室/設備管理・課金)を行う機能 |
| 適用外 | 参照・検索のみで状態を変えない操作。監査ログの保持期間・改ざん防止などの品質要求は NFR-006。記録項目・保存方式などの実装は機能設計で定義する |

# 4. 業務ルール

この共通機能の判断基準となる業務上のルールを定義する。

| No | ルール |
|---|---|
| 1 | データを変更する重要操作(予約・変更・キャンセル・会議室/設備管理・課金)の実行時に、操作の記録(いつ・誰が・何を)を監査ログとして残す |
| 2 | 参照・検索のみで状態を変えない操作は監査ログの記録対象としない |
| 3 | 監査ログの改ざん防止・保持は NFR-006(監査)に従う |

# 5. 受入条件

この要件を満たしていることを確認する条件を定義する。

| No | 条件 |
|---|---|
| 1 | 対象の重要操作の実行時に監査ログが記録される |
| 2 | 記録された監査ログから、いつ・誰が・何を行ったかを追跡できる |
| 3 | 参照・検索のみの操作では監査ログを記録しない |

# 6. 前提条件

この要件が成り立つために、先に満たしておく条件を定義する。

| No | 条件 |
|---|---|
| 1 | 監査ログの対象となる重要操作が各機能要件(FR)で定義されている |

# 7. 制限事項

この要件での制限や範囲外などを定義する。

| No | 制限 |
|---|---|
| 1 | 記録する項目・保持期間・保存方式などの詳細は NFR-006 および機能設計で定義する(本要件は記録する行為と対象を扱う) |
| 2 | 監査ログの参照・分析機能自体は本要件の範囲外とする(記録を対象とする) |

# 8. システムユースケース

この共通機能要件を実現する利用者・システムの振る舞いを、ユースケース単位で定義する。

なし(監査ログ記録は重要操作を行う各機能のシステムユースケース内で付随的に行われ、独立した利用シナリオを持たない)。
