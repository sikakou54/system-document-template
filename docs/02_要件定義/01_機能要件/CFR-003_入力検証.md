# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | CFR-003 |
| 要件名 | 入力検証 |
| 概要 | 利用者の入力について、必須・型・形式・値の範囲・項目間の相関の妥当性を検証し、不正な入力は受け付けない |
| 目的 | 不足・不正な入力による誤処理を防ぎ、利用者が誤りを修正できるようにする |
| 適用範囲 | 利用者からの入力を受け付ける全機能 |
| 対象ロール | 全ユーザー(一般 / 管理者) |

# 2. 共通機能要件

適用範囲の各機能に共通して課す振る舞いを定義する。

| 項目 | 内容 |
|---|---|
| 利用者の操作 | 機能が求める項目を入力する |
| システムの処理 | 入力内容の必須・型・形式・値の範囲・項目間の相関の妥当性を検証する。妥当でない入力は受け付けず、どの項目がどのルールに反したかを利用者が理解できる形で示す |

# 3. 適用条件

この共通機能が適用される対象と適用外を定義する。

| 区分 | 内容 |
|---|---|
| 適用対象 | 利用者から入力を受け付けるすべての機能。入力単体で機械的に判定できる妥当性(必須・型・形式・単項目制約・項目間の相関) |
| 適用外 | 存在確認・重複・期間制約・状態遷移など、データ参照や業務判断を伴う検証(各機能要件・機能設計で定義) |

# 4. 業務ルール

この共通機能の判断基準となる業務上のルールを定義する。

| No | ルール |
|---|---|
| 1 | 検証対象は、入力単体で機械的に判定できる範囲(必須・型・形式・単項目制約・項目間の相関)に限る |
| 2 | 妥当でない入力は処理を進めず、対象項目と反したルールを利用者に示す |

# 5. 受入条件

この要件を満たしていることを確認する条件を定義する。

| No | 条件 |
|---|---|
| 1 | 必須項目が未入力の場合、処理されず対象項目が示される |
| 2 | 形式が不正な入力(日時・コード等)は受け付けられない |
| 3 | 開始が終了以降などの項目間の相関違反は受け付けられない |

# 6. 前提条件

この要件が成り立つために、先に満たしておく条件を定義する。

| No | 条件 |
|---|---|
| 1 | 各機能が必要とする入力項目と妥当性の条件が、当該機能要件で定義されている |

# 7. 制限事項

この要件での制限や範囲外などを定義する。

| No | 制限 |
|---|---|
| 1 | エラーの表示・返却形式や項目名などの実現方式は機能設計で定義し、本要件では扱わない |
