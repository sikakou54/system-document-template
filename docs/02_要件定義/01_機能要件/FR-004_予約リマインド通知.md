# FR-004：予約リマインド通知機能 仕様書

## 1. 基本情報
| 項目 | 内容 |
|---|---|
| 要件ID | FR-004 |
| 要件名 | 予約リマインド通知 |
| 概要 | 予約開始前に予約者へメールでリマインド通知する |
| 目的 | 予約忘れによる会議室の空押さえを減らし、予定通りの利用を促す |
| 対象ロール | 利用者 / 管理者 |

## 2. 業務ルール（制約事項）
システムが遵守すべき業務上の絶対ルールを定義する。このルールがユースケースのシナリオ分岐の根拠となる。

| No | ルール |
|---|---|
| 1 | リマインドは予約開始の一定時間前に1回通知する |
| 2 | キャンセル済みの予約は通知の対象外とする |
| 3 | 通知先は予約者とする |
| 4 | リマインドは1予約につき開始前1回とする |
| 5 | メール以外の通知手段はこの要件では扱わない |

## 3. 受入条件
この要件を満たしていることを確認する条件（テストの合否判定基準）を定義する。

| No | 条件 |
|---|---|
| 1 | 予約開始前に対象の予約者へリマインドが1回届く |
| 2 | キャンセル済みの予約にはリマインドが届かない |

## 4. システムユースケース
なし。本機能は利用者が対話するユースケースを持たず、予約開始前のリマインド通知を JOB-001（システムによる自動通知）で実現する。

- 前提: 予約が登録され、予約者に通知可能な宛先(メールアドレス)が登録されていること。
- 出力: 予約者へ、予約の日時・会議室・タイトルを知らせるリマインド通知が届く。
