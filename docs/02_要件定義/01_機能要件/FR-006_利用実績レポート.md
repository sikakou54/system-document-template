# FR-006：利用実績レポート機能 仕様書

## 1. 基本情報
| 項目 | 内容 |
|---|---|
| 要件ID | FR-006 |
| 要件名 | 利用実績レポート |
| 概要 | 管理者が会議室ごとの月次利用実績(件数・利用時間)を確認できる |
| 目的 | 会議室ごとの利用状況を可視化し、増設・統廃合や運用改善の判断材料にする |
| 対象ロール | 管理者 |

## 2. 業務ルール（制約事項）
システムが遵守すべき業務上の絶対ルールを定義する。このルールがユースケースのシナリオ分岐の根拠となる。

| No | ルール |
|---|---|
| 1 | 利用実績を確認できるのは管理者のみとする |
| 2 | 集計対象は利用が完了した予約に限る |
| 3 | 集計は対象月・会議室単位で行う |
| 4 | 利用実績の確認操作では予約データを変更しない |
| 5 | 課金金額の集計はこの要件では扱わない(FR-008) |

## 3. 受入条件
この要件を満たしていることを確認する条件（テストの合否判定基準）を定義する。

| No | 条件 |
|---|---|
| 1 | 管理者が対象月を指定すると会議室ごとの利用件数・利用時間が表示される |
| 2 | 利用が完了していない予約は集計に含まれない |
| 3 | 一般利用者は利用実績を確認できない |
| 4 | 利用実績を確認しても予約データは変更されない |

## 4. システムユースケース
この要件を実現する振る舞いを、データなどのパターンに応じてユースケース単位で定義する。

### 4.1 【UC-01】利用実績を確認する（管理者向け）
管理者が会議室ごとの月次利用実績を確認する。

**事前条件**

| No | 条件 |
|---|---|
| 1 | 管理者としてログイン済み |
| 2 | 集計対象となる予約の実績が蓄積されている |

**事後条件**

| No | 条件 |
|---|---|
| 1 | 指定月の会議室別利用実績が管理者に提示されている |
| 2 | 予約データは変更されていない |

**入力データ**

| 情報 | 要否 | 内容 |
|---|---|---|
| 対象月 | 必須 | 利用実績を集計する対象の月 |

**出力データ**

| 情報 | 内容 |
|---|---|
| 会議室名 | 集計対象の会議室の名称 |
| 利用件数 | 対象月に利用が完了した予約の件数 |
| 利用時間 | 対象月の会議室の合計利用時間 |

**基本フロー**

| Step | アクター | 操作/処理 | 結果 |
|---|---|---|---|
| 1 | 管理者 | 利用実績を開く | 利用実績画面が表示される |
| 2 | 管理者 | 対象月を指定する | 集計が要求される |
| 3 | システム | 指定月に利用が完了した予約を会議室ごとに集計する | 会議室別の利用件数・利用時間が算出される |
| 4 | システム | 会議室別の利用実績を提示する | 利用実績が表示される |

**代替フロー**

| ALT ID | 分岐Step | 条件 | フロー |
|---|---|---|---|
| ALT-1 | Step4 | 対象月に完了した予約が0件 | 実績が0件であることを確認し、対象月を変えて再確認する |

**例外フロー**

| EXC ID | 発生Step | 条件 | エラーメッセージ | 対応 |
|---|---|---|---|---|
| - | - | なし | - | - |
