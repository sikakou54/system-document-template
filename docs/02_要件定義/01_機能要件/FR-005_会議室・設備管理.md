# FR-005：会議室・設備管理機能 仕様書

## 1. 基本情報
| 項目 | 内容 |
|---|---|
| 要件ID | FR-005 |
| 要件名 | 会議室・設備管理 |
| 概要 | 管理者が会議室と設備を登録・編集できる |
| 目的 | 会議室・設備・利用単価を最新に保ち、検索・予約・課金の前提を正確にする |
| 対象ロール | 管理者 |

## 2. 業務ルール（制約事項）
システムが遵守すべき業務上の絶対ルールを定義する。このルールがユースケースのシナリオ分岐の根拠となる。

| No | ルール |
|---|---|
| 1 | 会議室・設備の登録・編集は管理者のみが行える(RULE-COM-001) |
| 2 | 会議室名・収容人数・利用単価は必須とする |
| 3 | 収容人数は1人以上とする |
| 4 | 利用単価は0以上とし、0は無料会議室として扱う |
| 5 | 利用停止中の会議室は新規予約の対象外とする |
| 6 | 会議室・設備の登録・編集は管理者に限る |
| 7 | 予約データの編集はこの要件では扱わない |

## 3. 受入条件
この要件を満たしていることを確認する条件（テストの合否判定基準）を定義する。

| No | 条件 |
|---|---|
| 1 | 管理者が会議室(利用単価を含む)を登録・編集すると内容が保存される |
| 2 | 管理者が設備を登録し会議室に紐付けられる |
| 3 | 会議室名・収容人数・利用単価が未入力の場合は保存されずエラーになる |
| 4 | 収容人数が1人未満の場合は保存されずエラーになる |
| 5 | 利用単価が0未満の場合は保存されずエラーになる |
| 6 | 利用単価0の会議室を無料会議室として保存できる |
| 7 | 利用停止にした会議室は新規予約の検索・予約対象から外れる |
| 8 | 一般利用者は会議室・設備の登録・編集ができない |

## 4. システムユースケース
この要件を実現する振る舞いを、データなどのパターンに応じてユースケース単位で定義する。

### 4.1 【UC-01】会議室を登録・編集する（管理者向け）
管理者が会議室と設備を登録・編集する。

**事前条件**

| No | 条件 |
|---|---|
| 1 | 管理者としてログイン済み |

**事後条件**

| No | 条件 |
|---|---|
| 1 | 会議室・設備情報が登録・更新された内容で保存されている |
| 2 | 利用停止にした会議室は新規予約の対象から外れている |

**入力データ**

| 情報 | 要否 | 内容 |
|---|---|---|
| 会議室名 | 必須 | 会議室の名称 |
| 収容人数 | 必須 | 会議室に収容できる人数 |
| 設置場所 | 任意 | 会議室の場所 |
| 利用単価 | 必須 | 1時間あたりの利用単価(0は無料) |
| 利用状態 | 必須 | 会議室の利用可否(利用可 / 利用停止) |
| 備え付け設備 | 任意 | 会議室に紐付ける設備 |
| 設備名 | 必須 | 設備を登録する際の設備の名称 |

**出力データ**

| 情報 | 内容 |
|---|---|
| 登録・編集結果 | 会議室・設備情報が保存されたことが分かる情報 |
| 会議室情報 | 登録・編集された会議室の内容 |

**基本フロー**

| Step | アクター | 操作/処理 | 結果 |
|---|---|---|---|
| 1 | 管理者 | 会議室・設備の管理を開く | 管理画面が表示される |
| 2 | 管理者 | 会議室を新規登録、または既存の会議室を選択する | 入力に進む |
| 3 | 管理者 | 会議室名・収容人数・場所・利用単価・利用状態・設備を入力して保存する | 保存が要求される |
| 4 | システム | 会議室・設備情報を登録・更新する | 情報が保存される |
| 5 | システム | 完了メッセージを表示する | 完了メッセージが表示される |

**代替フロー**

| ALT ID | 分岐Step | 条件 | フロー |
|---|---|---|---|
| ALT-1 | Step2 | 会議室に紐付ける設備が未登録 | 設備を新規に登録し、会議室に紐付ける |

**例外フロー**

| EXC ID | 発生Step | 条件 | 対応 |
|---|---|---|---|
| EXC-1 | Step4 | 会議室名・収容人数・利用単価のいずれかが未入力 | 必須項目の入力を促し、Step3 へ戻る |
