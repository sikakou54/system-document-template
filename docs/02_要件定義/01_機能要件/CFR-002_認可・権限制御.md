# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | CFR-002 |
| 要件名 | 認可・権限制御 |
| 概要 | 利用者のロールに応じて、利用可能な機能とアクセス可能なデータを制御する |
| 目的 | 権限外の操作・データ参照を防ぎ、ロールに応じた利用範囲を保証する |
| 適用範囲 | 全機能 |
| 対象ロール | 全ユーザー(一般 / 管理者) |

# 2. 共通機能要件

適用範囲の各機能に共通して課す振る舞いを定義する。

| 項目 | 内容 |
|---|---|
| 利用者の操作 | 自身のロールで許可された機能を利用し、許可されたデータを参照する |
| システムの処理 | 利用者のロール(一般 / 管理者)に応じて利用可能な機能とアクセス可能なデータを制御し、許可されていない機能の実行・データの参照は拒否する |

# 3. 適用条件

この共通機能が適用される対象と適用外を定義する。

| 区分 | 内容 |
|---|---|
| 適用対象 | すべての機能。各機能の対象ロールは個別の機能要件(FR)で定める |
| 適用外 | 認証そのもの(CFR-001 で扱う) |

# 4. 業務ルール

この共通機能の判断基準となる業務上のルールを定義する。

| No | ルール |
|---|---|
| 1 | 利用者は自身に許可されていない機能を利用できない(RULE-COM-001) |
| 2 | 利用者は自身に許可されていないデータを参照できない(RULE-COM-002) |
| 3 | 利用者区分は「利用者(一般)」と「管理者」とする |

# 5. 受入条件

この要件を満たしていることを確認する条件を定義する。

| No | 条件 |
|---|---|
| 1 | 一般ユーザーは管理者専用の機能を実行できない |
| 2 | 利用者は自身に許可されていないデータを参照できない |
| 3 | 各機能が許可するロールに従って、利用可否が判定される |

# 6. 前提条件

この要件が成り立つために、先に満たしておく条件を定義する。

| No | 条件 |
|---|---|
| 1 | 利用者が認証されている(CFR-001) |
| 2 | 各機能の対象ロールが個別の機能要件(FR)で定義されている |

# 7. 制限事項

この要件での制限や範囲外などを定義する。

| No | 制限 |
|---|---|
| 1 | 権限判定の実現方式(認可の実施箇所・データ範囲の判定方法)およびロールのコード値は機能設計で定義し、本要件では機能×ロールの利用可否(§9)までを扱う |

# 8. システムユースケース

この共通機能要件を実現する利用者・システムの振る舞いを、ユースケース単位で定義する。

なし(認可は各機能のシステムユースケース内で判定され、独立した利用シナリオを持たない)。

# 9. 権限マトリクス(機能×ロール)

この共通機能要件が制御する、機能×ロールの利用可否とデータ範囲を集約して定義する。ロール区分は CFR-000(利用者区分: 利用者(一般)/ 管理者)に基づく。凡例: ○=利用可 / ×=利用不可。

| 機能 | 利用者(一般) | 管理者 | 補足(データ範囲・前提) |
|---|---|---|---|
| ログイン(CFR-001) | ○ | ○ | - |
| 会議室検索(FR-001) | ○ | ○ | - |
| 会議室予約(FR-002) | ○ | ○ | 有料会議室は支払い方法の事前登録が前提(FR-008) |
| 支払い方法登録・請求確認(FR-008) | ○ | ○ | 本人の支払い方法・請求のみ(RULE-COM-002) |
| 予約変更・キャンセル(FR-003) | ○ | ○ | 本人の予約のみ(RULE-COM-002) |
| 会議室・設備管理(FR-005) | × | ○ | 管理者のみ(RULE-COM-001) |
| 利用実績レポート(FR-006) | × | ○ | 管理者のみ(RULE-COM-001) |
