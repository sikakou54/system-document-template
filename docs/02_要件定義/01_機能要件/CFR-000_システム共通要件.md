# 1. 概要

MeetRoom のシステム全体または複数機能に共通する要件を統括する親ファイル。

- システム全体の要件、共通機能要件(CFR)の一覧、共通業務ルール(RULE-COM)、共通制限事項を定義する。
- 非機能要件(NFR)は 02_要件定義/02_非機能要件/ に分離して管理する。
- 各共通機能要件(CFR)の詳細は本ディレクトリの CFR-XXX_要件名.md に記載する（本ファイルは一覧のみ）。
- 個別機能に固有の要件は各 FR-XXX_機能名.md に記載し、本ファイルには重複記載しない（採番の詳細は 99_その他/ID採番ルール.md）。

# 2. システム全体の要件

システム全体で満たすべき要件を定義する。

| No | 要件 |
|---|---|
| 1 | 本システムの全機能は、認証済みの登録利用者のみが利用できる |
| 2 | 利用者区分は「利用者(一般)」と「管理者」とし、各機能の対象ロールは個別の機能要件(FR)で定める |
| 3 | 利用日時は日本時間(Asia/Tokyo)を基準に扱う |

# 3. 共通機能要件(CFR)一覧

複数機能に横断して適用される共通の機能要件を一覧で管理する。各 CFR の詳細は CFR-XXX_要件名.md に記載する。

| ID | 名称 | 概要 | 適用範囲 |
|---|---|---|---|
| CFR-001 | 認証 | 登録利用者が本人確認(ログイン)を経てシステムを利用可能な状態にする | 全機能 |
| CFR-002 | 認可・権限制御 | 利用者のロールに応じて利用可能な機能・データを制御する | 全機能 |
| CFR-003 | 入力検証 | 利用者の入力の妥当性(必須・型・形式・範囲・項目間相関)を検証し不正入力を受け付けない | 利用者入力を受け付ける全機能 |
| CFR-004 | エラー表示 | エラー発生時に利用者が理解できる形で内容を通知する | 全機能 |
| CFR-005 | 一覧検索・ページング | 一覧表示で検索・絞り込みとページングを可能にする | 一覧表示を行う機能 |

# 4. 非機能要件(NFR)

非機能要件(NFR)は 02_要件定義/02_非機能要件/ に個別文書(NFR-XXX)として集約し、本ファイルでは管理しない。

- 機能横断で満たすべき品質・特定機能固有の品質のいずれも同ディレクトリで定義する。
- 一覧は 02_要件定義/02_非機能要件/index.md、FR との適用関係は 02_要件定義/04_トレーサビリティ/traceability_fr_nfr.md を参照する。

# 5. 共通業務ルール(RULE-COM)

複数機能に共通して適用される業務上の判断基準・制約を定義する。個別機能固有の業務ルールは各 FR に記載する。

| ID | ルール名 | 内容 |
|---|---|---|
| RULE-COM-001 | 権限外操作禁止 | 利用者は自身に許可されていない機能を利用できない |
| RULE-COM-002 | 権限外データ参照禁止 | 利用者は自身に許可されていないデータを参照できない |
| RULE-COM-003 | 削除済みデータ非表示 | 論理削除(削除日時 が非NULL)として扱うデータは、通常の検索・一覧に表示しない |
