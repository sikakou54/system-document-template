# FR-002：会議室予約機能 仕様書

## 1. 基本情報
| 項目 | 内容 |
|---|---|
| 要件ID | FR-002 |
| 要件名 | 会議室予約 |
| 概要 | 空き会議室を予約できる。同一会議室・同一時間帯の二重予約は不可 |
| 目的 | 会議室のダブルブッキングを防ぎ、確実に会議室を確保できるようにする |
| 対象ロール | 利用者 |

## 2. 業務ルール（制約事項）
システムが遵守すべき業務上の絶対ルールを定義する。このルールがユースケースのシナリオ分岐の根拠となる。

| No | ルール |
|---|---|
| 1 | 同一会議室で時間帯が重複する予約は登録できない |
| 2 | 過去日時の予約は登録できない |
| 3 | 予約は開始時刻＜終了時刻であること |
| 4 | 予約変更と予約キャンセルはこの要件では扱わない |
| 5 | 予約できる会議室は、検索結果で空きとして表示された会議室に限る |

## 3. 受入条件
この要件を満たしていることを確認する条件（テストの合否判定基準）を定義する。

| No | 条件 |
|---|---|
| 1 | 空き会議室を選んで予約すると予約が登録され、マイ予約一覧に表示される |
| 2 | 既存予約と時間帯が重複する場合は登録されずエラーになる |
| 3 | 過去日時を指定すると登録されずエラーになる |
| 4 | 開始時刻が終了時刻以降の場合は登録されずエラーになる |

## 4. システムユースケース
この要件を実現する振る舞いを、データなどのパターンに応じてユースケース単位で定義する。

### 4.1 【UC-01】会議室を予約する（一般ユーザー向け）
空き会議室を検索・選択し、タイトルを入力して予約を登録する。

**事前条件**

| No | 条件 |
|---|---|
| 1 | ログイン済み |
| 2 | 予約したい日時が決まっている |
| 3 | 予約対象の会議室が登録済みである |

**事後条件**

| No | 条件 |
|---|---|
| 1 | 予約が「予約済」状態で登録されている |
| 2 | マイ予約一覧に表示される |

**入力データ**

| 情報 | 要否 | 内容 |
|---|---|---|
| 予約対象の会議室 | 必須 | 予約する会議室(検索結果で空きとして表示された会議室) |
| 利用開始日時 | 必須 | 利用を開始する日時 |
| 利用終了日時 | 必須 | 利用を終了する日時 |
| 予約タイトル | 必須 | 予約の用件・名称 |

**出力データ**

| 情報 | 内容 |
|---|---|
| 予約内容 | 登録された予約(会議室・利用日時・タイトル) |
| 登録結果 | 予約が成立したことが分かる情報 |
| マイ予約一覧で確認可能な予約情報 | 予約者本人が後から確認できる予約情報 |

**基本フロー**

| Step | アクター | 操作/処理 | 結果 |
|---|---|---|---|
| 1 | 利用者 | 日時・人数を入力し検索する | 空き会議室一覧が表示される |
| 2 | 利用者 | 会議室を選択する | 予約入力に進む |
| 3 | 利用者 | タイトルを入力し予約ボタンを押下する | 予約登録が要求される |
| 4 | システム | 予約を登録する | 予約が登録される |
| 5 | システム | 完了メッセージを表示する | 完了メッセージが表示される |

**代替フロー**

| ALT ID | 分岐Step | 条件 | フロー |
|---|---|---|---|
| ALT-1 | Step1 | 空き会議室が0件 | 条件を変えて再検索する |

**例外フロー**

| EXC ID | 発生Step | 条件 | 対応 |
|---|---|---|---|
| EXC-1 | Step4 | 時間帯重複(他者が先に予約) | エラー表示、Step1 へ戻る |
| EXC-2 | Step4 | 過去日時 | エラー表示、Step3 へ戻る |
