# エンティティ定義：会議室設備

## 1. 基本情報
* **エンティティID:** MDL-008
* **概要:** どの会議室にどの設備が設置されているかの紐付け（会議室と設備の多対多関係）
* **データ管理者:** 総務・管理部門

## 2. データ項目定義
当エンティティが保持する具体的なデータ項目（属性）の詳細を定義する。

| No | 属性名 (論理名) | データ型 | 長さ | 主キー | 必須 | 説明 (初期値など) |
|:---|:---|:---|:---:|:---:|:---|:---|
| 1 | 会議室設備ID | 数値 | - | 〇 | 〇 | システムで自動採番 |
| 2 | 会議室ID | 数値 | - | - | 〇 | 紐付け先の会議室（会議室(MDL-002)を参照） |
| 3 | 設備ID | 数値 | - | - | 〇 | 紐付ける設備（設備(MDL-004)を参照） |

## 3. データライフサイクル・制約事項（CRUD）
本データの発生から廃棄までのルールやデータ操作における制約を定義する。

| 操作 | ルール・制約 |
|:---|:---|
| 作成 (Create) | 管理者が会議室・設備管理機能（FR-005）で、会議室に設備を紐付けたときに登録される |
| 参照 (Read) | 会議室検索（FR-001）の設備条件や会議室・設備管理から参照される |
| 更新 (Update) | 紐付けの内容は更新せず、追加・解除で管理する |
| 削除 (Delete) | 会議室から設備を外したとき、当該紐付けを削除する。紐付け(中間データ)は会議室・設備の関連そのものであり、論理削除(RULE-COM-003)の対象とせず、参照整合性のため即時に物理削除する(履歴保持は行わない) |

## 4. 関連機能・業務ルール
本データが影響を与える機能や、依存する業務ルールを定義する。

**関連機能（機能要件）**

| 関連要件ID | 要件名 |
|:---|:---|
| FR-001 | 会議室検索 |
| FR-005 | 会議室・設備管理 |

**業務ルールへの影響**

| No | 内容 |
|:---|:---|
| 1 | 同一会議室に同一設備を重複して紐付けない（会議室と設備の組は一意） |

## 5. セキュリティ・アクセス権限
本データに対する参照・更新権限や機密性を定義する。

| 項目 | 内容 |
|:---|:---|
| 参照権限 | システムにログインしている全ユーザーが参照可能 |
| 更新権限 | 「管理者権限」を持つユーザーのみ紐付けの追加・削除が可能 |
| 機密性レベル | 低（社内公開情報であり、特別な保護は不要） |
