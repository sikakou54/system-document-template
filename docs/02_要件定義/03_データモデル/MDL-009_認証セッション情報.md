# エンティティ定義：認証セッション

## 1. 基本情報
* **エンティティID:** MDL-009
* **概要:** 利用者のログイン認証に伴う一時的な認証状態（一時データとして保持）
* **データ管理者:** システム管理部門

## 2. データ項目定義
当エンティティが保持する具体的なデータ項目（属性）の詳細を定義する。

| No | 属性名 (論理名) | データ型 | 長さ | 主キー | 必須 | 説明 (初期値など) |
|:---|:---|:---|:---:|:---:|:---|:---|
| 1 | 認証セッション識別子 | 文字列 | - | 〇 | 〇 | 認証セッションを一意に識別する情報 |
| 2 | 利用者ID | 数値 | - | - | 〇 | 認証対象の利用者（利用者(MDL-001)を参照） |
| 3 | 発行日時 | 日時 | - | - | 〇 | 認証セッションを発行した日時 |
| 4 | 有効期限 | 日時 | - | - | 〇 | この認証状態が有効な期限 |
| 5 | 失効状態 | 数値 | 1 | - | 〇 | `1`:有効, `2`:失効。初期値は`1` |

## 3. データライフサイクル・制約事項（CRUD）
本データの発生から廃棄までのルールやデータ操作における制約を定義する。

| 操作 | ルール・制約 |
|:---|:---|
| 作成 (Create) | ログイン認証（CFR-001）の成功時にシステムが発行する |
| 参照 (Read) | 各操作の認証・認可（CFR-001／CFR-002）でセッションの有効性を確認するために参照される |
| 更新 (Update) | ログアウト・失効時にシステムが失効状態を更新する |
| 削除 (Delete) | 一時データのため、有効期限の経過により自動的に消失する |

## 4. 関連機能・業務ルール
本データが影響を与える機能や、依存する業務ルールを定義する。

**関連機能（機能要件）**

| 関連要件ID | 要件名 |
|:---|:---|
| CFR-001 | 認証 |
| CFR-002 | 認可・権限制御 |

**業務ルールへの影響**

| No | 内容 |
|:---|:---|
| 1 | 有効期限を過ぎた、または失効した認証セッションは認証に使用できない |

## 5. セキュリティ・アクセス権限
本データに対する参照・更新権限や機密性を定義する。

| 項目 | 内容 |
|:---|:---|
| 参照権限 | システム（認証基盤）のみが参照する（利用者・管理者が直接参照する情報ではない） |
| 更新権限 | システムが発行・失効させる |
| 機密性レベル | 高（認証トークンに相当し、漏えいするとなりすましに悪用されうる。通信暗号化・失効管理で保護。NFR-003） |
