# エンティティ定義：利用量記録

## 1. 基本情報
* **エンティティID:** MDL-006
* **概要:** 従量課金の対象となる、完了予約ごとの利用量（利用時間・金額）の記録
* **データ管理者:** 経理・管理部門

## 2. データ項目定義
当エンティティが保持する具体的なデータ項目（属性）の詳細を定義する。

| No | 属性名 (論理名) | データ型 | 長さ | 主キー | 必須 | 説明 (初期値など) |
|:---|:---|:---|:---:|:---:|:---|:---|
| 1 | 利用量記録ID | 数値 | - | 〇 | 〇 | システムで自動採番 |
| 2 | 予約ID | 数値 | - | - | 〇 | 対応する完了予約（予約(MDL-003)を参照。完了予約1件につき1記録） |
| 3 | 利用者ID | 数値 | - | - | 〇 | 課金対象の利用者（利用者(MDL-001)を参照） |
| 4 | 会議室ID | 数値 | - | - | 〇 | 利用した会議室（会議室(MDL-002)を参照） |
| 5 | 利用時間 | 数値 | 8 | - | 〇 | 従量課金対象の利用時間（分） |
| 6 | 単価 | 数値 | 6 | - | 〇 | 計上時点の時間あたり単価（円） |
| 7 | 金額 | 数値 | 8 | - | 〇 | 利用時間×単価で算出した課金金額（円） |
| 8 | 外部決済サービス利用量イベント識別 | 文字列 | 64 | - | - | 外部決済サービスへ送信した利用量イベントを対応づける識別情報 |
| 9 | 計上状態 | 数値 | 1 | - | 〇 | `1`:未送信, `2`:送信済, `3`:失敗。初期値は`1` |

## 3. データライフサイクル・制約事項（CRUD）
本データの発生から廃棄までのルールやデータ操作における制約を定義する。

| 操作 | ルール・制約 |
|:---|:---|
| 作成 (Create) | 利用量の従量課金計上の定期処理（JOB）が、完了予約ごとに登録する |
| 参照 (Read) | 課金処理・利用量計上および利用量・請求の確認から参照される |
| 更新 (Update) | 計上状態・利用量イベント識別はシステム（計上処理）が更新する |
| 削除 (Delete) | 課金の証跡のため削除しない |

## 4. 関連機能・業務ルール
本データが影響を与える機能や、依存する業務ルールを定義する。

**関連機能（機能要件）**

| 関連要件ID | 要件名 |
|:---|:---|
| FR-008 | 会議室利用の従量課金 |

**業務ルールへの影響**

| No | 内容 |
|:---|:---|
| 1 | 完了した予約1件につき利用量記録は1件 |
| 2 | 金額は利用時間×単価で算出し、0以上であること |

## 5. セキュリティ・アクセス権限
本データに対する参照・更新権限や機密性を定義する。

| 項目 | 内容 |
|:---|:---|
| 参照権限 | 課金業務の範囲で管理者が参照可能 |
| 更新権限 | システム（計上処理）が作成・更新する |
| 機密性レベル | 中（課金額に関わる情報を含む） |
