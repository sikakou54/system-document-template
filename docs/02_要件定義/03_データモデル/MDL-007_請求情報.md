# エンティティ定義：請求

## 1. 基本情報
* **エンティティID:** MDL-007
* **概要:** 利用者ごと・請求対象期間ごとの月次請求
* **データ管理者:** 経理・管理部門

## 2. データ項目定義
当エンティティが保持する具体的なデータ項目（属性）の詳細を定義する。

| No | 属性名 (論理名) | データ型 | 長さ | 主キー | 必須 | 説明 (初期値など) |
|:---|:---|:---|:---:|:---:|:---|:---|
| 1 | 請求ID | 数値 | - | 〇 | 〇 | システムで自動採番 |
| 2 | 利用者ID | 数値 | - | - | 〇 | 請求先の利用者（利用者(MDL-001)を参照） |
| 3 | 請求対象期間 | 文字列 | 7 | - | 〇 | 請求の対象期間（YYYY-MM）。利用者ごと・期間ごとに1件 |
| 4 | 請求金額 | 数値 | 8 | - | 〇 | 期間内の課金合計金額（円） |
| 5 | 請求状態 | 数値 | 1 | - | 〇 | `1`:確定前, `2`:請求済, `3`:支払済, `4`:失敗。初期値は`1` |
| 6 | 外部決済サービス請求識別 | 文字列 | 64 | - | 〇 | 外部決済サービス上の請求を対応づける識別情報（一意） |

## 3. データライフサイクル・制約事項（CRUD）
本データの発生から廃棄までのルールやデータ操作における制約を定義する。

| 操作 | ルール・制約 |
|:---|:---|
| 作成 (Create) | 従量課金の請求処理・外部決済サービスからの請求イベント受信により、対象期間ごとに登録される |
| 参照 (Read) | 支払い方法・請求の確認（FR-008）から利用者本人が参照する |
| 更新 (Update) | 請求状態・請求金額は外部決済サービスの請求イベントに応じてシステムが更新する |
| 削除 (Delete) | 課金の証跡のため削除しない |

## 4. 関連機能・業務ルール
本データが影響を与える機能や、依存する業務ルールを定義する。

**関連機能（機能要件）**

| 関連要件ID | 要件名 |
|:---|:---|
| FR-008 | 会議室利用の従量課金 |

**業務ルールへの影響**

| No | 内容 |
|:---|:---|
| 1 | 利用者と請求対象期間の組は一意（同一利用者・同一期間の請求は1件） |
| 2 | 請求金額は0以上であること |

## 5. セキュリティ・アクセス権限
本データに対する参照・更新権限や機密性を定義する。

| 項目 | 内容 |
|:---|:---|
| 参照権限 | 請求先本人および課金業務の範囲で管理者が参照可能 |
| 更新権限 | システム（請求処理）が作成・更新する |
| 機密性レベル | 中（請求・支払いに関わる情報を含む） |
