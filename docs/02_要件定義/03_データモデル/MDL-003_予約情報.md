# エンティティ定義：予約

## 1. 基本情報
* **エンティティID:** MDL-003
* **概要:** 利用者が会議室を特定の時間帯で予約した記録
* **データ管理者:** 各利用者（予約者本人）

## 2. データ項目定義
当エンティティが保持する具体的なデータ項目（属性）の詳細を定義する。

| No | 属性名 (論理名) | データ型 | 長さ | 主キー | 必須 | 説明 (初期値など) |
|:---|:---|:---|:---:|:---:|:---|:---|
| 1 | 予約ID | 数値 | - | 〇 | 〇 | システムで自動採番 |
| 2 | 会議室ID | 数値 | - | - | 〇 | 予約対象の会議室（会議室(MDL-002)を参照） |
| 3 | 利用者ID | 数値 | - | - | 〇 | 予約者（利用者(MDL-001)を参照） |
| 4 | 予約タイトル | 文字列 | 100 | - | 〇 | 会議・利用の件名 |
| 5 | 利用開始日時 | 日時 | - | - | 〇 | 会議室の利用開始日時 |
| 6 | 利用終了日時 | 日時 | - | - | 〇 | 会議室の利用終了日時 |
| 7 | 予約状態 | 数値 | 1 | - | 〇 | `1`:予約済, `2`:キャンセル, `3`:完了。初期値は`1` |
| 8 | リマインド通知状態 | 数値 | 1 | - | 〇 | `1`:未送信, `2`:送信済, `3`:失敗。初期値は`1` |
| 9 | 予約確定時単価 | 数値 | 6 | - | 〇 | 予約を確定した時点の会議室の時間単価(円/時)。無料は`0`。確定後は会議室の単価変更に追随しない |

## 3. データライフサイクル・制約事項（CRUD）
本データの発生から廃棄までのルールやデータ操作における制約を定義する。

| 操作 | ルール・制約 |
|:---|:---|
| 作成 (Create) | 利用者が会議室予約（FR-002）で登録する |
| 参照 (Read) | 予約一覧・変更・キャンセル（FR-003）、予約リマインド通知（FR-004）などから参照される |
| 更新 (Update) | 予約者本人が予約内容の変更・キャンセルを行える（FR-003）。予約状態・リマインド通知状態はシステムが更新する。予約確定時単価は予約後に更新しない |
| 削除 (Delete) | システム上からの完全消去は行わず、キャンセル（予約状態=2）または削除済みとして扱う論理削除とする（RULE-COM-003） |

## 4. 関連機能・業務ルール
本データが影響を与える機能や、依存する業務ルールを定義する。

**関連機能（機能要件）**

| 関連要件ID | 要件名 |
|:---|:---|
| FR-002 | 会議室予約 |
| FR-003 | 予約変更・キャンセル |
| FR-004 | 予約リマインド通知 |
| FR-008 | 会議室利用の従量課金 |

**業務ルールへの影響**

| No | 内容 |
|:---|:---|
| 1 | 同一会議室・同一時間帯の二重予約は不可 |
| 2 | 利用開始日時は利用終了日時より前であること |
| 3 | 予約確定時単価は予約確定時の会議室利用単価を保存し、後続の会議室単価変更では変えない |

## 5. セキュリティ・アクセス権限
本データに対する参照・更新権限や機密性を定義する。

| 項目 | 内容 |
|:---|:---|
| 参照権限 | 予約者本人および管理者が参照可能 |
| 更新権限 | 予約者本人が自身の予約を変更・キャンセルできる |
| 機密性レベル | 中（個人の予定・利用情報を含む） |
