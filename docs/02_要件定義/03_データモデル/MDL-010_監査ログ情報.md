# エンティティ定義：監査ログ

## 1. 基本情報
* **エンティティID:** MDL-010
* **概要:** 予約・課金・会議室管理などの重要操作の記録（監査証跡）
* **データ管理者:** システム管理部門

## 2. データ項目定義
当エンティティが保持する具体的なデータ項目（属性）の詳細を定義する。

| No | 属性名 (論理名) | データ型 | 長さ | 主キー | 必須 | 説明 (初期値など) |
|:---|:---|:---|:---:|:---:|:---|:---|
| 1 | 監査ログID | 数値 | - | 〇 | 〇 | システムで自動採番 |
| 2 | 操作日時 | 日時 | - | - | 〇 | 操作が行われた日時 |
| 3 | 利用者ID | 数値 | - | - | 〇 | 操作を行った利用者（利用者(MDL-001)を参照） |
| 4 | 操作種別 | 文字列 | 20 | - | 〇 | 記録対象の操作の種別（予約操作／課金操作／会議室管理操作 など） |
| 5 | 操作対象種別 | 文字列 | 50 | - | 〇 | 予約、会議室、設備、課金契約など操作対象の種別 |
| 6 | 操作対象ID | 文字列 | 100 | - | 〇 | 操作対象を一意に特定する識別情報 |
| 7 | 操作結果 | 文字列 | 10 | - | 〇 | 操作の結果（成功／失敗） |
| 8 | 相関ID | 文字列 | 64 | - | 〇 | API・JOB・外部イベント・関連ログを横断して同一処理を追跡する識別情報 |

## 3. データライフサイクル・制約事項（CRUD）
本データの発生から廃棄までのルールやデータ操作における制約を定義する。

| 操作 | ルール・制約 |
|:---|:---|
| 作成 (Create) | 重要操作（予約・課金・会議室管理など）の実行時に、システムが記録する（CFR-007／NFR-006） |
| 参照 (Read) | 監査・追跡の際に管理者が参照する |
| 更新 (Update) | 追記のみとし、記録後の内容は更新しない |
| 削除 (Delete) | 作成から1年間は削除しない。保持期間満了後のみ、監査担当が承認した運用処理で削除する（NFR-006） |

## 4. 関連機能・業務ルール
本データが影響を与える機能や、依存する業務ルールを定義する。

**関連機能（機能要件）**

| 関連要件ID | 要件名 |
|:---|:---|
| CFR-007 | 監査ログ記録 |
| NFR-006 | 監査 |

**業務ルールへの影響**

| No | 内容 |
|:---|:---|
| 1 | 予約・課金・会議室管理などの重要操作は必ず記録する |
| 2 | 作成から1年間は記録済み監査ログを更新・削除しない（追記のみ）。保持期間満了後の削除は承認済み運用処理に限る |

## 5. セキュリティ・アクセス権限
本データに対する参照・更新権限や機密性を定義する。

| 項目 | 内容 |
|:---|:---|
| 参照権限 | 監査担当の管理者のみが参照可能 |
| 更新権限 | システムが追記のみ行う。記録後の変更・削除は不可 |
| 機密性レベル | 中（操作履歴。改ざん・消去の防止が求められる） |
