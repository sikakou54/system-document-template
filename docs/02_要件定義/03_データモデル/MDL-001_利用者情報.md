# エンティティ定義：利用者

## 1. 基本情報
* **エンティティID:** MDL-001
* **概要:** システムを利用する社員・管理者の基本情報（認証情報・従量課金の契約情報を含む）
* **データ管理者:** システム管理部門

## 2. データ項目定義
当エンティティが保持する具体的なデータ項目（属性）の詳細を定義する。

| No | 属性名 (論理名) | データ型 | 長さ | 主キー | 必須 | 説明 (初期値など) |
|:---|:---|:---|:---:|:---:|:---|:---|
| 1 | 利用者ID | 数値 | - | 〇 | 〇 | システムで自動採番 |
| 2 | 氏名 | 文字列 | 50 | - | 〇 | 利用者の氏名 |
| 3 | メールアドレス | 文字列 | 256 | - | 〇 | ログインID兼連絡先。利用者を一意に識別する（システム内で一意） |
| 4 | パスワード | 文字列 | - | - | 〇 | ログイン時の本人確認に用いる認証情報（ハッシュ化して保存。NFR-003） |
| 5 | ロール | 数値 | 1 | - | 〇 | `1`:一般, `2`:管理者。初期値は`1` |
| 6 | 課金契約状態 | 数値 | 1 | - | 〇 | `1`:未契約, `2`:有効, `3`:停止。初期値は`1` |
| 7 | 外部決済サービス顧客識別 | 文字列 | 64 | - | - | 従量課金を行う外部決済サービス上の顧客を対応づける識別情報 |
| 8 | 外部決済サービス契約識別 | 文字列 | 64 | - | - | 従量課金のサブスクリプション契約を対応づける識別情報 |

## 3. データライフサイクル・制約事項（CRUD）
本データの発生から廃棄までのルールやデータ操作における制約を定義する。

| 操作 | ルール・制約 |
|:---|:---|
| 作成 (Create) | アプリ上の登録（サインアップ）機能は持たず（スコープ外）、運用手順によりシステム管理部門がレコードを投入（シード）する |
| 参照 (Read) | 認証（CFR-001）・認可（CFR-002）および各機能から本人・操作者として随時参照される |
| 更新 (Update) | 課金契約状態・外部決済サービス識別は支払い方法登録（FR-008）に伴いシステムが更新する。本人情報の管理は管理者権限で行う |
| 削除 (Delete) | システム上からの完全消去は行わず、削除済みとして扱う論理削除とする（RULE-COM-003） |

## 4. 関連機能・業務ルール
本データが影響を与える機能や、依存する業務ルールを定義する。

**関連機能（機能要件）**

| 関連要件ID | 要件名 |
|:---|:---|
| CFR-001 | 認証 |
| CFR-002 | 認可・権限制御 |
| FR-008 | 会議室利用の従量課金 |

**業務ルールへの影響**

| No | 内容 |
|:---|:---|
| 1 | ロールにより利用可能な機能を制御する（管理者／一般） |
| 2 | 課金契約状態=有効の利用者のみ有料会議室を利用できる |

## 5. セキュリティ・アクセス権限
本データに対する参照・更新権限や機密性を定義する。

| 項目 | 内容 |
|:---|:---|
| 参照権限 | 本人の情報は本人が、管理業務の範囲で管理者が参照可能 |
| 更新権限 | 本人情報の管理・登録は「管理者権限」を持つユーザーが行う。課金関連はシステムが更新する |
| 機密性レベル | 高（認証情報・個人情報を含む。パスワードはハッシュ化して保存し、暗号化通信・バックアップで保護。NFR-003/NFR-005） |
