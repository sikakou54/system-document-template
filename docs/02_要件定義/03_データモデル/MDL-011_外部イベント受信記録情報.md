# エンティティ定義：外部イベント受信記録

## 1. 基本情報
* **エンティティID:** MDL-011
* **概要:** 外部サービスから受信した契約・請求イベントを一意に識別し、重複・順序逆転・一時失敗に対して安全に再処理するための受信記録
* **データ管理者:** システム管理部門

## 2. データ項目定義
当エンティティが保持する具体的なデータ項目（属性）の詳細を定義する。

| No | 属性名 (論理名) | データ型 | 長さ | 主キー | 必須 | 説明 (初期値など) |
|:---|:---|:---|:---:|:---:|:---|:---|
| 1 | 外部イベント受信記録ID | 数値 | - | 〇 | 〇 | システムで自動採番 |
| 2 | 外部サービス | 文字列 | 30 | - | 〇 | イベント送信元のサービス識別 |
| 3 | 外部イベントID | 文字列 | 128 | - | 〇 | 送信元が付与した一意なイベント識別。外部サービスとの組で一意 |
| 4 | イベント種別 | 文字列 | 100 | - | 〇 | 契約有効化、契約状態変更、請求成功、請求失敗などの種別 |
| 5 | イベント発生日時 | 日時 | - | - | 〇 | 送信元でイベントが発生した日時。状態の順序判定に用いる |
| 6 | 受信日時 | 日時 | - | - | 〇 | システムが真正性を検証したイベントを受信した日時 |
| 7 | 処理状態 | 数値 | 1 | - | 〇 | `1`:受信済, `2`:処理済, `3`:失敗, `4`:対象外。初期値は`1` |
| 8 | 処理完了日時 | 日時 | - | - | - | 処理済または対象外として確定した日時 |
| 9 | 処理試行回数 | 数値 | 4 | - | 〇 | 初期値は`0`。処理を試行するたびに増加する |
| 10 | 最終エラー概要 | 文字列 | 500 | - | - | 最後に処理失敗した理由。秘密情報・決済情報本文は含めない |
| 11 | 相関ID | 文字列 | 64 | - | 〇 | API・請求・監査ログを横断して処理を追跡する識別 |
| 12 | 関連利用者ID | 数値 | - | - | - | イベントから識別できる場合の利用者（利用者(MDL-001)を参照） |
| 13 | 関連請求識別 | 文字列 | 128 | - | - | イベントが請求に関する場合の外部請求識別 |
| 14 | 再処理情報 | 文字列 | - | - | 〇 | 状態反映を再現するために必要な最小限のイベント情報。決済手段の機微情報は保持しない |

## 3. データライフサイクル・制約事項（CRUD）
本データの発生から廃棄までのルールやデータ操作における制約を定義する。

| 操作 | ルール・制約 |
|:---|:---|
| 作成 (Create) | 外部イベントの真正性検証後、状態反映より先に外部サービスと外部イベントIDの組で一意に作成する。同じイベントを再受信した場合は新規作成しない |
| 参照 (Read) | 重複判定、イベント順序判定、失敗イベントの再処理、障害調査で参照する |
| 更新 (Update) | システムが処理状態・完了日時・試行回数・最終エラー概要を更新する。イベント識別・種別・発生日時・再処理情報は作成後に変更しない |
| 削除 (Delete) | 処理済みまたは対象外となった日から1年間保持し、期間満了後に運用処理で削除する。未解決の失敗状態は削除しない |

## 4. 関連機能・業務ルール
本データが影響を与える機能や、依存する業務ルールを定義する。

**関連機能（機能要件）**

| 関連要件ID | 要件名 |
|:---|:---|
| FR-008 | 会議室利用の従量課金 |
| NFR-007 | JOB・外部連携信頼性 |
| NFR-008 | 監視・障害検知 |

**業務ルールへの影響**

| No | 内容 |
|:---|:---|
| 1 | 外部サービスと外部イベントIDの組は一意とし、同じイベントを複数回反映しない |
| 2 | 処理済み状態から失敗・受信済みへ戻さない |
| 3 | 現在の契約・請求状態を過去へ戻す遅延イベントは処理済みとして記録するが、業務状態へ反映しない |
| 4 | 請求・契約状態の更新とイベントの処理済み更新は同じ確定単位で扱う |

## 5. セキュリティ・アクセス権限
本データに対する参照・更新権限や機密性を定義する。

| 項目 | 内容 |
|:---|:---|
| 参照権限 | 外部連携処理と障害調査を担当する管理者のみ参照できる |
| 更新権限 | 外部イベント処理を行うシステムのみ作成・更新できる |
| 機密性レベル | 高（契約・請求に関する情報を含む。再処理に不要な決済手段・秘密情報は保存しない） |
