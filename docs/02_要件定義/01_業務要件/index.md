# 1. 概要

MeetRoom で実現したい業務上の目的・到達状態を業務要件(BR)として定義する。各業務要件の詳細は BR-XXX_要件名.md に記載する。実現手段である機能要件(FR)・共通機能要件(CFR)・非機能要件(NFR)はトレース先として関連付ける。

# 2. 一覧

| ID | 名称 | 実現したい状態 | トレース先(実現FR/CFR/NFR) |
|---|---|---|---|
| BR-001 | 空き会議室を見つけやすくする | 日時・人数・設備から空き会議室を短時間で見つけられる | FR-001 |
| BR-002 | 重複予約をなくす | 同一会議室・同一時間帯の二重予約が発生しない | FR-002 |
| BR-003 | 予約を見直せるようにする | 予約者が自分の予約を変更・取消できる | FR-003 |
| BR-004 | 予約忘れを減らす | 予約開始前にリマインドが届く | FR-004 |
| BR-005 | 会議室・設備・料金を一元管理する | 管理者が会議室・設備・利用単価を最新に保てる | FR-005 |
| BR-006 | 利用状況を見える化する | 管理者が会議室ごとの利用実績を把握できる | FR-006 |
| BR-007 | 登録者だけが使える | 登録利用者のみが本人確認の上で利用する | CFR-001 |
| BR-008 | 利用量に応じて公平に課金する | 有料会議室の利用時間に応じ従量で課金し、月次で請求する | FR-008 |
| BR-009 | 共通操作を一貫して安全に扱う | 入力検証・エラー表示・一覧表示・削除済みデータ制御が機能間で一貫している | CFR-003, CFR-004, CFR-005 |
| BR-010 | 安定性・保護・監査を担保する | 業務時間中に安定し、認証情報・通信・業務データ・重要操作を保護・追跡できる | CFR-000, NFR-002, NFR-003, NFR-004, NFR-005, NFR-006 |
