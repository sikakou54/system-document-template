# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | BR-010 |
| 要件名 | 安定性・保護・監査を担保する |
| 概要 | 業務時間中の安定利用、認証情報・通信・業務データの保護、重要操作の追跡を実現する |
| 実現したい状態 | 業務時間中に安定し、認証情報・通信・業務データ・重要操作を保護・追跡できる |
| 背景 | 会議室予約は日常業務に使われるため、停止・情報漏えい・操作追跡不能が起きると業務継続や信頼性に影響する |
| 関連要求 | RQ-068〜RQ-073 |

# 2. 業務要件

この業務要件で実現したいことを定義する。

| No | 実現したいこと |
|---|---|
| 1 | 業務時間中に安定して利用できるようにする |
| 2 | パスワードを平文保存せず安全に保護する |
| 3 | 利用者との通信とシステムからの送信メールを信頼できる状態にする |
| 4 | 業務データをバックアップし、障害時のデータ消失に備える |
| 5 | 重要操作を後から確認できるようにする |
| 6 | 利用日時を日本時間を基準に扱う |

# 3. 成功指標・期待効果

この業務要件の達成度を測る指標と期待効果を定義する。

| No | 区分 | 内容 |
|---|---|---|
| 1 | 成功指標 | 業務時間中の稼働状況、保護、バックアップ、監査の各品質要求を満たしている |
| 2 | 期待効果 | 社員・管理者が日常業務で安心して会議室予約を利用できる |
| 3 | 期待効果 | 障害や重要操作の確認が必要になった場合に、復旧・追跡に必要な根拠を確保できる |

# 4. 関連(実現する要件)

この業務要件を実現する機能要件(FR)・共通機能要件(CFR)・非機能要件(NFR)を定義する。

| ID | 名称 | 実現内容 |
|---|---|---|
| CFR-000 | システム共通要件 | 全機能の認証前提と利用日時の日本時間基準を定義する |
| NFR-002 | 可用性 | 平日8:00〜20:00に安定して利用できる状態を定義する |
| NFR-003 | セキュリティ | パスワードを平文保存せず安全に保護する |
| NFR-004 | 通信暗号化 | 通信経路を暗号化し、送信メールのなりすましを防ぐ |
| NFR-005 | データ保護 | 業務データを定期的にバックアップする |
| NFR-006 | 監査 | 重要操作を記録し、後から追跡できるようにする |

# 5. 前提・制約

この業務要件の前提と制約を定義する。

| No | 区分 | 内容 |
|---|---|---|
| 1 | 前提 | 業務時間、保護対象、重要操作の範囲が要求定義で確認されている |
| 2 | 制約 | 具体的な実装方式・監視方式・バックアップ方式は機能設計で定義する |
