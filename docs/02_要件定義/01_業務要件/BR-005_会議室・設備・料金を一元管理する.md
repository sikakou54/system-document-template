# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | BR-005 |
| 要件名 | 会議室・設備・料金を一元管理する |
| 概要 | 管理者が会議室・設備・利用単価を登録・編集し、常に最新の状態に保てるようにする |
| 実現したい状態 | 管理者が会議室・設備・利用単価を最新に保てる |
| 背景 | 会議室・設備・料金の情報が散在し最新化されず、検索や課金の前提が不正確になっている |
| 関連要求 | RQ-004, RQ-029〜RQ-038 |

# 2. 業務要件

この業務要件で実現したいことを定義する。

| No | 実現したいこと |
|---|---|
| 1 | 管理者が会議室(名称・収容人数・場所・利用単価)を登録・編集できるようにする |
| 2 | 管理者が設備を登録し、会議室に紐付けられるようにする |
| 3 | 利用を停止した会議室は新規予約の対象から外せるようにする |
| 4 | 利用単価を0にすることで無料会議室として扱えるようにする |

# 3. 成功指標・期待効果

この業務要件の達成度を測る指標と期待効果を定義する。

| No | 区分 | 内容 |
|---|---|---|
| 1 | 成功指標 | 会議室・設備・利用単価の変更が管理者の操作で反映される |
| 2 | 期待効果 | 検索・予約・課金が常に最新の会議室情報に基づいて行われる |
| 3 | 期待効果 | 情報管理が一元化され、二重管理や記載ミスが減る |

# 4. 関連(実現する機能要件)

この業務要件を実現する機能要件(FR)・共通機能要件(CFR)を定義する。

| ID | 名称 | 実現内容 |
|---|---|---|
| FR-005 | 会議室・設備管理 | 管理者が会議室(利用単価を含む)と設備を登録・編集する |

# 5. 前提・制約

この業務要件の前提と制約を定義する。

| No | 区分 | 内容 |
|---|---|---|
| 1 | 前提 | 操作する利用者が管理者である |
| 2 | 制約 | 会議室・設備の管理は管理者のみが行える |
| 3 | 制約 | 利用停止中の会議室は新規予約の対象外とする |
