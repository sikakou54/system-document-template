# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | BR-002 |
| 要件名 | 重複予約をなくす |
| 概要 | 同一会議室・同一時間帯の二重予約(ダブルブッキング)が発生しないようにする |
| 実現したい状態 | 同一会議室・同一時間帯の二重予約が発生しない |
| 背景 | 予約が個別管理されており、同じ会議室・時間帯に予約が重なりトラブルが起きている |
| 関連要求 | RQ-009〜RQ-014 |

# 2. 業務要件

この業務要件で実現したいことを定義する。

| No | 実現したいこと |
|---|---|
| 1 | 空いている会議室を選んで予約を確定できるようにする |
| 2 | 同一会議室で時間帯が重なる予約は成立しないようにする |
| 3 | 予約が成立したかどうかを利用者が確認できるようにする |

# 3. 成功指標・期待効果

この業務要件の達成度を測る指標と期待効果を定義する。

| No | 区分 | 内容 |
|---|---|---|
| 1 | 成功指標 | 同一会議室・同一時間帯で重複する予約が成立した件数が0件である |
| 2 | 期待効果 | ダブルブッキングによる会議室の取り合い・調整の手間がなくなる |
| 3 | 期待効果 | 予約した会議室を確実に利用できる安心感が得られる |

# 4. 関連(実現する機能要件)

この業務要件を実現する機能要件(FR)・共通機能要件(CFR)を定義する。

| ID | 名称 | 実現内容 |
|---|---|---|
| FR-002 | 会議室予約 | 空き会議室を予約し、同一会議室・同一時間帯の二重予約を防止する |

# 5. 前提・制約

この業務要件の前提と制約を定義する。

| No | 区分 | 内容 |
|---|---|---|
| 1 | 前提 | 利用者が本人確認(ログイン)を済ませている |
| 2 | 前提 | 予約対象の会議室が登録されている |
| 3 | 制約 | 予約の変更・キャンセルはこの業務要件では扱わない(BR-003) |
