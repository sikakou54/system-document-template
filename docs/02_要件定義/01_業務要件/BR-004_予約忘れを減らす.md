# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | BR-004 |
| 要件名 | 予約忘れを減らす |
| 概要 | 予約開始前に予約者へリマインドが届くようにし、予約忘れによる無断キャンセルを減らす |
| 実現したい状態 | 予約開始前にリマインドが届く |
| 背景 | 予約したこと自体を忘れ、会議室が使われないまま空押さえになるケースがある |
| 関連要求 | RQ-024〜RQ-028 |

# 2. 業務要件

この業務要件で実現したいことを定義する。

| No | 実現したいこと |
|---|---|
| 1 | 予約開始の一定時間前に予約者へリマインドを届けられるようにする |
| 2 | キャンセル済みの予約はリマインドの対象外とする |

# 3. 成功指標・期待効果

この業務要件の達成度を測る指標と期待効果を定義する。

| No | 区分 | 内容 |
|---|---|---|
| 1 | 成功指標 | 対象の予約に対して開始前のリマインドが届く |
| 2 | 期待効果 | 予約忘れによる会議室の空押さえが減る |
| 3 | 期待効果 | 会議の開始遅れが減り、会議室が予定通り利用される |

# 4. 関連(実現する機能要件)

この業務要件を実現する機能要件(FR)・共通機能要件(CFR)を定義する。

| ID | 名称 | 実現内容 |
|---|---|---|
| FR-004 | 予約リマインド通知 | 予約開始前に予約者へリマインド通知を行う |

# 5. 前提・制約

この業務要件の前提と制約を定義する。

| No | 区分 | 内容 |
|---|---|---|
| 1 | 前提 | 予約者に連絡可能な宛先(メールアドレス)が登録されている |
| 2 | 制約 | リマインドは予約開始前に1回の通知とする |
| 3 | 制約 | キャンセル済みの予約は通知しない |
