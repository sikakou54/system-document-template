# 1. 概要

MeetRoom の要件定義に関わる関連・トレースを一元管理する。
要求(RQ)を実現する要件の対応、機能要件への CFR/NFR 適用、機能とデータの CRUD 相関を扱う。機能設計に関わるトレース(要件UC→画面・API、画面→API→モジュール→テーブル)は 03_機能設計/11_トレーサビリティ/ で管理する。

# 2. 一覧

| ファイル | 対象 | 目的 |
|---|---|---|
| [要求要件トレーサビリティ.md](要求要件トレーサビリティ.md) | RQ → FR/CFR/NFR | 要求が実現要件へ落ちているかを追跡する(要求→要件) |
| [機能非機能トレーサビリティ.md](機能非機能トレーサビリティ.md) | FR → CFR/NFR | 機能要件に適用される共通機能要件(CFR)・非機能要件(NFR)を追跡する |
| [機能データCRUD相関.md](機能データCRUD相関.md) | FR/CFR × データ(エンティティ) | どの機能がどのデータを作成・参照・更新・削除するかを俯瞰する |

# 3. 更新ルール

- 各表は関連文書(RQ/FR/CFR/NFR/MDL)の作成・更新と同一作業内で該当行を更新する(正本: docs/CLAUDE.md §6・§10-7)。
- 記載する ID はすべて実在するもの(各 index.md／各文書のインライン定義に存在)に限る。採番・廃止の扱いは docs/99_その他/ID採番ルール.md に従う。
