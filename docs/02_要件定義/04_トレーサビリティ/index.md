# 1. 概要

MeetRoom の要件定義層(機能要件 FR・共通機能要件 CFR・非機能要件 NFR・データモデル MDL)を横断する関連・トレースを一元管理する。
機能とデータの CRUD 相関、機能要件への CFR/NFR の適用関係などを扱う。工程をまたぐトレース(RQ→要件→設計)は最上位 04_トレーサビリティ/ で管理する。

# 2. 一覧

| ファイル | 対象 | 目的 |
|---|---|---|
| [機能データCRUD相関.md](機能データCRUD相関.md) | FR/CFR × データ(エンティティ) | どの機能がどのデータを作成・参照・更新・削除するかを俯瞰する |
| [traceability_fr_nfr.md](traceability_fr_nfr.md) | FR → CFR/NFR | 機能要件に適用される共通機能要件(CFR)・非機能要件(NFR)を追跡する |

# 3. 更新ルール

- 各表は関連文書(FR/CFR/NFR/MDL)の作成・更新と同一作業内で該当行を更新する(正本: docs/CLAUDE.md §6・§10-7)。
- 記載する ID はすべて実在するもの(各 index.md／各文書のインライン定義に存在)に限る。採番・廃止の扱いは docs/99_その他/ID採番ルール.md に従う。
