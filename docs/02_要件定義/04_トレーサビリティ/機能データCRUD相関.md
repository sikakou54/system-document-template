# 1. 概要

MeetRoom の機能要件(FR)・共通機能要件(CFR)と各データ(エンティティ)の関係を CRUD 観点で示す。

- どの機能がどのデータを作成・参照・更新・削除するかを俯瞰し、機能とデータの網羅性を確認する。
- 各機能・各データの詳細は該当 FR/CFR 文書および各データ定義書(MDL-001〜011／一覧は 02_要件定義/03_データモデル/index.md)を正本とする。
- 廃止・欠番(FR-007)は含めない。

# 2. CRUD相関マトリクス

列の各データは 02_要件定義/03_データモデル/ のエンティティ(利用者=MDL-001／会議室=MDL-002／予約=MDL-003／設備=MDL-004／会議室利用実績=MDL-005／利用量記録=MDL-006／請求=MDL-007／会議室設備=MDL-008／認証セッション=MDL-009／監査ログ=MDL-010／外部イベント受信記録=MDL-011)に対応する。

凡例: C=作成 / R=参照 / U=更新 / D=削除（論理削除を含む）。空欄は当該機能が当該データを操作しないことを示す。作成・更新は該当データの検証参照を含む。

補足:
- 利用者操作を伴わない自動処理も、その処理を要求するFRへ帰属させる。月次利用実績集計はFR-006、リマインド状態更新はFR-004、予約完了・利用量・請求・外部イベント更新はFR-008、認証失効はCFR-001として示す。
- 監査ログ(MDL-010)の作成(C)は CFR-007 監査ログ記録に基づき、重要操作(予約=FR-002／予約変更・キャンセル=FR-003／会議室・設備管理=FR-005／課金の支払い方法登録=FR-008)の実行時に、その操作を行った利用者を伴って記録する。利用者を伴わない自動処理はNFR-008の運用ログ・メトリクスで追跡する。
- CFR-003 入力検証・CFR-004 エラー表示・CFR-005 一覧検索・ページングは全機能へ横断適用される共通機能要件であり、特定データへの CRUD を持たないため本表には含めない。
- 利用者の登録(Create)・論理削除(Delete)はアプリ機能を持たず運用手順で行う（スコープ外）。
- 監査ログの参照は監査目的で管理者が行う（NFR-006）。
- 外部イベント受信記録(MDL-011)はFR-008/UC-04の請求・契約結果を冪等に反映し、重複・順序逆転・再処理を管理する。

# 3. 更新ルール

| 更新契機 | 更新内容 |
|---|---|
| FR/CFR の新規作成・変更 | 該当機能の行を追加／更新し、各データへの CRUD を各 FR/CFR の業務ルール・ユースケースに合わせて記入する |
| エンティティ(MDL)の追加・変更・廃止 | 該当データの列を追加／更新し、各 MDL の CRUD・関連機能に整合させる |
| ID廃止 | 行・列は削除せず、該当機能・データを廃止として扱い本表から除外する |

- 本表の更新は関連文書(FR/CFR/MDL)の作成・更新と同一作業内で行う(正本: docs/CLAUDE.md §6・§10-7)。
