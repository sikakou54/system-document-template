# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | FR-005 |
| 要件名 | 会議室・設備管理 |
| 概要 | 管理者が会議室と設備を登録・編集できる |
| 目的 | 会議室・設備・利用単価を最新に保ち、検索・予約・課金の前提を正確にする |
| 対象ロール | 管理者 |

# 2. 機能要件

利用者の操作とシステムの処理を定義する。

| 項目 | 内容 |
|---|---|
| 利用者の操作 | 管理者が会議室(名称・収容人数・場所・利用単価・利用状態)と設備を登録・編集する |
| システムの処理 | 管理者の操作に限り会議室・設備の情報を登録・更新する |

# 3. 入力情報

この機能が処理のために必要とする入力情報を業務レベルで定義する。

| 情報 | 要否 | 内容 |
|---|---|---|
| 会議室名 | 必須 | 会議室の名称 |
| 収容人数 | 必須 | 会議室に収容できる人数 |
| 設置場所 | 任意 | 会議室の場所 |
| 利用単価 | 必須 | 1時間あたりの利用単価(0は無料) |
| 利用状態 | 必須 | 会議室の利用可否(利用可 / 利用停止) |
| 備え付け設備 | 任意 | 会議室に紐付ける設備 |
| 設備名 | 必須 | 設備を登録する際の設備の名称 |

# 4. 出力情報

この機能が結果として提供する出力情報を業務レベルで定義する。

| 情報 | 内容 |
|---|---|
| 登録・編集結果 | 会議室・設備情報が保存されたことが分かる情報 |
| 会議室情報 | 登録・編集された会議室の内容 |

# 5. 業務ルール

この機能の判断基準となる業務上のルールを定義する。

| No | ルール |
|---|---|
| 1 | 会議室・設備の登録・編集は管理者のみが行える(RULE-COM-001) |
| 2 | 会議室名・収容人数・利用単価は必須とする |
| 3 | 収容人数は1人以上とする |
| 4 | 利用単価は0以上とし、0は無料会議室として扱う |
| 5 | 利用停止中の会議室は新規予約の対象外とする |

# 6. 受入条件

この要件を満たしていることを確認する条件を定義する。

| No | 条件 |
|---|---|
| 1 | 管理者が会議室(利用単価を含む)を登録・編集すると内容が保存される |
| 2 | 管理者が設備を登録し会議室に紐付けられる |
| 3 | 会議室名・収容人数・利用単価が未入力の場合は保存されずエラーになる |
| 4 | 利用停止にした会議室は新規予約の検索・予約対象から外れる |
| 5 | 一般利用者は会議室・設備の登録・編集ができない |

# 7. 非機能要件

本機能に適用される非機能要件(NFR)は 02_要件定義/03_非機能要件/ で定義する。適用関係は 04_トレーサビリティ/traceability_fr_nfr.md を参照する。

# 8. 前提条件

この要件が成り立つために、先に満たしておく条件を定義する。

| No | 条件 |
|---|---|
| 1 | 操作者が管理者としてログイン済みである |

# 9. 制限事項

この要件での制限や上限などを定義する。

| No | 制限 |
|---|---|
| 1 | 会議室・設備の登録・編集は管理者に限る |
| 2 | 予約データの編集はこの要件では扱わない |
