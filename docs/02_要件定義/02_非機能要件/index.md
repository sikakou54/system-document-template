# 1. 概要

MeetRoom の非機能要件(NFR)の一覧を定義する。

- 性能・可用性・セキュリティ・データ保護・監査など、機能横断または特定機能に求める品質要求を管理する。
- 各非機能要件の詳細は本ディレクトリの NFR-XXX_名称.md に記載する。
- FR との適用関係は 02_要件定義/04_トレーサビリティ/traceability_fr_nfr.md で管理する。

# 2. 非機能要件一覧

| ID | 分類 | 概要 | 対象範囲 |
|---|---|---|---|
| NFR-001 | 性能 | 会議室検索は通常利用時に利用者を待たせない時間で応答する | 会議室検索機能 |
| NFR-002 | 可用性 | 平日8:00〜20:00は安定して利用できる | 全機能 |
| NFR-003 | セキュリティ | パスワードは平文保存せずハッシュ化して保存する | 全機能 |
| NFR-004 | セキュリティ(通信) | 通信を暗号化し、システムが送信するメールのなりすましを防ぐ | 全機能(利用者との通信・システムからの送信メール) |
| NFR-005 | データ保護 | 業務データを定期的にバックアップし、障害時のデータ消失に備える | システム全体(業務データ) |
| NFR-006 | 監査 | 重要操作の記録を保持し、後から追跡できるようにする | 予約・課金・会議室管理などの重要操作 |
