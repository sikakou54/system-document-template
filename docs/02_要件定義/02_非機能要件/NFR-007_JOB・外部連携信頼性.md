# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | NFR-007 |
| 分類 | 信頼性・外部連携 |
| 対象範囲 | 予約リマインド、月次利用実績集計、利用量計上、請求結果反映 |
| 概要 | 定期処理・外部連携を再実行しても処理を欠落・重複させず、許容時間内に結果を反映する |
| 状態 | 確定 |

# 2. 品質要求・達成基準・検証方法

| 品質要求 | 達成基準 | 検証方法 |
|---|---|---|
| リマインドを適時に処理する | 通知対象時刻到来後15分以内に99%以上を送信処理し、一時障害を除く対象は30分以内に送信済みまたは失敗として結果を確定する | 定期処理試験と月次の処理時刻・状態ログで確認する |
| 利用量を欠落・重複なく計上する | 予約終了後30分以内に99%以上、4時間以内に100%を計上済みまたは再試行対象として記録する。同一予約の利用量を複数回外部計上しない | 障害注入を含む再実行試験と、予約・利用量・外部計上の月次照合で確認する |
| 月次実績を最新化する | 当月・前月の完了予約を90分以内に月次利用実績へ反映する | 定期処理の最終成功時刻と集計元・集計結果の照合で確認する |
| 請求結果を適時かつ冪等に反映する | 正当な請求結果の受信から5分以内に99%以上、1時間以内に100%を反映または再試行対象として記録する。同一イベントの再受信で請求を重複作成・状態巻戻ししない | 重複・順序逆転・一時障害を含む連携試験と月次照合で確認する |
| 一時障害から自動回復する | 外部送信は最大3回まで自動再試行し、継続失敗はデータを失わず再処理対象として保持する | 外部サービス障害を模擬し、再試行回数・保持データ・復旧後の再処理を確認する |
