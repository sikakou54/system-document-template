# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | NFR-008 |
| 分類 | 運用・監視 |
| 対象範囲 | システム全体、定期処理、外部連携、バックアップ |
| 概要 | 利用不能、処理停滞、外部連携失敗、バックアップ失敗を運用者が早期に検知できる |
| 状態 | 確定 |

# 2. 品質要求・達成基準・検証方法

| 品質要求 | 達成基準 | 検証方法 |
|---|---|---|
| 利用不能と重大エラーを検知する | 主要画面・主要APIを1分間隔で外形監視する。連続5分の利用不能、または5分窓のリクエストが20件以上ある状態で5xx率が5%以上となった場合に、条件成立から5分以内に管理者へ通知する | 監視設定確認と四半期ごとの通知試験で確認する |
| 定期処理の停止・滞留を検知する | 各JOBの最終成功時刻、処理件数、失敗件数、再試行対象件数を記録し、NFR-007の反映期限を超える停滞を5分以内に通知する | JOB停止・失敗・滞留を模擬した障害試験で確認する |
| 外部連携障害を検知する | 外部送信・Webhook検証・状態反映の成功率と失敗件数を監視し、3回の継続失敗または未反映発生を5分以内に通知する | 外部連携の障害注入試験で通知と再処理対象を確認する |
| バックアップ失敗を検知する | 予定したバックアップの未取得・失敗を予定完了時刻から15分以内に通知する | バックアップ停止を模擬した通知試験で確認する |
| 障害調査に必要な相関を確保する | API、JOB、外部イベント、監査ログを相関IDで追跡できる | 代表的な正常・失敗シナリオでログを横断検索して確認する |
