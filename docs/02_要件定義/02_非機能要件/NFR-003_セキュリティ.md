# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | NFR-003 |
| 分類 | セキュリティ |
| 対象範囲 | 全機能 |
| 概要 | パスワードは平文保存せずハッシュ化して保存する |
| 状態 | 確定 |

# 2. 品質要求・達成基準・検証方法

この非機能要件の品質要求と、その達成基準・検証方法を定義する。

| 品質要求 | 達成基準 | 検証方法 |
|---|---|---|
| パスワードは平文保存せず、十分な強度の一方向ハッシュとして保存する | bcrypt cost 12以上と利用者ごとのsaltを適用し、平文または復号可能な値を保存しない。保存済みハッシュが現行基準を下回る場合は、次回の認証成功時に現行基準で再ハッシュする。強度基準は年1回以上見直す | コードレビューでcost・salt・再ハッシュ判定を確認し、保存データ検査で平文・復号可能値がないこと、基準未満ハッシュを用いた認証試験で成功後に再ハッシュされること、年次レビュー記録があることを確認する |
