# 1. 概要

MeetRoom の全 REST API のエラーコード(ERR-XXX)を一元管理する正本。エラー名・HTTPステータス・開発者向けメッセージを §2 一覧で定義し、各 API 文書・API-COM はエラーコードで参照する(定義を再掲しない)。

- エラーコードはシステム全体を通した連番(共通・固有を区別せず ERR-001 から 1 系列で採番)。採番は §2 一覧の最大値 +1、欠番の再利用は禁止、廃止は行を残し再利用しない。
- 区分は「共通」(全 API 共通処理フローで発生。API-COM §7)/「固有」(個別 API の処理フローで発生。各 API §4/§5)。
- 開発者向けメッセージ中の可変値は {0}, {1}(0始まり)のプレースホルダで表し、パラメータ列に各プレースホルダの意味を記載する(例: {0}=予約ID)。可変値がない場合はパラメータ列はブランクとする。
- エラーレスポンスの封筒構造は API-COM_共通設計.md §4 が正本。発生条件は本ファイルに持たせず、共通は API-COM §7 共通処理フロー、固有は各 API §4/§5 個別処理フローで表現する。
- ERRの追加・変更前に関連SEQの例外系と `../11_トレーサビリティ/traceability_sequence_design.md` を確認し、発生条件・処理方針を該当API/JOB/MODへ反映する。
