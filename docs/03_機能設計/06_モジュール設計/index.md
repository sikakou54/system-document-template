# 1. 概要

サービス・ドメインロジック・内部処理のモジュール(MOD)を定義する。各モジュールの責務・インターフェース・処理詳細・データアクセスは MOD-XXX_モジュール名.md に記載する。

- 作成・更新前に `../11_トレーサビリティ/シーケンス設計トレーサビリティ.md` で関連SEQを特定し、責務・入出力・業務ルール・分岐・CRUD・他MOD連携・エラーを反映する。

# 2. 一覧

| ID | 名称 | 種別 | 概要 |
|---|---|---|---|
| MOD-001 | 認証サービス | Service | ログイン時の資格情報検証と JWT の発行、および JWT の検証。暗号処理は MOD-008 を利用する |
| MOD-002 | 会議室検索サービス | Service | 条件に合う空き会議室の抽出(空き判定)と会議室詳細の取得 |
| MOD-003 | 予約サービス | Service | 予約の登録・変更・キャンセル・完了。D1の条件付き書込みで二重予約と状態競合を防ぎ、予約確定時単価を保持する |
| MOD-004 | 会議室管理サービス | Service | 会議室(利用単価)・設備の登録編集、会議室一覧(利用停止含む)・設備一覧の取得、設備の登録 |
| MOD-005 | 利用実績サービス | Service | 完了予約の月次利用実績の集計・保存・取得(利用時間算出は廃止) |
| MOD-006 | 通知サービス | Service | リマインド対象抽出・Queueメッセージ1件のメール送信・送信状態更新(再送・DLQ・管理者アラートは Queue境界・運用) |
| MOD-007 | 課金サービス | Service | 支払い方法登録・Stripe Webhook 処理・利用量計上・利用量再送・請求状態管理 |
| MOD-008 | 認証暗号サービス | Utility | JWT の発行・検証とパスワードハッシュ照合を行う |
| MOD-009 | 監査ログサービス | Service | 重要操作(予約・課金・会議室管理)の監査ログ記録。呼び出し元の業務トランザクション内で追記する |
