# 1. 基本情報

|---|---|

| シーケンス名 | 会議室・設備管理シーケンス |
| 目的 | 管理者による会議室(利用単価・利用状態・設備紐付けを含む)と設備の登録・編集を、入力検証・存在確認・トランザクション境界とともにブロック間連携として明確にする。 |
| 対象範囲 | 開始: 管理者がSCR-005で会議室・設備管理を開く / 終了: 会議室・設備の保存完了、またはエラー結果が管理者へ表示される |

| 状態 | 確定 |

# 3. シーケンス

本シーケンスは管理者による会議室・設備の登録・編集の連携を扱い、操作種別(新規登録・編集・設備登録・設備編集)・入力の妥当性・編集対象の会議室／設備の存在・設備名の重複を確認して、保存可能なものだけを確定する。網羅する状態パターン(FR-005/UC-01)を示す。

| パターンID | 状態パターン(条件) | 本シーケンスでの表現 |
|---|---|---|
| FR-005/UC-01/SP-1 | 新規登録・入力妥当(必須充足・制約内) | 3.1 正常系(新規登録・設備紐付け) |
| FR-005/UC-01/SP-2 | 編集・入力妥当・編集対象の会議室が存在 | 3.2 代替系 3.2.1(既存会議室の編集) |
| FR-005/UC-01/SP-3 | 設備登録・設備名が重複なし | 3.2 代替系 3.2.2(設備の新規登録) |

| FR-005/UC-01/SP-5 | 収容人数が未入力 | 3.3 例外系「入力値が不正(必須項目の欠落)」 |
| FR-005/UC-01/SP-6 | 利用単価が未入力 | 3.3 例外系「入力値が不正(必須項目の欠落)」 |
| FR-005/UC-01/SP-7 | 収容人数が1人未満または利用単価が0未満(制約違反) | 3.3 例外系「入力値が不正(制約違反)」 |

## 3.1 正常系シーケンス

管理者が管理画面を開いて会議室一覧・設備選択肢を取得し、会議室を新規登録して設備を紐付ける基本の流れを示す。

```mermaid
sequenceDiagram
    actor U as 管理者
    participant UI as SCR-005 会議室・設備管理画面
    participant LIST as API-014 会議室一覧取得API
    participant EQL as API-009 設備一覧取得API
    participant API as API-007 会議室登録・編集API
    participant MOD as MOD-004 会議室管理サービス
    participant AUDIT as MOD-009 監査ログサービス
    participant DB as データベース

    U->>UI: 会議室・設備の管理を開く
    UI->>LIST: 会議室一覧の取得を要求する
    Note over LIST: API-COMの共通前処理を適用する
    LIST->>MOD: 会議室一覧の取得を依頼する
    MOD->>DB: 会議室一覧を参照する（会議室, 会議室設備, 設備）
    DB-->>MOD: 会議室一覧を返す
    MOD-->>LIST: 会議室一覧を返す
    LIST-->>UI: 会議室一覧を返す
    UI->>EQL: 設備一覧の取得を要求する
    EQL->>MOD: 設備一覧の取得を依頼する
    MOD->>DB: 設備一覧を参照する（設備）
    DB-->>MOD: 設備一覧を返す
    MOD-->>EQL: 設備一覧を返す
    EQL-->>UI: 設備一覧を返す
    UI-->>U: 管理画面を表示する
    U->>UI: 会議室情報を入力して保存する
    UI->>API: 会議室登録を要求する
    Note over API: API-COMの共通前処理を適用する
    API->>MOD: 会議室登録を依頼する
    MOD->>DB: 設備の存在を確認する（設備）
    DB-->>MOD: 設備の存在を返す
    MOD->>DB: 会議室を登録する（会議室）
    DB-->>MOD: 登録した会議室を返す
    MOD->>DB: 会議室と設備の紐付けを登録する（会議室設備）
    DB-->>MOD: 登録した紐付けを返す
    MOD->>AUDIT: 監査ログ記録を依頼する
    AUDIT->>DB: 監査ログを記録する（監査ログ）
    DB-->>AUDIT: 記録した監査ログを返す
    AUDIT-->>MOD: 記録完了を返す
    MOD-->>API: 登録した会議室情報を返す
    API-->>UI: 登録成功を返す
    UI-->>U: 登録完了を表示する
```

## 3.2 代替系シーケンス

### 3.2.1 既存会議室の編集

管理者が会議室一覧から編集対象を選び、内容を更新して設備紐付けを付け替える流れを示す。編集対象の選択と現在値表示は画面内で完結し、保存時に会議室編集を要求する。

```mermaid
sequenceDiagram
    actor U as 管理者
    participant UI as SCR-005 会議室・設備管理画面
    participant API as API-007 会議室登録・編集API
    participant MOD as MOD-004 会議室管理サービス
    participant AUDIT as MOD-009 監査ログサービス
    participant DB as データベース

    U->>UI: 会議室一覧から編集対象を選択する
    UI-->>U: 現在値を入力フォームに表示する
    U->>UI: 内容を編集して保存する
    UI->>API: 会議室編集を要求する
    Note over API: API-COMの共通前処理を適用する
    API->>MOD: 会議室編集を依頼する
    MOD->>DB: 設備の存在を確認する（設備）
    DB-->>MOD: 設備の存在を返す
    MOD->>DB: 編集対象の会議室を取得する（会議室）
    DB-->>MOD: 会議室を返す
    MOD->>DB: 会議室を更新する（会議室）
    DB-->>MOD: 更新した会議室を返す
    MOD->>DB: 会議室と設備の紐付けを付け替える（会議室設備）
    DB-->>MOD: 付け替えた紐付けを返す
    MOD->>AUDIT: 監査ログ記録を依頼する
    AUDIT->>DB: 監査ログを記録する（監査ログ）
    DB-->>AUDIT: 記録した監査ログを返す
    AUDIT-->>MOD: 記録完了を返す
    MOD-->>API: 更新した会議室情報を返す
    API-->>UI: 編集成功を返す
    UI-->>U: 保存完了を表示する
```

### 3.2.2 設備の新規登録（FR-005/UC-01/ALT-1）

会議室に紐付ける設備が未登録の場合に、管理者が設備を新規登録し、設備選択肢を再取得して更新する流れを示す。

```mermaid
sequenceDiagram
    actor U as 管理者
    participant UI as SCR-005 会議室・設備管理画面
    participant EQREG as API-013 設備登録API
    participant EQL as API-009 設備一覧取得API
    participant MOD as MOD-004 会議室管理サービス
    participant AUDIT as MOD-009 監査ログサービス
    participant DB as データベース

    U->>UI: 設備名を入力して設備を登録する
    UI->>EQREG: 設備登録を要求する
    Note over EQREG: API-COMの共通前処理を適用する
    EQREG->>MOD: 設備登録を依頼する
    MOD->>DB: 同名の設備の有無を確認する（設備）
    DB-->>MOD: 同名設備の有無を返す
    MOD->>DB: 設備を登録する（設備）
    DB-->>MOD: 登録した設備を返す
    MOD->>AUDIT: 監査ログ記録を依頼する
    AUDIT->>DB: 監査ログを記録する（監査ログ）
    DB-->>AUDIT: 記録した監査ログを返す
    AUDIT-->>MOD: 記録完了を返す
    MOD-->>EQREG: 登録した設備情報を返す
    EQREG-->>UI: 登録成功を返す
    UI->>EQL: 設備一覧の再取得を要求する
    EQL->>MOD: 設備一覧の取得を依頼する
    MOD->>DB: 設備一覧を参照する（設備）
    DB-->>MOD: 設備一覧を返す
    MOD-->>EQL: 設備一覧を返す
    EQL-->>UI: 設備一覧を返す
    UI-->>U: 設備登録完了を表示し設備選択肢を更新する
```

### 3.2.3 設備名の編集（FR-005/UC-01/ALT-2）

管理者が設備一覧から編集対象の設備を選び、設備名を変更して保存する流れを示す。編集対象の選択と現在値表示は画面内で完結し、保存時に設備更新を要求する。

```mermaid
sequenceDiagram
    actor U as 管理者
    participant UI as SCR-005 会議室・設備管理画面
    participant EQEDIT as API-015 設備更新API
    participant MOD as MOD-004 会議室管理サービス
    participant AUDIT as MOD-009 監査ログサービス
    participant DB as データベース

    U->>UI: 設備一覧から編集対象の設備を選択する
    UI-->>U: 現在の設備名を入力フォームに表示する
    U->>UI: 設備名を変更して保存する
    UI->>EQEDIT: 設備更新を要求する

    MOD->>DB: 編集対象の設備を取得する（設備）
    DB-->>MOD: 設備を返す
    MOD->>DB: 同名の設備の有無を確認する（設備）
    DB-->>MOD: 同名設備の有無を返す
    MOD->>DB: 設備名を更新する（設備）
    DB-->>MOD: 更新した設備を返す
    MOD->>AUDIT: 監査ログ記録を依頼する
    AUDIT->>DB: 監査ログを記録する（監査ログ）
    DB-->>AUDIT: 記録した監査ログを返す
    AUDIT-->>MOD: 記録完了を返す
    MOD-->>EQEDIT: 更新した設備情報を返す
    EQEDIT-->>UI: 更新成功を返す
    UI-->>U: 設備名の更新完了を表示する
```

## 3.3 例外系シーケンス

入力値不正(必須欠落・制約違反)、編集対象の会議室が存在しない、設備名が重複する(設備登録・設備編集)、編集対象の設備が存在しない各分岐を示す。

```mermaid
sequenceDiagram
    actor U as 管理者
    participant UI as SCR-005 会議室・設備管理画面
    participant API as API-007 会議室登録・編集API
    participant EQREG as API-013 設備登録API
    participant EQEDIT as API-015 設備更新API
    participant MOD as MOD-004 会議室管理サービス
    participant DB as データベース

    alt 入力値が不正（必須項目の欠落・制約違反）
        U->>UI: 会議室情報を入力して保存する
        UI->>API: 会議室登録を要求する
        API-->>UI: 入力エラーを返す
        UI-->>U: 入力エラーを表示する
    else 編集対象の会議室が存在しない
        U->>UI: 会議室を編集して保存する
        UI->>API: 会議室編集を要求する
        API->>MOD: 会議室編集を依頼する
        MOD->>DB: 編集対象の会議室を取得する（会議室）
        DB-->>MOD: 該当なしを返す
        MOD-->>API: 会議室不在のエラーを返す
        API-->>UI: 会議室不在のエラーを返す
        UI-->>U: 会議室が見つからないエラーを表示し一覧再取得を促す
    else 設備名が重複
        U->>UI: 設備名を入力して設備を登録する
        UI->>EQREG: 設備登録を要求する
        EQREG->>MOD: 設備登録を依頼する
        MOD->>DB: 同名の設備の有無を確認する（設備）
        DB-->>MOD: 同名設備ありを返す
        MOD-->>EQREG: 設備名重複のエラーを返す
        EQREG-->>UI: 設備名重複のエラーを返す
        UI-->>U: 設備名重複エラーを表示する
    else 編集対象の設備が存在しない
        U->>UI: 設備名を変更して保存する
        UI->>EQEDIT: 設備更新を要求する
        EQEDIT->>MOD: 設備更新を依頼する
        MOD->>DB: 編集対象の設備を取得する（設備）
        DB-->>MOD: 該当なしを返す
        MOD-->>EQEDIT: 設備不在のエラーを返す
        EQEDIT-->>UI: 設備不在のエラーを返す
        UI-->>U: 設備が見つからないエラーを表示し一覧再取得を促す
    else 設備名が重複(設備編集)
        U->>UI: 設備名を変更して保存する
        UI->>EQEDIT: 設備更新を要求する
        EQEDIT->>MOD: 設備更新を依頼する
        MOD->>DB: 編集対象の設備を取得する（設備）
        DB-->>MOD: 設備を返す
        MOD->>DB: 同名の設備の有無を確認する（設備）
        DB-->>MOD: 同名設備ありを返す
        MOD-->>EQEDIT: 設備名重複のエラーを返す
        EQEDIT-->>UI: 設備名重複のエラーを返す
        UI-->>U: 設備名重複エラーを表示する
    end
```

## 4.4 補足事項

| 観点 | 内容 |
|---|---|
| 同期/非同期 | 初期表示の一覧取得、会議室の登録・編集、設備の登録はいずれも同期処理で、結果を同一操作内で返す。 |
| 冪等性・再試行 | 会議室登録(POST)・設備登録(POST)は冪等でなく、再送で会議室が重複登録される、または同名設備として重複エラーになる。会議室編集(PUT)・設備編集(PUT)は冪等で、同一内容の再送でも結果は変わらない。 |
| 排他制御 | 明示的な行ロックは行わず、データベースが書き込みを直列化する。会議室更新と設備紐付けの付け替えは同一トランザクション内で直列に実行する。 |
| 外部連携 | なし。 |
