# 3. シーケンス

本シーケンスは予約キャンセルの連携を扱い、対象が予約者本人・予約済・未開始であることを確認し、キャンセル可能な予約だけをキャンセル状態へ更新して時間帯を開放する。網羅する状態パターン(FR-003/UC-02)を示す。キャンセル前提の不成立（他者・予約なし・キャンセル済み・開始済み（完了を含む）。SP-3〜SP-5）は責務が共通のため 3.3 例外系の単一の前提判定分岐に束ねて表現する。なお認証失敗はCFRの共通前処理由来であり、対応するUCの状態パターンを持たない。

| パターンID | 状態パターン(条件) | 本シーケンスでの表現 |
|---|---|---|
| FR-003/UC-02/SP-1 | 本人・予約済・未開始 | 3.1 正常系 |
| FR-003/UC-02/SP-2 | キャンセルを取りやめる | 3.2 代替系 |
| FR-003/UC-02/SP-3 | 対象が他者の予約 | 3.3 例外系「予約なし・他者・キャンセル不可状態・開始済み」 |
| FR-003/UC-02/SP-4 | 対象がキャンセル済み | 3.3 例外系「予約なし・他者・キャンセル不可状態・開始済み」 |
| FR-003/UC-02/SP-5 | 対象が開始済み（完了を含む） | 3.3 例外系「予約なし・他者・キャンセル不可状態・開始済み」 |

## 3.1 正常系シーケンス

予約者本人が、予約済かつ未開始の自分の予約をキャンセルする基本の流れを示す。

```mermaid
sequenceDiagram
    actor U as 一般ユーザー／管理者
    participant UI as SCR-004 予約一覧・変更・キャンセル画面
    participant API as API-005 予約キャンセルAPI
    participant RES as MOD-003 予約サービス
    participant AUDIT as MOD-009 監査ログサービス
    participant DB as データベース

    U->>UI: キャンセルする予約を選択する
    UI-->>U: キャンセル確認を表示する
    U->>UI: キャンセルを確定する
    UI->>API: 予約キャンセルを要求する
    Note over API: API-COMの共通前処理を適用する
    API->>RES: 予約キャンセルを依頼する
    RES->>DB: 自分の予約を取得する（予約, 会議室）
    DB-->>RES: 対象の予約を返す
    Note over RES: キャンセル前提（本人・予約済・未開始）を満たす
    RES->>DB: 予約をキャンセル状態に更新する（予約）
    DB-->>RES: 更新した予約を返す
    RES->>AUDIT: 監査ログ記録を依頼する
    AUDIT->>DB: 監査ログを記録する（監査ログ）
    DB-->>AUDIT: 記録した監査ログを返す
    AUDIT-->>RES: 記録完了を返す
    RES-->>API: キャンセル結果を返す
    API-->>UI: キャンセル成功を返す
    UI-->>U: キャンセル完了を表示し予約一覧を再取得する
```

## 3.2 代替系シーケンス

確認ダイアログでキャンセルを取りやめた場合（FR-003/UC-02/ALT-1）は、APIを呼ばず予約を残したまま一覧に戻る。

```mermaid
sequenceDiagram
    actor U as 一般ユーザー／管理者
    participant UI as SCR-004 予約一覧・変更・キャンセル画面

    U->>UI: キャンセルする予約を選択する
    UI-->>U: キャンセル確認を表示する
    U->>UI: キャンセルを取りやめる
    Note over UI: APIを呼ばず予約を残す
    UI-->>U: 確認を閉じ予約一覧に戻る
```

## 3.3 例外系シーケンス

認証エラー、および対象が予約者本人でない・存在しない・キャンセル不可状態（キャンセル済み・開始済み（完了を含む））の前提エラーを示す。

```mermaid
sequenceDiagram
    actor U as 一般ユーザー／管理者
    participant UI as SCR-004 予約一覧・変更・キャンセル画面
    participant API as API-005 予約キャンセルAPI
    participant RES as MOD-003 予約サービス
    participant DB as データベース

    U->>UI: キャンセルを確定する
    UI->>API: 予約キャンセルを要求する
    alt 認証失敗・トークン失効
        API-->>UI: 認証エラーを返す
        UI-->>U: ログイン画面へ遷移する
    else 予約なし・他者・キャンセル不可状態・開始済み
        Note over API: API-COMの共通前処理を適用する
        API->>RES: 予約キャンセルを依頼する
        RES->>DB: 自分の予約を取得する（予約, 会議室）
        DB-->>RES: 対象の予約または該当なしを返す
        Note over RES: キャンセル前提を満たさない
        RES-->>API: キャンセル不可のエラーを返す
        API-->>UI: キャンセル不可のエラーを返す
        UI-->>U: 予約状態変化のエラーを表示し予約一覧を再取得する
    end
```

# 4. 連携定義

## 4.1 条件分岐

| 条件ID | 判定箇所 | 条件 | 成立時 | 不成立時 | 根拠 |
|---|---|---|---|---|---|
| COND-01 | SCR-004 | キャンセル確認で確定する | 予約キャンセルを要求 | 予約を残し一覧へ戻る | FR-003/UC-02/ALT-1 / FR-003/UC-02/SP-2 |
| COND-02 | API-005 / MOD-003 | 対象予約が存在し、予約者本人の予約である | キャンセルを継続 | キャンセル不可のエラー | FR-003 業務ルール1, FR-003/UC-02/EXC-3 / FR-003/UC-02/SP-3 |
| COND-03 | API-005 / MOD-003 | 対象予約がキャンセル済みでない | キャンセルを継続 | キャンセル不可のエラー | FR-003 業務ルール4, FR-003/UC-02/EXC-2 / FR-003/UC-02/SP-4 |
| COND-04 | API-005 / MOD-003 | 対象予約が未開始（現在日時 ＜ 利用開始日時）である（完了・開始済みを除く） | 予約をキャンセル状態に更新 | キャンセル不可のエラー | FR-003 業務ルール3, FR-003/UC-02/EXC-1 / FR-003/UC-02/SP-5 |

## 4.4 補足事項

| 観点 | 内容 |
|---|---|
| 同期/非同期 | 予約キャンセルは同期処理。正常・エラー結果を同一操作内で返す。 |
| 冪等性・再試行 | API-005は冪等ではない。ただしキャンセル済みの予約の再送は前提判定でキャンセル不可のエラーとなり、二重取消は発生しない。 |
| 排他制御 | MOD-003が対象予約の取得から更新までを単一トランザクションで直列化する。会議室の時間帯開放は予約ステータスをキャンセルに更新することで実現し、時間帯を占有する状態を解くだけのため会議室行ロックは伴わない（FR-003 業務ルール7）。 |
| 外部連携 | なし。キャンセル対象は未開始の予約に限られ、利用量計上（完了予約を対象とする課金処理）の対象外のため、課金サービス（MOD-007）・利用量記録（MDL-006）は本シーケンスに関与しない。 |
