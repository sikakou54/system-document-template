# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-031 |
| クエリ名 | 利用量記録存在確認 |
| 概要 | 指定予約の利用量記録の存在件数を返す(二重計上防止用)。 |
| 用途 | 利用量計上の冪等判定(同一予約の再計上防止)。 |
| 使用元 | MOD-007 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-007 | T_USAGE_RECORDS | 存在確認対象(予約ごとの利用量記録) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :reservation_id | 予約ID | INTEGER | Yes | 確認対象の予約 ID |

# 4. クエリ

予約ID(RESERVATION_ID)一致かつ有効(DELETED_AT IS NULL)の利用量記録の件数を数える。呼び出し側で件数=0 なら未計上と判定する。

```sql
SELECT
  COUNT(*) AS RECORD_COUNT
FROM T_USAGE_RECORDS ur
WHERE ur.RESERVATION_ID = :reservation_id
  AND ur.DELETED_AT IS NULL;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| RECORD_COUNT | 記録件数 | INTEGER | 指定予約の利用量記録の件数(0 なら未計上) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | TBL-007/IDX-1(UX_USAGE_RECORDS_RESERVATION: RESERVATION_ID) |
| 想定件数 | 1件(件数を1行で返す) |
| 注意点 | 予約ごと1件に限定される(UX_USAGE_RECORDS_RESERVATION)。件数 ＞ 0 のときは計上済みとして冪等に正常復帰する |
