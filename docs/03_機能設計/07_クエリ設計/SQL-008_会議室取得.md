# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-008 |
| クエリ名 | 会議室取得 |
| 概要 | 会議室ID一致かつ論理削除されていない有効な会議室を1件取得する。該当が無い場合は 0件。 |
| 用途 | 会議室詳細取得(RoomSearchService)、会議室編集前の存在確認・現在値取得(RoomManagementService)。 |
| 使用元 | MOD-002 / MOD-004 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-002 | M_ROOMS | 抽出対象(IDによる会議室取得) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :id | 会議室ID | INTEGER | Yes | 取得対象会議室の ID(主キー) |

# 4. クエリ

会議室ID(ID)一致かつ有効(DELETED_AT IS NULL)の会議室を1件取得する。利用停止(DEF-001/SET-004)の会議室も取得対象に含む(利用可否の判定は呼び出し側)。

```sql
SELECT
  r.ID          AS ID,
  r.NAME        AS NAME,
  r.CAPACITY    AS CAPACITY,
  r.LOCATION    AS LOCATION,
  r.HOURLY_RATE AS HOURLY_RATE,
  r.STATUS      AS STATUS,
  r.NOTE        AS NOTE
FROM M_ROOMS r
WHERE r.ID = :id
  AND r.DELETED_AT IS NULL
LIMIT 1;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 会議室ID | INTEGER | 会議室の ID |
| NAME | 会議室名 | TEXT | 会議室名 |
| CAPACITY | 収容人数 | INTEGER | 収容人数 |
| LOCATION | 設置場所 | TEXT | 設置場所 |
| HOURLY_RATE | 利用単価 | INTEGER | 1時間あたり利用単価(円)。0=無料 |
| STATUS | 会議室ステータス | INTEGER | DEF-001/CODE-003 |
| NOTE | 備考 | TEXT | 備考(NULL可) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | 主キー(ID)による一意検索 |
| 想定件数 | 最大1件 |
| 注意点 | 利用停止(DEF-001/SET-004)の会議室も返す。予約可否・利用停止の判定は呼び出しモジュール側で行う |
