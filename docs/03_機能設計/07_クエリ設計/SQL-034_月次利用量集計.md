# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-034 |
| クエリ名 | 月次利用量集計 |
| 概要 | 利用者の対象月の利用時間(分)と金額(請求見込み)を、利用量記録から集計する。 |
| 用途 | 当月利用量・請求見込みの取得(BillingService)。 |
| 使用元 | MOD-007 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-007 | T_USAGE_RECORDS | 集計対象(利用時間・金額) |
| TBL-003 | T_RESERVATIONS | 対象月の判定(予約の利用開始日時) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :user_id | ユーザーID | INTEGER | Yes | 集計対象の利用者ID |
| :target_month | 対象月 | TEXT | Yes | 'YYYY-MM' 形式(Asia/Tokyo 基準の年月)。未指定時は呼び出し側で当月を設定 |

# 4. クエリ

利用者(USER_ID)の有効な利用量記録を、RESERVATION_ID で T_RESERVATIONS に結合し、対象月(予約の START_AT を Asia/Tokyo(+9時間)へ変換した年月)で絞り込んで利用時間・金額を集計する。

```sql
SELECT
  SUM(ur.USAGE_MINUTES) AS TOTAL_MINUTES,
  SUM(ur.AMOUNT)        AS TOTAL_AMOUNT
FROM T_USAGE_RECORDS ur
INNER JOIN T_RESERVATIONS res
  ON res.ID = ur.RESERVATION_ID
  AND res.DELETED_AT IS NULL
WHERE ur.USER_ID = :user_id
  AND ur.DELETED_AT IS NULL
  AND strftime('%Y-%m', datetime(res.START_AT, '+9 hours')) = :target_month;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| TOTAL_MINUTES | 利用時間(分) | INTEGER | 対象月の利用時間(分)の合計。記録が無い場合は NULL(呼び出し側で 0 として扱う) |
| TOTAL_AMOUNT | 請求見込み金額 | INTEGER | 対象月の金額(AMOUNT)の合計(円)。記録が無い場合は NULL(呼び出し側で 0 として扱う) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | T_USAGE_RECORDS の USER_ID による絞り込み。対象月判定は結合先 T_RESERVATIONS の START_AT |
| 想定件数 | 1件(合計値を1行で返す) |
| 注意点 | 対象月は予約の START_AT を Asia/Tokyo(+9時間)へ変換した年月で判定する(SQL-002 と整合。利用は予約の利用月に帰属)。記録が無い月は集計値が NULL となるため呼び出し側で 0 に補正する |
