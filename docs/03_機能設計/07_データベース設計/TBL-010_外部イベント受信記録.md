# 1. 基本情報

| 項目 | 内容 |
|---|---|
| テーブルID | TBL-010 |
| 物理名 | T_INBOUND_EVENTS |
| 論理名 | 外部イベント受信記録トランザクション |
| 概要 | 外部サービス(Stripe 等)から受信した契約・請求イベントを、状態反映より先に一意記録する冪等受信記録(Stripe Webhook Inbox)。重複・順序逆転・一時失敗に対して安全に再処理するための受信台帳(MDL-011)。 |

# 4. 制約

| 制約 | 内容 |
|---|---|
| 冪等一意 | (EXTERNAL_SERVICE, EXTERNAL_EVENT_ID) を一意とし、同一イベントの再受信で重複レコードを作成しない(UX_INBOUND_EVENTS_SERVICE_EVENT。MDL-011 準拠) |
| 状態区分 | STATUS は CHECK(STATUS IN (1,2,3,4)) で受信済／処理済／失敗／対象外に限定(共通コード定義/CODE-008)。処理済から失敗・受信済へ戻さない |
| 作成後不変 | 外部サービス・外部イベントID・イベント種別・発生日時・再処理情報は作成後に更新しない(更新は処理状態・処理完了日時・処理試行回数・最終エラー概要のみ) |
