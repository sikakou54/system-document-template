# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-026 |
| クエリ名 | 予約更新 |
| 概要 | 予約1件のタイトル・利用開始日時・利用終了日時を更新し、更新後の予約を返す。 |
| 用途 | 予約の変更(ReservationService)。 |
| 使用元 | MOD-003 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-003 | T_RESERVATIONS | 更新対象(予約の UPDATE) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :id | 予約ID | INTEGER | Yes | 更新対象の予約 ID |
| :title | 予約タイトル | TEXT | Yes | 変更後の予約タイトル |
| :start_at | 利用開始日時 | TEXT | Yes | ISO8601(UTC)。:start_at ＜ :end_at |
| :end_at | 利用終了日時 | TEXT | Yes | ISO8601(UTC) |
| :updated_at | 更新日時 | TEXT | Yes | ISO8601(UTC)。アプリで現在時刻を設定(共通カラム.md) |
| :user_id | 予約者ユーザーID | INTEGER | Yes | 認証済み利用者本人であることを更新条件に含める |
| :now | 判定基準時刻 | TEXT | Yes | 更新前予約・変更後開始日時が未開始であることを判定 |
| :expected_updated_at | 期待更新日時 | TEXT | Yes | 一覧・詳細取得時のUPDATED_AT。変更対変更の楽観ロックに使用 |

# 4. クエリ

本人の予約済・未開始予約であり、変更後日時が未開始かつ他予約と重複しない場合だけ更新する。事前判定と更新の間に状態が変化した場合は0件となる。

```sql
UPDATE T_RESERVATIONS
SET TITLE      = :title,
    START_AT   = :start_at,
    END_AT     = :end_at,
    UPDATED_AT = :updated_at
WHERE ID = :id
  AND USER_ID = :user_id
  AND STATUS = 1
  AND START_AT > :now
  AND UPDATED_AT = :expected_updated_at
  AND DELETED_AT IS NULL
  AND :start_at >= :now
  AND :start_at < :end_at
  AND NOT EXISTS (
    SELECT 1
    FROM T_RESERVATIONS x
    WHERE x.ROOM_ID = T_RESERVATIONS.ROOM_ID
      AND x.ID <> T_RESERVATIONS.ID
      AND x.STATUS = 1
      AND x.DELETED_AT IS NULL
      AND x.START_AT < :end_at
      AND x.END_AT > :start_at
  )
RETURNING ID, ROOM_ID, USER_ID, TITLE, START_AT, END_AT, UNIT_PRICE, STATUS, REMIND_STATUS, UPDATED_AT;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 予約ID | INTEGER | 予約の ID |
| ROOM_ID | 会議室ID | INTEGER | 予約対象の会議室ID |
| USER_ID | ユーザーID | INTEGER | 予約者の ID |
| TITLE | 予約タイトル | TEXT | 更新後の予約タイトル |
| START_AT | 利用開始日時 | TEXT | 更新後の利用開始日時(ISO8601/UTC) |
| END_AT | 利用終了日時 | TEXT | 更新後の利用終了日時(ISO8601/UTC) |
| STATUS | 予約ステータス | INTEGER | 共通コード定義/CODE-004 |
| REMIND_STATUS | リマインド通知ステータス | INTEGER | 共通コード定義/CODE-005 |
| UNIT_PRICE | 予約確定時単価 | INTEGER | 予約時の単価スナップショット |
| UPDATED_AT | 更新日時 | TEXT | 更新後のRFC3339 UTC。次回更新の期待値 |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | 主キー(ID)による一意更新 |
| 想定件数 | 最大1件 |
| 注意点 | 事前判定はエラー分類に用いるが、最終保証は本SQLの条件とする。UPDATED_AT不一致を含む0件時はMOD-003が最新状態を再取得し、不在・他者・開始済み・重複・同時更新へ分類する |
