# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-026 |
| クエリ名 | 予約更新 |
| 概要 | 予約1件のタイトル・利用開始日時・利用終了日時を更新し、更新後の予約を返す。 |
| 用途 | 予約の変更(ReservationService)。 |
| 使用元 | MOD-003 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-003 | T_RESERVATIONS | 更新対象(予約の UPDATE) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :id | 予約ID | INTEGER | Yes | 更新対象の予約 ID |
| :title | 予約タイトル | TEXT | Yes | 変更後の予約タイトル |
| :start_at | 利用開始日時 | TEXT | Yes | ISO8601(UTC)。:start_at ＜ :end_at |
| :end_at | 利用終了日時 | TEXT | Yes | ISO8601(UTC) |
| :updated_at | 更新日時 | TEXT | Yes | ISO8601(UTC)。アプリで現在時刻を設定(共通カラム.md) |

# 4. クエリ

有効(DELETED_AT IS NULL)な予約のタイトル・利用日時を更新する。更新後の予約を返す。

```sql
UPDATE T_RESERVATIONS
SET TITLE      = :title,
    START_AT   = :start_at,
    END_AT     = :end_at,
    UPDATED_AT = :updated_at
WHERE ID = :id
  AND DELETED_AT IS NULL
RETURNING ID, ROOM_ID, USER_ID, TITLE, START_AT, END_AT, STATUS, REMIND_STATUS;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 予約ID | INTEGER | 予約の ID |
| ROOM_ID | 会議室ID | INTEGER | 予約対象の会議室ID |
| USER_ID | ユーザーID | INTEGER | 予約者の ID |
| TITLE | 予約タイトル | TEXT | 更新後の予約タイトル |
| START_AT | 利用開始日時 | TEXT | 更新後の利用開始日時(ISO8601/UTC) |
| END_AT | 利用終了日時 | TEXT | 更新後の利用終了日時(ISO8601/UTC) |
| STATUS | 予約ステータス | INTEGER | DEF-001/CODE-004 |
| REMIND_STATUS | リマインド通知ステータス | INTEGER | DEF-001/CODE-005 |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | 主キー(ID)による一意更新 |
| 想定件数 | 最大1件 |
| 注意点 | 予約者本人・変更可能状態(予約済・未開始)の判定と重複予約判定(SQL-022)・会議室行ロック(SQL-009)は呼び出しモジュール(MOD-003)で事前に行う。UPDATED_AT はアプリで現在時刻を設定する |
