# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-043 |
| クエリ名 | 外部イベント受信記録処理状態更新 |
| 概要 | 受信済・失敗の受信記録1件だけを処理済／失敗／対象外へ条件付きで更新する。 |
| 用途 | 外部イベントの業務反映後の状態確定・失敗記録・再処理(BillingService)。 |
| 使用元 | MOD-007 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-010 | T_INBOUND_EVENTS | 更新対象(処理状態・処理完了日時・試行回数・最終エラーの UPDATE) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :id | 外部イベント受信記録ID | INTEGER | Yes | 更新対象の受信記録 ID |
| :status | 処理状態 | INTEGER | Yes | 共通コード定義/CODE-008(処理済=2 / 失敗=3 / 対象外=4) |
| :processed_at | 処理完了日時 | TEXT | No | 処理済・対象外で確定した日時(RFC3339 UTC)。失敗時は NULL |
| :last_error | 最終エラー概要 | TEXT | No | 失敗時の理由。成功・対象外時は NULL。機微情報は含めない |
| :updated_at | 更新日時 | TEXT | Yes | RFC3339 UTC。アプリで現在時刻を設定(共通カラム.md) |

# 4. クエリ

受信済または失敗の受信記録だけを対象に、処理状態・処理完了日時・最終エラーを更新し、試行回数を加算する。

- WHERE で STATUS を受信済(1)・失敗(3)に限定し、処理済(2)・対象外(4)の確定済みレコードは更新しない(共通コード定義/CODE-008。処理済から戻さない)。
- 失敗レコードは受信済と同様に再処理対象となり、再処理のたびに ATTEMPT_COUNT を加算する。
- 更新件数(changes())が0の場合は対象外(既に確定済み・不存在)として後続処理を行わない。

```sql
UPDATE T_INBOUND_EVENTS
SET STATUS        = :status,                 -- 共通コード定義/CODE-008(2/3/4)
    PROCESSED_AT  = :processed_at,
    LAST_ERROR    = :last_error,
    ATTEMPT_COUNT = ATTEMPT_COUNT + 1,
    UPDATED_AT    = :updated_at
WHERE ID = :id
  AND STATUS IN (1, 3)
RETURNING ID, STATUS, ATTEMPT_COUNT, PROCESSED_AT;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 外部イベント受信記録ID | INTEGER | 受信記録の ID |
| STATUS | 処理状態 | INTEGER | 更新後の処理状態(共通コード定義/CODE-008) |
| ATTEMPT_COUNT | 処理試行回数 | INTEGER | 加算後の試行回数 |
| PROCESSED_AT | 処理完了日時 | TEXT | 処理済・対象外で確定した日時(失敗時 NULL) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | 主キー(ID)による一意更新。再処理対象抽出は TBL-010/IDX-2(IDX_INBOUND_EVENTS_STATUS)を別クエリで使用 |
| 想定件数 | 最大1件 |
| 注意点 | SELECT..FOR UPDATE は用いず、WHERE の STATUS 条件と changes() で確定済みレコードの二重更新を防ぐ(Cloudflare D1(SQLite)で実行可能)。処理済(2)・対象外(4)へは確定として遷移し、失敗(3)のみ再度この更新で再処理できる。PROCESSED_AT は処理済・対象外で設定し失敗時は NULL、UPDATED_AT はアプリで現在時刻を設定する |
