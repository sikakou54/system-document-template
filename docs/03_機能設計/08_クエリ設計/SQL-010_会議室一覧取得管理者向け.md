# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-010 |
| クエリ名 | 会議室一覧取得(管理者向け・設備名付き) |
| 概要 | 利用停止(共通コード定義/SET-004)を含む有効な全会議室を、備える設備名付きで会議室名昇順に取得する。 |
| 用途 | 管理者の会議室一覧取得(会議室編集対象の選択)。 |
| 使用元 | MOD-004 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-002 | M_ROOMS | 抽出対象(利用停止含む全会議室) |
| TBL-005 | M_ROOM_EQUIPMENTS | 会議室と設備の紐付け |
| TBL-004 | M_EQUIPMENTS | 設備名の取得 |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| なし | - | - | - | 引数なし(全会議室が対象) |

# 4. クエリ

有効(DELETED_AT IS NULL)な全会議室を、会議室が備える設備名(M_ROOM_EQUIPMENTS を EQUIPMENT_ID で M_EQUIPMENTS に結合)を連結して取得する。

- 利用停止(共通コード定義/SET-004)も含む。
- 会議室名昇順で並べる。

```sql
SELECT
  r.ID                 AS ROOM_ID,
  r.NAME               AS ROOM_NAME,
  r.CAPACITY           AS CAPACITY,
  r.LOCATION           AS LOCATION,
  r.HOURLY_RATE        AS HOURLY_RATE,
  r.STATUS             AS STATUS,
  r.NOTE               AS NOTE,
  GROUP_CONCAT(e.NAME) AS EQUIPMENT_NAMES
FROM M_ROOMS r
LEFT JOIN M_ROOM_EQUIPMENTS re
  ON re.ROOM_ID = r.ID
  AND re.DELETED_AT IS NULL
LEFT JOIN M_EQUIPMENTS e
  ON e.ID = re.EQUIPMENT_ID
  AND e.DELETED_AT IS NULL
WHERE r.DELETED_AT IS NULL
GROUP BY r.ID
ORDER BY r.NAME ASC, r.ID ASC;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ROOM_ID | 会議室ID | INTEGER | 会議室の ID |
| ROOM_NAME | 会議室名 | TEXT | 会議室名 |
| CAPACITY | 収容人数 | INTEGER | 収容人数 |
| LOCATION | 設置場所 | TEXT | 設置場所 |
| HOURLY_RATE | 利用単価 | INTEGER | 1時間あたり利用単価(円)。0=無料 |
| STATUS | 会議室ステータス | INTEGER | 共通コード定義/CODE-003 |
| NOTE | 備考 | TEXT | 備考(NULL可) |
| EQUIPMENT_NAMES | 設備名一覧 | TEXT | 会議室が備える設備名のカンマ連結(GROUP_CONCAT)。設備が無い会議室は NULL |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | 全件走査(会議室は最大数百件規模)。設備結合は TBL-005/IDX-1(UX_ROOM_EQUIPMENTS_ROOM_EQUIP)が寄与 |
| 想定件数 | 会議室の全件(利用停止含む) |
| 注意点 | 利用停止(共通コード定義/SET-004)も含む点が SQL-001(検索・利用可のみ)と異なる。設備名は GROUP_CONCAT でカンマ連結する。ページングは行わず全件返す |
