# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-009 |
| クエリ名 | 利用可能会議室取得 |
| 概要 | 指定会議室が有効かつ利用可能かを参照し、予約時単価を取得する。D1(SQLite)で非対応の `SELECT ... FOR UPDATE` は使用しない。 |
| 用途 | 予約登録・変更前の会議室存在、利用状態、予約時単価の取得。最終的な重複防止はSQL-025/026の条件付き書込みで行う。 |
| 使用元 | MOD-003 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-002 | M_ROOMS | 利用可能な会議室と予約時単価の取得 |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :id | 会議室ID | INTEGER | Yes | 対象会議室ID |

# 4. クエリ

```sql
SELECT
  r.ID,
  r.NAME,
  r.HOURLY_RATE,
  r.STATUS
FROM M_ROOMS r
WHERE r.ID = :id
  AND r.STATUS = 1
  AND r.DELETED_AT IS NULL
LIMIT 1;
```

# 5. 出力（列）

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 会議室ID | INTEGER | 会議室ID |
| NAME | 会議室名 | TEXT | 会議室名 |
| HOURLY_RATE | 予約時単価 | INTEGER | 予約確定時にTBL-003へ保存する1時間単価 |
| STATUS | 会議室ステータス | INTEGER | 共通コード定義/CODE-003 |

# 6. 補足（競合制御）

- 本クエリは事前表示・エラー分類のための参照であり、排他保証を担わない。
- 登録はSQL-025、変更はSQL-026が重複なし・期待状態を同一文で再確認する。
- 条件付き書込みの結果が0件の場合、MOD-003は最新状態を再取得して不在・利用不可・重複・同時更新を判別する。
