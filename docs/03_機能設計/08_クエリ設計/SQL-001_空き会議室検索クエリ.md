# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-001 |
| クエリ名 | 空き会議室検索クエリ |
| 概要 | 指定した時間帯に重複予約が無い会議室を、収容人数・設備条件で絞り込んで抽出する。利用停止中の会議室は除外する。 |
| 用途 | 会議室検索(空き会議室の抽出)。空き判定ロジックの正本。 |
| 使用元 | MOD-002 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-002 | M_ROOMS | 抽出対象(利用可・収容人数条件) |
| TBL-003 | T_RESERVATIONS | 重複予約の判定(予約済のみ対象) |
| TBL-005 | M_ROOM_EQUIPMENTS | 設備条件の判定(会議室が備える設備) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :start | 利用開始日時 | TEXT | Yes | ISO8601(UTC)。この時刻から利用する |
| :end | 利用終了日時 | TEXT | Yes | ISO8601(UTC)。:start ＜ :end |
| :capacity | 必要収容人数 | INTEGER | Yes | 1以上。CAPACITY がこの値以上の会議室を抽出 |
| :equipment_ids | 設備IDリスト | INTEGER | No | 必要設備の EQUIPMENT_ID(M_EQUIPMENTS.ID)。IN 句のプレースホルダはアプリ側で件数分展開する |
| :equipment_count | 設備条件件数 | INTEGER | Yes | :equipment_ids の要素数。0 のとき設備条件なし |

# 4. クエリ

同一会議室(ROOM_ID)で共通コード定義/SET-005の予約が `既存予約.START_AT ＜ :end AND 既存予約.END_AT ＞ :start` を満たすとき時間帯が重複するとみなし、重複が無い会議室のみを抽出する。

- 会議室ステータスが利用停止(共通コード定義/SET-004)の会議室は除外する(共通コード定義/SET-003 のみ)。
- 設備は指定した設備をすべて備える会議室に限定する。

```sql
SELECT
  r.ID          AS ROOM_ID,
  r.NAME        AS ROOM_NAME,
  r.CAPACITY    AS CAPACITY,
  r.LOCATION    AS LOCATION,
  r.HOURLY_RATE AS HOURLY_RATE,
  r.NOTE        AS NOTE
FROM M_ROOMS r
WHERE r.DELETED_AT IS NULL
  AND r.STATUS = 1                       -- 共通コード定義/CODE-003
  AND r.CAPACITY >= :capacity            -- 収容人数条件
  AND NOT EXISTS (                       -- 指定時間帯に重複する予約が無い
        SELECT 1
        FROM T_RESERVATIONS res
        WHERE res.ROOM_ID = r.ID
          AND res.DELETED_AT IS NULL
          AND res.STATUS = 1             -- 共通コード定義/CODE-004
          AND res.START_AT < :end
          AND res.END_AT   > :start
      )
  AND (
        :equipment_count = 0             -- 設備条件が無い場合は全会議室が対象
        OR (                             -- 指定設備をすべて備える会議室に限定
          SELECT COUNT(DISTINCT re.EQUIPMENT_ID)
          FROM M_ROOM_EQUIPMENTS re
          WHERE re.ROOM_ID = r.ID
            AND re.DELETED_AT IS NULL
            AND re.EQUIPMENT_ID IN (:equipment_ids)
        ) = :equipment_count
      )
ORDER BY r.NAME ASC, r.ID ASC;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ROOM_ID | 会議室ID | INTEGER | 空き会議室の ID |
| ROOM_NAME | 会議室名 | TEXT | 会議室名 |
| CAPACITY | 収容人数 | INTEGER | 会議室の収容人数 |
| LOCATION | 設置場所 | TEXT | 会議室の設置場所 |
| HOURLY_RATE | 利用単価 | INTEGER | 1時間あたりの利用単価(円)。0=無料 |
| NOTE | 備考 | TEXT | 会議室の備考(NULL可) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | 重複判定は TBL-003/IDX-1(IDX_RESERVATIONS_ROOM_TIME: ROOM_ID, START_AT, END_AT)、利用可判定は TBL-002/IDX-1(IDX_ROOMS_STATUS)を利用する |
| 想定件数 | 会議室は最大数百件規模。1リクエストで全会議室を走査し空き判定する |
| 注意点 | 日時は TEXT(ISO8601, UTC) の辞書順比較で判定するため、保存形式(桁数・タイムゾーン表記)を統一しておく。設備条件は「指定設備をすべて備える(AND 条件)」で判定する。ページングは API-COM §5 に従い呼び出し側で適用する。並び順は会議室名昇順(SCR-002 §3・API-002 の返却順と一致)を正とし、ORDER BY r.NAME ASC, r.ID ASC で確定する |
