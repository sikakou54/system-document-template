# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-027 |
| クエリ名 | 予約キャンセル |
| 概要 | 予約1件のステータスを 共通コード定義/SET-006 に更新し、更新後の予約を返す。 |
| 用途 | 予約のキャンセル(ReservationService)。 |
| 使用元 | MOD-003 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-003 | T_RESERVATIONS | 更新対象(予約ステータスの UPDATE) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :id | 予約ID | INTEGER | Yes | キャンセル対象の予約 ID |
| :updated_at | 更新日時 | TEXT | Yes | ISO8601(UTC)。アプリで現在時刻を設定(共通カラム.md) |
| :user_id | 予約者ユーザーID | INTEGER | Yes | 認証済み利用者本人であることを更新条件に含める |
| :now | 判定基準時刻 | TEXT | Yes | 利用開始前であることを更新条件に含める |
| :expected_updated_at | 期待更新日時 | TEXT | Yes | 一覧・詳細取得時のUPDATED_AT。変更/キャンセル競合の楽観ロックに使用 |

# 4. クエリ

本人の予約済・未開始予約だけをキャンセルへ更新する。事前判定後に完了・キャンセル・開始済みへ変化した場合は0件となる。

```sql
UPDATE T_RESERVATIONS
SET STATUS     = 2,                       -- 共通コード定義/SET-006
    UPDATED_AT = :updated_at
WHERE ID = :id
  AND USER_ID = :user_id
  AND STATUS = 1
  AND START_AT > :now
  AND UPDATED_AT = :expected_updated_at
  AND DELETED_AT IS NULL
RETURNING ID, ROOM_ID, USER_ID, TITLE, START_AT, END_AT, UNIT_PRICE, STATUS, REMIND_STATUS, UPDATED_AT;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 予約ID | INTEGER | 予約の ID |
| ROOM_ID | 会議室ID | INTEGER | 予約対象の会議室ID |
| USER_ID | ユーザーID | INTEGER | 予約者の ID |
| TITLE | 予約タイトル | TEXT | 予約タイトル |
| START_AT | 利用開始日時 | TEXT | ISO8601(UTC) |
| END_AT | 利用終了日時 | TEXT | ISO8601(UTC) |
| STATUS | 予約ステータス | INTEGER | 共通コード定義/CODE-004 |
| REMIND_STATUS | リマインド通知ステータス | INTEGER | 共通コード定義/CODE-005 |
| UNIT_PRICE | 予約確定時単価 | INTEGER | 予約時の単価スナップショット |
| UPDATED_AT | 更新日時 | TEXT | キャンセル後のRFC3339 UTC |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | 主キー(ID)による一意更新 |
| 想定件数 | 最大1件 |
| 注意点 | 事前判定はエラー分類に用いるが、最終保証は本SQLの期待所有者・期待状態・期待更新日時・未開始条件とする。0件時はMOD-003が最新状態を再取得してERR-005/013/025へ変換する |
