# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-009 |
| クエリ名 | 会議室行ロック取得 |
| 概要 | 予約の直列化のため、会議室1件を行ロック付き(SELECT ... FOR UPDATE)で取得する。 |
| 用途 | 予約登録・変更時の同一会議室に対する同時予約の直列化(行ロック取得)。 |
| 使用元 | MOD-003 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-002 | M_ROOMS | 行ロック取得対象(会議室の排他取得) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :id | 会議室ID | INTEGER | Yes | ロック取得対象会議室の ID(主キー) |

# 4. クエリ

会議室ID(ID)一致かつ有効(DELETED_AT IS NULL)の会議室を行ロック付きで1件取得する。ロック下で利用停止判定(共通コード定義/SET-004)・有料判定(HOURLY_RATE ＞ 0)に用いる。

```sql
SELECT
  r.ID          AS ID,
  r.NAME        AS NAME,
  r.HOURLY_RATE AS HOURLY_RATE,
  r.STATUS      AS STATUS
FROM M_ROOMS r
WHERE r.ID = :id
  AND r.DELETED_AT IS NULL
LIMIT 1
FOR UPDATE;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 会議室ID | INTEGER | 会議室の ID |
| NAME | 会議室名 | TEXT | 会議室名 |
| HOURLY_RATE | 利用単価 | INTEGER | 1時間あたり利用単価(円)。0=無料 / ＞0=有料 |
| STATUS | 会議室ステータス | INTEGER | 共通コード定義/CODE-003 |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | 主キー(ID)による一意検索 |
| 想定件数 | 最大1件 |
| 注意点 | 行ロック取得用(FOR UPDATE)。ロックの範囲・トランザクション境界は呼び出しモジュール(MOD-003 §6)で管理する。ロック取得後に STATUS 再判定(共通コード定義/SET-004)と有料判定を行う |
