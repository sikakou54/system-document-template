# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-041 |
| クエリ名 | 予約完了条件付き更新 |
| 概要 | 予約済みかつ終了時刻を経過した予約だけを完了へ更新する。 |
| 用途 | JOB-002 ConsumerからMOD-003を介した予約ライフサイクル更新。 |
| 使用元 | MOD-003 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-003 | T_RESERVATIONS | 予約済みから完了への状態遷移 |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明 |
|---|---|---|---|---|
| :id | 予約ID | INTEGER | Yes | 対象予約 |
| :now | 判定基準時刻 | TEXT | Yes | RFC3339 UTC |
| :updated_at | 更新日時 | TEXT | Yes | RFC3339 UTC |

# 4. クエリ

```sql
UPDATE T_RESERVATIONS
SET STATUS = 3,
    UPDATED_AT = :updated_at
WHERE ID = :id
  AND STATUS = 1
  AND END_AT <= :now
  AND DELETED_AT IS NULL
RETURNING ID, USER_ID, ROOM_ID, START_AT, END_AT, UNIT_PRICE, STATUS;
```

# 5. 出力

更新成功時は完了後予約1件、既完了・キャンセル・未終了・不在は0件を返す。既完了はエラーとせず、呼出元がSQL-040のスナップショットを用いて利用量作成へ進む。

# 6. 補足

- 本更新とTBL-007への利用量記録は別トランザクションである。
- 後段失敗時はSQL-040が「完了済みかつ利用量記録なし」を再抽出する。
