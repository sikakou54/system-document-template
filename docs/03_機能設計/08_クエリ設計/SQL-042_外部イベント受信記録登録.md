# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-042 |
| クエリ名 | 外部イベント受信記録登録 |
| 概要 | 外部サービス×外部イベントIDの一意制約に対し受信記録を1件登録し、重複受信はno-opにする(冪等Inbox)。 |
| 用途 | 外部サービス(Stripe 等)からのイベントを状態反映より先に一意記録する冪等受信(BillingService)。 |
| 使用元 | MOD-007 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-010 | T_INBOUND_EVENTS | 登録対象(受信記録の INSERT。重複時は no-op) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :external_service | 外部サービス | TEXT | Yes | イベント送信元サービス識別(例: stripe)。UX_INBOUND_EVENTS_SERVICE_EVENT |
| :external_event_id | 外部イベントID | TEXT | Yes | 送信元が付与した一意なイベント識別(例: Stripe event id)。UX_INBOUND_EVENTS_SERVICE_EVENT |
| :event_type | イベント種別 | TEXT | Yes | 送信元のイベント種別(例: customer.subscription.updated) |
| :occurred_at | イベント発生日時 | TEXT | Yes | 送信元でのイベント発生日時(RFC3339 UTC) |
| :received_at | 受信日時 | TEXT | Yes | 真正性検証後に受信した日時(RFC3339 UTC)。アプリで現在時刻を設定 |
| :correlation_id | 相関ID | TEXT | Yes | API・請求・監査ログを横断して処理を追跡する識別 |
| :user_id | 関連利用者ID | INTEGER | No | イベントから利用者を識別できる場合のみ設定。不明時は NULL |
| :invoice_ref | 関連請求識別 | TEXT | No | 請求関連イベント時の外部請求識別。無ければ NULL |
| :payload | 再処理情報 | TEXT | Yes | 状態反映を再現する最小限のイベント情報。機微情報は含めない |

# 4. クエリ

外部サービス×外部イベントIDの一意制約(UX_INBOUND_EVENTS_SERVICE_EVENT)で受信記録を INSERT する。

- 同一イベントの再受信は ON CONFLICT DO NOTHING で重複レコードを作成しない(冪等)。
- STATUS は既定値の受信済(共通コード定義/CODE-008=1)で登録し、ATTEMPT_COUNT・PROCESSED_AT・LAST_ERROR は既定のまま登録する。
- 実際に挿入した場合のみ行を返す。重複受信では行を返さない(RETURNING の有無=changes() で重複を判定)。

```sql
INSERT INTO T_INBOUND_EVENTS (
  EXTERNAL_SERVICE, EXTERNAL_EVENT_ID, EVENT_TYPE, OCCURRED_AT, RECEIVED_AT,
  CORRELATION_ID, USER_ID, INVOICE_REF, PAYLOAD
) VALUES (
  :external_service, :external_event_id, :event_type, :occurred_at, :received_at,
  :correlation_id, :user_id, :invoice_ref, :payload   -- STATUS は既定値 1(受信済) 共通コード定義/CODE-008
)
ON CONFLICT (EXTERNAL_SERVICE, EXTERNAL_EVENT_ID) DO NOTHING
RETURNING ID, EXTERNAL_SERVICE, EXTERNAL_EVENT_ID, STATUS;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 外部イベント受信記録ID | INTEGER | 採番された受信記録の ID(新規挿入時のみ) |
| EXTERNAL_SERVICE | 外部サービス | TEXT | イベント送信元サービス識別 |
| EXTERNAL_EVENT_ID | 外部イベントID | TEXT | 送信元が付与した一意なイベント識別 |
| STATUS | 処理状態 | INTEGER | 登録後の処理状態(共通コード定義/CODE-008。新規は受信済=1) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | TBL-010/IDX-1(UX_INBOUND_EVENTS_SERVICE_EVENT: EXTERNAL_SERVICE, EXTERNAL_EVENT_ID) |
| 想定件数 | 最大1件(1イベント1行) |
| 注意点 | ON CONFLICT の対象は一意インデックス UX_INBOUND_EVENTS_SERVICE_EVENT。SELECT..FOR UPDATE は用いず、ON CONFLICT DO NOTHING と RETURNING/changes() で重複受信を判定する(Cloudflare D1(SQLite)で実行可能)。行が返れば新規受信、返らなければ重複受信(no-op)として後続の状態反映をスキップする。RECEIVED_AT はアプリで現在時刻を設定する |
