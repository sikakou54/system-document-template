# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-040 |
| クエリ名 | 完了・利用量再処理対象予約取得 |
| 概要 | 終了済みの予約済予約、または完了済みかつ利用量記録がない有料予約を取得する。 |
| 用途 | JOB-002 Producerが、予約完了と利用量作成を再実行可能な単位へ分割する。 |
| 使用元 | MOD-003 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-003 | T_RESERVATIONS | 終了時刻、状態、予約時単価の取得 |
| TBL-007 | T_USAGE_RECORDS | 完了済み予約の利用量未作成判定 |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明 |
|---|---|---|---|---|
| :now | 判定基準時刻 | TEXT | Yes | RFC3339 UTC |
| :limit | 取得上限 | INTEGER | Yes | 1〜100 |

# 4. クエリ

```sql
SELECT
  r.ID,
  r.USER_ID,
  r.ROOM_ID,
  r.START_AT,
  r.END_AT,
  r.UNIT_PRICE,
  r.STATUS
FROM T_RESERVATIONS r
WHERE r.DELETED_AT IS NULL
  AND r.END_AT <= :now
  AND (
    r.STATUS = 1
    OR (
      r.STATUS = 3
      AND r.UNIT_PRICE > 0
      AND NOT EXISTS (
        SELECT 1
        FROM T_USAGE_RECORDS u
        WHERE u.RESERVATION_ID = r.ID
          AND u.DELETED_AT IS NULL
      )
    )
  )
ORDER BY r.END_AT, r.ID
LIMIT :limit;
```

# 5. 出力

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 予約ID | INTEGER | 処理単位 |
| USER_ID | 利用者ID | INTEGER | 課金対象 |
| ROOM_ID | 会議室ID | INTEGER | 利用対象 |
| START_AT / END_AT | 利用日時 | TEXT | 利用時間算出元 |
| UNIT_PRICE | 予約確定時単価 | INTEGER | 会議室マスタの現在値を参照しない |
| STATUS | 予約状態 | INTEGER | 予約済みは完了更新、完了済みは利用量作成のみ再実行 |

# 6. 補足

- 「予約完了TX成功・利用量記録TX失敗」の中間状態を意図的に再抽出する。
- TBL-003/IDX-4とTBL-007の予約ID一意インデックスを使用する。
