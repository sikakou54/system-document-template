# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-007 |
| クエリ名 | ユーザー課金契約状態更新 |
| 概要 | StripeサブスクリプションID一致のユーザーの課金契約状態(BILLING_STATUS)を更新する。 |
| 用途 | Webhook(customer.subscription.updated / deleted)による契約状態反映。 |
| 使用元 | MOD-007 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-001 | M_USERS | 更新対象(課金契約状態の反映) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :billing_status | 課金契約状態 | INTEGER | Yes | 共通コード定義/CODE-002 |
| :stripe_subscription_id | StripeサブスクリプションID | TEXT | Yes | 対象ユーザーのサブスクリプションID |
| :updated_at | 更新日時 | TEXT | Yes | ISO8601(UTC)。アプリで現在時刻を設定(共通カラム.md) |

# 4. クエリ

StripeサブスクリプションID(STRIPE_SUBSCRIPTION_ID)一致かつ有効(DELETED_AT IS NULL)のユーザーの BILLING_STATUS を更新する。

```sql
UPDATE M_USERS
SET BILLING_STATUS = :billing_status,                 -- 共通コード定義/CODE-002
    UPDATED_AT     = :updated_at
WHERE STRIPE_SUBSCRIPTION_ID = :stripe_subscription_id
  AND DELETED_AT IS NULL
RETURNING ID, BILLING_STATUS;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | ユーザーID | INTEGER | 更新したユーザーの ID |
| BILLING_STATUS | 課金契約状態 | INTEGER | 反映後の課金契約状態(共通コード定義/CODE-002) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | STRIPE_SUBSCRIPTION_ID による絞り込み(Stripe 側でサブスク一意) |
| 想定件数 | 最大1件 |
| 注意点 | サブスク状態(active / deleted 等)を BILLING_STATUS(共通コード定義/CODE-002)へ写像する判定は呼び出しモジュール(MOD-007)で行う。区分値は 共通コード定義/CODE-002 |
