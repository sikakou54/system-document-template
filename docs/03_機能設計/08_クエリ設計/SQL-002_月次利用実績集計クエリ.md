# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-002 |
| クエリ名 | 月次利用実績集計クエリ |
| 概要 | 会議室ごとに、対象月の完了予約の件数と利用時間(分)を集計する。 |
| 用途 | 利用実績レポート(会議室別の月次利用実績集計)。 |
| 使用元 | MOD-005 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-003 | T_RESERVATIONS | 集計対象(完了予約の件数・利用時間) |
| TBL-002 | M_ROOMS | 会議室名の取得 |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :target_month | 対象月 | TEXT | Yes | 'YYYY-MM' 形式(Asia/Tokyo 基準の年月) |

# 4. クエリ

会議室(ROOM_ID)ごとに、対象月に開始した 共通コード定義/SET-007 の予約を集計する。

- 件数は予約件数、利用時間(分)は予約1件ごとの `CAST((julianday(END_AT) - julianday(START_AT)) * 24 * 60 AS INTEGER)` の合計とする。
- 対象月の判定は、UTC 保存の START_AT を Asia/Tokyo(+9時間)に変換した年月で行う。

```sql
SELECT
  res.ROOM_ID   AS ROOM_ID,
  r.NAME        AS ROOM_NAME,
  :target_month AS TARGET_MONTH,
  COUNT(res.ID) AS RESERVATION_COUNT,
  SUM(
    CAST((julianday(res.END_AT) - julianday(res.START_AT)) * 24 * 60 AS INTEGER)
  )             AS TOTAL_MINUTES
FROM T_RESERVATIONS res
INNER JOIN M_ROOMS r
  ON r.ID = res.ROOM_ID
WHERE res.DELETED_AT IS NULL
  AND res.STATUS = 3                                              -- 完了予約のみ集計
  AND strftime('%Y-%m', datetime(res.START_AT, '+9 hours')) = :target_month
GROUP BY res.ROOM_ID, r.NAME
ORDER BY res.ROOM_ID ASC;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ROOM_ID | 会議室ID | INTEGER | 集計対象の会議室ID |
| ROOM_NAME | 会議室名 | TEXT | 会議室名 |
| TARGET_MONTH | 対象月 | TEXT | 集計対象の年月('YYYY-MM') |
| RESERVATION_COUNT | 予約件数 | INTEGER | 対象月の完了予約の件数(COUNT) |
| TOTAL_MINUTES | 利用時間(分) | INTEGER | 完了予約の利用時間(分)の合計。1件=CAST((julianday(END_AT)-julianday(START_AT))*24*60 AS INTEGER) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | TBL-003 の STATUS・START_AT による絞り込み。時系列の絞り込みには TBL-003/IDX-2(IDX_RESERVATIONS_USER_START の START_AT)等が寄与する |
| 想定件数 | 月あたりの完了予約を全会議室分集計する。集計結果は TBL-006(T_USAGE_REPORTS)へ ROOM_ID×TARGET_MONTH 単位で保存する |
| 注意点 | 対象月の境界は Asia/Tokyo 基準で判定するため START_AT を +9時間補正してから年月抽出する。当月に完了予約が無い会議室は結果に現れない(表示側で件数0として扱う)。julianday による差分は分単位に丸めるため秒未満は切り捨てられる |
