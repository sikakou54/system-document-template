# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-039 |
| クエリ名 | 監査ログ登録 |
| 概要 | 重要操作の監査ログを1件追記し、採番された監査ログを返す。 |
| 用途 | 予約・課金・会議室管理などの重要操作時の監査ログ記録(監査ログサービス)。 |
| 使用元 | MOD-009 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-009 | T_AUDIT_LOGS | 登録対象(監査ログの INSERT) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :operated_at | 操作日時 | TEXT | Yes | 操作が行われた日時(ISO8601) |
| :user_id | 利用者ID | INTEGER | Yes | 操作を行った利用者の ID |
| :operation_type | 操作種別 | TEXT | Yes | 予約操作／課金操作／会議室管理操作 など |
| :operation_target | 操作対象 | TEXT | Yes | 操作対象の識別(対象の種別＋対象ID等) |
| :operation_result | 操作結果 | TEXT | Yes | 成功／失敗 |

# 4. クエリ

監査ログを INSERT する。登録した監査ログを返す。

```sql
INSERT INTO T_AUDIT_LOGS (
  OPERATED_AT, USER_ID, OPERATION_TYPE, OPERATION_TARGET, OPERATION_RESULT
) VALUES (
  :operated_at, :user_id, :operation_type, :operation_target, :operation_result
)
RETURNING ID, OPERATED_AT, USER_ID, OPERATION_TYPE, OPERATION_TARGET, OPERATION_RESULT;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 監査ログID | INTEGER | 採番された監査ログの ID |
| OPERATED_AT | 操作日時 | TEXT | 操作が行われた日時(ISO8601) |
| USER_ID | 利用者ID | INTEGER | 操作を行った利用者の ID |
| OPERATION_TYPE | 操作種別 | TEXT | 予約操作／課金操作／会議室管理操作 など |
| OPERATION_TARGET | 操作対象 | TEXT | 操作対象の識別 |
| OPERATION_RESULT | 操作結果 | TEXT | 成功／失敗 |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | IDX_AUDIT_LOGS_USER(USER_ID)・IDX_AUDIT_LOGS_OPERATED_AT(OPERATED_AT)は参照(追跡)用。登録では使用しない |
| 想定件数 | 1件 |
| 注意点 | 呼び出し元(MOD-009)の業務トランザクション内で実行する。監査ログは追記のみとし、更新・削除は行わない(NFR-006)。操作日時は呼び出しモジュールで設定する |
