# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-003 |
| クエリ名 | ユーザー取得 |
| 概要 | メールアドレス一致かつ論理削除されていない有効なユーザーを1件取得する。該当が無い場合は 0件。 |
| 用途 | ログイン認証(資格情報検証時のユーザー取得)。 |
| 使用元 | MOD-001 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-001 | M_USERS | 抽出対象(メールアドレスによるユーザー取得) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :email | メールアドレス | TEXT | Yes | ログインID。UX_USERS_EMAIL で一意 |

# 4. クエリ

メールアドレス(EMAIL)一致かつ有効(DELETED_AT IS NULL)のユーザーを1件取得する。パスワード照合は MOD-008 を経由して行うため PASSWORD_HASH を返す。

```sql
SELECT
  u.ID             AS ID,
  u.NAME           AS NAME,
  u.EMAIL          AS EMAIL,
  u.PASSWORD_HASH  AS PASSWORD_HASH,
  u.ROLE           AS ROLE,
  u.BILLING_STATUS AS BILLING_STATUS
FROM M_USERS u
WHERE u.EMAIL = :email
  AND u.DELETED_AT IS NULL
LIMIT 1;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | ユーザーID | INTEGER | ユーザーの ID |
| NAME | ユーザー名 | TEXT | ユーザー名 |
| EMAIL | メールアドレス | TEXT | メールアドレス(ログインID) |
| PASSWORD_HASH | パスワードハッシュ | TEXT | MOD-008 で定義した方式のパスワードハッシュ |
| ROLE | ユーザーロール | INTEGER | 共通コード定義/CODE-001 |
| BILLING_STATUS | 課金契約状態 | INTEGER | 共通コード定義/CODE-002 |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | TBL-001/IDX-1(UX_USERS_EMAIL: EMAIL) |
| 想定件数 | EMAIL は一意のため最大1件 |
| 注意点 | パスワード照合は MOD-001 から MOD-008 を利用して行う。本クエリは有効ユーザーの取得のみを行う |
