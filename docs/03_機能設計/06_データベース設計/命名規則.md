# 1. 基本方針

| 対象 | 規則 |
|---|---|
| テーブル(物理名) | 種別接頭辞 + 複数形スネークケース(すべて大文字)。接頭辞: M_=マスタ / T_=トランザクション / H_=履歴 / TP_=一時テーブル |
| テーブル(論理名) | 「XXX + 種別」。種別接尾辞: マスタ / トランザクション / 履歴 / 一時テーブル |
| カラム | スネークケース(すべて大文字) |
| PK | ID |
| FK | 参照先単数形_ID |
| Index | IDX_{テーブル}_{内容}(大文字。テーブルは接頭辞を除いた基幹名) |
| UQ(一意インデックス) | UX_{テーブル}_{カラム}(大文字) |

# 2. 命名例

| 対象 | 例 | 補足 |
|---|---|---|
| テーブル(マスタ) | 物理名 M_USERS / 論理名 ユーザーマスタ | 接頭辞 M_、接尾辞 マスタ |
| テーブル(トランザクション) | 物理名 T_RESERVATIONS / 論理名 予約トランザクション | 接頭辞 T_、接尾辞 トランザクション |
| カラム | START_AT | スネークケース(大文字) |
| PK | ID | 全テーブル共通(各TBL §2 カラム定義) |
| FK | ROOM_ID | M_ROOMS(複数形)→ ROOM(単数形)+ _ID |
| Index | IDX_RESERVATIONS_ROOM_TIME | TBL-003/IDX-1 に対応 |
| UQ | UX_USERS_EMAIL | USERS.EMAIL の一意インデックス |

# 3. 型方針(Cloudflare D1 / SQLite)

DB基盤は Cloudflare D1(SQLite)。SQLite の型に合わせ、以下の型注記で記載する。

| 種別 | 型 | 補足 |
|---|---|---|
| 整数 | INTEGER | 収容人数・利用単価・金額・利用時間(分)など |
| 文字列 | TEXT | 文字数上限は型で表さず、制約・アプリケーション検証で担保する |
| 真偽 | INTEGER | 0=偽 / 1=真 |
| 日時 | TEXT | ISO8601形式(UTC保存・表示は Asia/Tokyo)。例: `2026-07-04T09:00:00Z` |
| 主キー | INTEGER PRIMARY KEY AUTOINCREMENT | 各TBL §2 カラム定義に PK=◯ で記載 |
| 区分値 | INTEGER + CHECK | 型は INTEGER とし `CHECK(列 IN (...))` を付与。値は 10_共通定義設計/DEF-001_共通コード定義.md で定義 |

- Cloudflare D1(SQLite)を前提とし、`unsigned` / `varchar(n)` / `datetime` / `ENUM` / `AUTO_INCREMENT` は使用しない。
- 接頭辞 M_/T_、スネークケース大文字、PK=ID、FK=参照先単数形_ID は本 D1 前提でも維持する。
