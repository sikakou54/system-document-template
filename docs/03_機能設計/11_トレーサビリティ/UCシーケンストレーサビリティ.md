# 1. 概要

要求・機能要件・システムユースケースからシーケンス設計への対応を一元管理する。
RQ→FR/CFRは要件定義側の `要求要件トレーサビリティ.md`、FR/CFR→UCは各機能要件文書を正本とし、本表はSEQへの接続を管理する。

# 3. 更新ルール

| 更新契機 | 更新内容 |
|---|---|
| RQ・FR/CFR・UCの追加・変更 | 根拠となるIDと対象SEQを更新する |
| SEQの作成・分割・統合 | SEQ ID、作成要否、状態を更新する |
| SEQを省略 | SEQ IDを「-」、作成要否を「省略」とし、静的表示・単純CRUDなど具体的な理由を記載する |
| ID廃止 | 行を削除せず状態を「廃止」にする |

- 本表はFR/UC→SEQ対応の正本であり、関連文書の作成・更新と同一作業内で更新する。
