# 1. 概要

システムユースケース(UC。各FR/CFR文書内に定義)を実現する画面(SCR)・API の対応を追跡する。
画面SCR は各 SCR のトレース元(02_画面設計)、API は各 API のトレース元(03_API設計)からの転記で構成する。

# 2. 対応表

| UC ID | 画面SCR | API | 備考 |
|---|---|---|---|
| FR-001/UC-01 空き会議室を検索する | SCR-002 | API-002, API-009 | API-009(設備一覧取得)は FR-005/UC-01 と共有 |
| FR-002/UC-01 会議室を予約する | SCR-002, SCR-003 | API-002, API-003 | SCR-002/API-002(検索)は FR-001/UC-01 と共有 |
| FR-003/UC-01 予約を変更する | SCR-004 | API-004, API-006 | - |
| FR-003/UC-02 予約をキャンセルする | SCR-004 | API-005, API-006 | - |
| FR-005/UC-01 会議室を登録・編集する | SCR-005 | API-007, API-009, API-013, API-014 | API-009(設備一覧取得)は FR-001/UC-01 と共有 |
| FR-006/UC-01 利用実績を確認する | SCR-006 | API-008 | - |
| CFR-001/UC-01 ログインする | SCR-001 | API-001 | - |

# 3. 更新ルール

| 更新契機 | 更新内容 |
|---|---|
| SCR の新規作成・トレース元変更 | 対応する UC 行の画面SCR を SCR のトレース元から転記する |
| API の新規作成・トレース元変更 | 対応する UC 行の API を API のトレース元から転記する |
| ID廃止 | 行は削除せず、備考に「廃止」と記す |

- 本対応表の更新は関連文書の作成・更新と同一作業内で行う(正本: docs/CLAUDE.md §6・§10-7)。
