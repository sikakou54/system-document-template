# 1. 概要

バッチ・定期実行・非同期処理のジョブを定義する。各ジョブの詳細は JOB-XXX_ジョブ名.md を参照する。

- 作成・更新前に `../11_トレーサビリティ/シーケンス設計トレーサビリティ.md` で関連SEQを特定し、起動契機・非同期連携・再試行・例外処理を反映する。
- JOBからTBL/SQLへ直接アクセスせず、データ処理はMODへ委譲する。

# 2. 一覧

| ID | 名称 | 概要 | 実行契機 |
|---|---|---|---|
| JOB-001 | 予約リマインド通知 | 開始30分以内の予約済・リマインド未送信の予約を MOD-006(通知サービス)で抽出し、対象ごとの送信メッセージを Cloudflare Queues へ投入する。Queue Consumer が MOD-006 で1件ずつ送信し送信状態を更新する(送信の再試行・DLQ は Queue境界、DLQ到達の管理者アラートは運用) | 定期 |
| JOB-002 | 利用量の従量課金計上 | 終了時刻を過ぎた予約済の予約を完了にし、有料会議室(利用単価>0)の予約は MOD-007(課金サービス)が利用量を計上して Stripe へ Meter Event として送信する(送信失敗は次回再送) | 定期 |
| JOB-003 | 月次利用実績集計 | 完了予約を会議室×対象月で集計し、月次利用実績(TBL-006)を MOD-005 経由で保存する。利用実績レポート(SCR-006/API-008)の参照元を最新化する | 定期 |
