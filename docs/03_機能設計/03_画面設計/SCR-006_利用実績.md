# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 画面ID | SCR-006 |
| 画面名 | 利用実績 |
| 概要 | 管理者が対象月を指定し、会議室ごとの月次利用実績（予約件数・利用時間）を確認する画面 |
| トレース元 | FR-006/UC-01 |
| URL / ルート | /admin/reports |
| 利用可能ロール | 管理者 |

# 2. 画面レイアウト

![利用実績 画面レイアウト](mockups/SCR-006.png)

# 3. 初期表示

| 項目 | 内容 |
|---|---|
| 表示時に呼び出すAPI | API-008 |
| デフォルト値 | 対象月=当月 |
| ソート順 | 会議室名 昇順(API-008 の返却順) |
| 0件時の表示 | MSG-025 を表示し、実績一覧を非表示にする |

# 4. 画面項目

| 項目ID | 項目名 | 種別 | 表示/入力 | 必須 | 初期値 | 備考 |
|---|---|---|---|---|---|---|
| ITM-01 | 対象月 | month | 入力 | Yes | 当月 | YYYY-MM 形式 |
| ITM-02 | 表示ボタン | button | 入力 | - | - | EVT-01 を発火 |
| ITM-03 | 実績一覧 | 一覧 | 表示 | - | - | 会議室名・予約件数・利用時間（分）を表示。集計対象は完了した予約 |
| ITM-04 | ページャ | button | 入力 | - | - | 前へ / 次へ / ページ番号。EVT-02 を発火（API-008 §5・API-COM §5 のページネーションに対応） |

# 5. 画面イベント

| イベントID | イベント名 | 発火条件 | 呼び出しAPI | 成功時 | 失敗時 |
|---|---|---|---|---|---|
| EVT-01 | 実績表示 | 表示ボタン押下 | API-008 | 指定した対象月の会議室別実績を実績一覧(ITM-03)に表示。総件数に応じてページャ(ITM-04)を表示 | ERR-018 発生時 MSG-058 を表示し画面に留まる。ERR-006 発生時 MSG-026 を表示し画面に留まる。ERR-002 発生時 MSG-057 を表示。ERR-001 発生時は SCR-001(ログイン)へ遷移 |
| EVT-02 | ページ切替 | ページャ(ITM-04)の操作 | API-008 | 指定ページの実績を実績一覧(ITM-03)に再表示(対象月は維持し page のみ変更) | ERR-006 発生時 MSG-026 を表示。ERR-001 発生時は SCR-001(ログイン)へ遷移 |

# 7. 表示制御

| 条件 | 対象 | 制御内容 |
|---|---|---|
| ロールが一般ユーザー | 画面全体 | 非表示（アクセス不可） |
| 実績が0件 | 実績一覧 | 非表示 |
| 実績が0件 | 0件メッセージ（MSG-025） | 表示 |
| 総件数 ＜＝ 1ページ表示件数（API-COM §5 の既定 limit） | ページャ（ITM-04） | 非表示 |

# 8. 画面遷移

| 遷移先 | トリガ |
|---|---|
| SCR-001 | API 呼び出しで ERR-001(認証失敗・トークン失効)を受信、または未認証で本画面へアクセス |
| - | 上記以外の他画面への遷移なし（利用実績の確認は本画面内で完結） |

# 9. メッセージ一覧

本画面が参照する画面表示文言(MSG)を以下にインライン定義する。対応ERR は当該メッセージの表示契機となるエラー(なしは -)。

| MSG ID | 種別 | 文言 | 対応ERR |
|---|---|---|---|
| MSG-025 | 情報 | 対象月の利用実績はありません。 | - |
| MSG-026 | エラー | 対象月は YYYY-MM 形式で指定してください。 | ERR-019 |
| MSG-057 | エラー | この機能は管理者のみ利用できます。 | ERR-002 |
| MSG-058 | エラー | 対象月を正しく指定してください。 | ERR-018 |
