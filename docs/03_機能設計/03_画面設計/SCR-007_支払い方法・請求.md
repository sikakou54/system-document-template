# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 画面ID | SCR-007 |
| 画面名 | 支払い方法・請求 |
| 概要 | 有料会議室の従量課金に用いる支払い方法（従量サブスク）を登録し、当月の利用量・請求見込みと請求履歴を確認する画面 |
| トレース元 | FR-008/UC-01, FR-008/UC-02 |
| URL / ルート | /billing |
| 利用可能ロール | 共通コード定義/CODE-001 |

# 2. 画面レイアウト

# 3. 初期表示

| 項目 | 内容 |
|---|---|
| 表示時に呼び出すAPI | API-012 |
| デフォルト値 | 対象月=当月。契約状態・対象月利用量・対象月請求見込み・請求履歴を表示 |
| ソート順 | 請求履歴は請求対象月 降順 |
| 0件時の表示 | 請求履歴が0件の場合は請求履歴が無い旨を表示し、請求履歴一覧を非表示にする |

# 4. 画面項目

| 項目ID | 項目名 | 種別 | 表示/入力 | 必須 | 初期値 | 備考 |
|---|---|---|---|---|---|---|
| ITM-01 | 契約状態 | label | 表示 | - | - | 課金契約状態（共通コード定義/CODE-002） |
| ITM-02 | 支払い方法登録ボタン | button | 入力 | - | - | EVT-01 を発火（Stripe Checkout へ遷移） |
| ITM-03 | 対象月利用量 | label | 表示 | - | - | 対象月の利用時間（分） |
| ITM-04 | 対象月請求見込み | label | 表示 | - | - | 対象月の請求見込み金額（円） |
| ITM-05 | 請求履歴一覧 | 一覧 | 表示 | - | - | 請求対象月・請求金額・請求ステータス（共通コード定義/CODE-007）を表示 |
| ITM-06 | 対象月 | month | 入力 | No | 当月 | 利用量・請求見込みの対象月。YYYY-MM 形式。変更で EVT-03 を発火 |
| ITM-07 | 請求履歴ページャ | button | 入力 | - | - | 前へ / 次へ / ページ番号。EVT-05 を発火（API-012 の請求履歴ページネーションに対応） |

# 5. 画面イベント

| イベントID | イベント名 | 発火条件 | 呼び出しAPI | 成功時 | 失敗時 |
|---|---|---|---|---|---|
| EVT-01 | 支払い方法登録 | 支払い方法登録ボタン押下 | API-010 | 本画面の success_url（成功時=登録完了確認、EVT-02）・cancel_url（中断時=登録中断、EVT-04）を API-010 に渡し、返却された checkout_url の Stripe Checkout（サブスク登録）画面へ遷移 | ERR-009 発生時 CFR-004 に従いエラー表示。ERR-001 発生時は SCR-001(ログイン)へ遷移 |
| EVT-02 | 登録完了確認 | Stripe Checkout の success_url から本画面へ戻った時 | API-012 | API-012 を再取得し、billing_status が 共通コード定義/SET-001 なら MSG-013 表示・契約状態/利用量/請求を更新。まだ 共通コード定義/CODE-002 の未完了状態なら Webhook 未達とみなし MSG-029 を表示して一定間隔で API-012 を再取得(ポーリング。上限回数まで)。上限まで 共通コード定義/SET-001 にならない場合は MSG-030 を表示し初期表示状態に戻す | ERR-001 発生時は SCR-001(ログイン)へ遷移 |
| EVT-03 | 対象月変更 | 対象月(ITM-06)の変更 | API-012 | 指定した対象月で API-012 を再取得し、対象月利用量(ITM-03)・対象月請求見込み(ITM-04)を更新 | ERR-006 発生時 MSG-031 表示。ERR-001 発生時は SCR-001(ログイン)へ遷移 |
| EVT-04 | 登録中断 | Stripe Checkout の cancel_url から本画面へ戻った時 | - | 完了メッセージを表示せず、初期表示状態(共通コード定義/CODE-002・登録ボタン活性)を維持する(登録は行われない) | - |
| EVT-05 | 請求履歴ページ切替 | 請求履歴ページャ(ITM-07)の操作 | API-012 | 指定ページで API-012 を再取得し、請求履歴一覧(ITM-05)を更新(対象月は維持) | ERR-001 発生時は SCR-001(ログイン)へ遷移 |

# 7. 表示制御

| 条件 | 対象 | 制御内容 |
|---|---|---|
| 契約状態が未契約（1） | 支払い方法登録ボタン | 活性 |
| 契約状態が未契約（1） | 従量課金の案内（MSG-036） | 表示 |
| 契約状態が有効（2） | 支払い方法登録ボタン | 非活性 |
| 契約状態が停止（3） | 支払い方法登録ボタン | 活性（再登録可） |
| 契約状態が停止（3） | 従量課金の案内（MSG-036） | 表示 |
| 請求履歴が0件 | 請求履歴一覧 | 非表示 |
| 請求履歴総件数 ＜＝ 1ページ表示件数（API-COM §5 の既定 limit） | 請求履歴ページャ（ITM-07） | 非表示 |

# 8. 画面遷移

| 遷移先 | トリガ |
|---|---|
| Stripe Checkout（外部・Stripe） | 支払い方法登録ボタン押下（EVT-01） |
| SCR-007 | Stripe Checkout 成功後に success_url から本画面へ戻る（EVT-02） |
| SCR-007 | Stripe Checkout 中断後に cancel_url から本画面へ戻る（EVT-04） |
| SCR-001 | API 呼び出しで ERR-001(認証失敗・トークン失効)を受信、または未認証で本画面へアクセス |

# 9. メッセージ一覧

本画面が参照する画面表示文言(MSG)を以下にインライン定義する。対応ERR は当該メッセージの表示契機となるエラー(なしは -)。

| MSG ID | 種別 | 文言 | 対応ERR |
|---|---|---|---|
| MSG-036 | 情報 | この会議室は有料です。利用時間に応じて従量課金されます。 | - |
| MSG-013 | 完了 | 支払い方法を登録しました。 | - |
| MSG-029 | 情報 | 支払い方法の登録処理中です。しばらくお待ちください。 | - |
| MSG-030 | エラー | 支払い方法を登録できませんでした。お手数ですが、もう一度お試しください。 | - |
| MSG-031 | エラー | 対象月は YYYY-MM 形式で指定してください。 | ERR-019 |
