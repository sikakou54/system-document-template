# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 名称 | 共通コード定義 |
| 種別 | Code |
| 概要 | テーブル、API、モジュール、JOB、画面、クエリから共通参照されるコード値を定義する |

# 2. 定義ルール

| 項目 | 内容 |
|---|---|
| 正本 | コード値・表示名・説明は本書を正本とする |
| 参照方法 | 他文書ではコード値を `共通コード定義/CODE-XXX`、設定・判定基準を `共通コード定義/SET-XXX` で参照する |
| 値の扱い | コード値の削除・再利用は禁止する。廃止する場合は状態を「廃止」として残す |
| 物理制約 | テーブル設計では CHECK 制約など実装上必要な制約を記載してよい。ただし値の意味は本書を参照する |

# 3. コード定義

## CODE-001 ユーザーロール

| 値 | 表示名 | 説明 | 状態 |
|---|---|---|---|
| 1 | 一般 | 一般ユーザー(社員) | 有効 |
| 2 | 管理者 | 管理者(総務) | 有効 |

## CODE-002 課金契約状態

| 値 | 表示名 | 説明 | 状態 |
|---|---|---|---|
| 1 | 未契約 | 支払い方法(従量サブスク)が未登録。有料会議室は利用不可(ERR-008) | 有効 |
| 2 | 有効 | 従量課金サブスク契約中。有料会議室を利用できる | 有効 |
| 3 | 停止 | 支払い失敗・解約等でサブスクが停止した状態 | 有効 |

## CODE-003 会議室ステータス

| 値 | 表示名 | 説明 | 状態 |
|---|---|---|---|
| 1 | 利用可 | 予約・検索対象の会議室 | 有効 |
| 2 | 利用停止 | 新規予約・検索の対象外 | 有効 |

## CODE-004 予約ステータス

| 値 | 表示名 | 説明 | 状態 |
|---|---|---|---|
| 1 | 予約済 | 有効な予約 | 有効 |
| 2 | キャンセル | キャンセルされた予約 | 有効 |
| 3 | 完了 | 利用が終了した予約 | 有効 |

## CODE-005 リマインド通知ステータス

| 値 | 表示名 | 説明 | 状態 |
|---|---|---|---|
| 1 | 未送信 | リマインドメール未送信 | 有効 |
| 2 | 送信済 | リマインドメール送信済み | 有効 |
| 3 | 失敗 | 送信リトライ上限に達し失敗 | 有効 |

## CODE-006 計上ステータス

| 値 | 表示名 | 説明 | 状態 |
|---|---|---|---|
| 1 | 未送信 | Stripe Meter Event 未送信 | 有効 |
| 2 | 送信済 | Stripe Meter Event 送信済み | 有効 |
| 3 | 失敗 | 送信リトライ上限に達し失敗(ERR-009) | 有効 |

## CODE-007 請求ステータス

| 値 | 表示名 | 説明 | 状態 |
|---|---|---|---|
| 1 | 確定前 | Stripe 上で draft(集計中・未確定) | 有効 |
| 2 | 請求済 | Stripe 上で open(請求書発行済・未払い) | 有効 |
| 3 | 支払済 | Stripe 上で paid(支払い完了) | 有効 |
| 4 | 失敗 | 支払い失敗(invoice.payment_failed) | 有効 |

# 4. 設定基準

コード値を設定・判定する場面の基準は本セクションを正本とする。他文書では基準IDまたはコードIDを参照し、値と意味の対応を再定義しない。

| 基準ID | 対象コード | 利用場面 | 設定・判定する状態 |
|---|---|---|---|
| SET-001 | CODE-002 課金契約状態 | Stripe Checkout 完了時 | 有効 |
| SET-002 | CODE-002 課金契約状態 | Stripe サブスクリプション停止・削除時 | 停止 |
| SET-003 | CODE-003 会議室ステータス | 会議室登録時の未指定既定 | 利用可 |
| SET-004 | CODE-003 会議室ステータス | 予約可否判定で拒否する状態 | 利用停止 |
| SET-005 | CODE-004 予約ステータス | 予約登録時の初期状態 | 予約済 |
| SET-006 | CODE-004 予約ステータス | 予約キャンセル時 | キャンセル |
| SET-007 | CODE-004 予約ステータス | 利用終了後の計上対象確定時 | 完了 |
| SET-008 | CODE-005 リマインド通知ステータス | 予約登録時の初期状態 | 未送信 |
| SET-009 | CODE-005 リマインド通知ステータス | リマインド送信成功時 | 送信済 |
| SET-010 | CODE-005 リマインド通知ステータス | リマインド送信失敗時 | 失敗 |
| SET-011 | CODE-006 計上ステータス | 利用量記録登録時の初期状態 | 未送信 |
| SET-012 | CODE-006 計上ステータス | Meter Event 送信成功時 | 送信済 |
| SET-013 | CODE-006 計上ステータス | Meter Event 送信失敗時 | 失敗 |
| SET-014 | CODE-007 請求ステータス | Stripe invoice.paid 受信時 | 支払済 |
| SET-015 | CODE-007 請求ステータス | Stripe invoice.payment_failed 受信時 | 失敗 |
