# 1. 基本情報

| 項目 | 内容 |
|---|---|
| API ID | API-012 |
| API名 | 利用量・請求取得 |
| メソッド | GET |
| パス | /api/billing/usage |
| 認証 | 要 |
| 認可 | 一般=可, 管理者=可(いずれも本人のデータのみ) |
| 冪等性 | あり(参照系) |
| トレース元 | UC-009 |
| 概要 | 認証済み利用者本人の当月の利用量(利用時間・請求見込み金額)、課金契約状態、および過去の請求履歴を取得する。 |

# 2. リクエスト

| 項目名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|
| 対象月 | string | No | YYYY-MM 形式。利用量・請求見込みの対象月。未指定時は当月(Asia/Tokyo 基準) |
| ページ | int | No | 請求履歴のページネーション(API-COM §5)。既定 1 |
| 取得件数 | int | No | 請求履歴のページネーション(API-COM §5)。既定 20 |

# 3. レスポンス

| 項目 | 内容 |
|---|---|
| HTTPステータス | 200 |

| 項目名 | 型 | 説明 |
|---|---|---|
| 課金契約状態 | int | TBL-001/ENM-2(1=未契約 / 2=有効 / 3=停止) |

| 対象月利用時間分 | int | 対象月の完了予約の合計利用時間(分) |

この API の基本フローをフローチャートで定義する。

```mermaid
flowchart TD
  Start([開始])
  P1["(1) 利用量・請求取得"]
  P2["(2) 処理結果"]
  End([終了])

```

# 5. 処理詳細

処理フローの各処理で行う内容を定義する。

## (1) 利用量・請求取得

利用者本人の対象月の利用量・請求見込み金額、課金契約状態、および請求履歴を取得して返す。該当が無い項目は 0 または空一覧を返す。

| MOD-ID | 処理名 |
|---|---|
| MOD-007 | 利用量・請求取得 |

| 引数項目 | 値 |
|---|---|
| ユーザーID | 認証済みユーザーID |
| 対象月 | リクエスト.対象月(未指定時は当月。Asia/Tokyo 基準) |
| ページ | リクエスト.ページ(未指定時は API-COM §5 の既定値) |
| 取得件数 | リクエスト.取得件数(未指定時は API-COM §5 の既定値) |

## (2) 処理結果

(1) 利用量・請求取得の結果をレスポンスとして返却する。請求履歴はページネーション適用済みの一覧と総件数を返す。

| 項目名 | データ型 | 値 | 説明 |
|---|---|---|---|
| 課金契約状態 | Integer | (1) 利用量・請求取得の結果 | 返却する課金契約状態 |
| 対象月 | String | (1) 利用量・請求取得の結果 | 返却する対象月 |
| 対象月利用時間分 | Integer | (1) 利用量・請求取得の結果 | 返却する対象月利用時間分 |
| 対象月請求見込み金額 | Integer | (1) 利用量・請求取得の結果 | 返却する対象月請求見込み金額 |
| 請求履歴 | Object[] | (1) 利用量・請求取得の結果(ページネーション適用済み) | 返却する請求履歴 |
| - 請求ID | Integer | (1) 利用量・請求取得の結果 | 返却する請求ID |
| - 請求対象月 | String | (1) 利用量・請求取得の結果 | 返却する請求対象月 |
| - 請求金額 | Integer | (1) 利用量・請求取得の結果 | 返却する請求金額 |
| - 請求ステータス | Integer | (1) 利用量・請求取得の結果 | 返却する請求ステータス |
| 請求履歴総件数 | Integer | (1) 利用量・請求取得の結果の請求履歴総件数 | 返却する請求履歴総件数 |

# 6. バリデーション

入力バリデーションの構文ルールを、成立条件(AND / OR の論理式)で定義する。成立条件を満たさない場合、エラー列のコードを返し、違反項目ごとに details[] へ {field=項目名, message=メッセージ列} を設定する。任意項目は「指定なし OR(指定あり AND 制約)」の形で表す。

| 項目名 | 成立条件 | エラー | メッセージ |
|---|---|---|---|
| 対象月 | 指定なし OR(指定あり AND string AND YYYY-MM形式) | ERR-006 | 対象月は YYYY-MM 形式で指定してください |
| ページ / 取得件数 | 指定なし OR(指定あり AND 1 ＜＝ 整数) | ERR-006 | ページ・取得件数は1以上の整数で指定してください |

# 7. エラー

認証・入力バリデーションで発生する共通エラーは API-COM_共通設計.md §4.1 共通エラー一覧を参照する。本 API に適用される共通エラーは ERR-001(認証失敗) / ERR-006(バリデーションエラー)。この API 固有のエラーはない。
