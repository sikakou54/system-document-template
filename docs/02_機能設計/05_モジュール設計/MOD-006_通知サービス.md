# 1. 基本情報

| 項目 | 内容 |
|---|---|
| モジュールID | MOD-006 |
| モジュール名 | 通知サービス |
| 種別 | Service |
| 概要 | 予約開始前のリマインド対象を抽出し、予約者へ Resend でメール送信して送信状態を更新する |

# 2. 責務

| No | 責務 |
|---|---|
| 1 | リマインド対象予約(開始前・未送信)の抽出 |
| 2 | 予約者へのリマインドメール送信(外部サービス Resend を利用) |
| 3 | 送信結果に応じた予約のリマインド送信状態の更新 |

# 3. インターフェース

## (1) リマインド送信処理

### 1. 概要

リマインド対象を抽出しメール送信、送信状態を更新する処理。

### 2. 入力

| 入力項目 | データ型 | 説明 |
|---|---|---|
| リマインド閾値分 | Integer | 現在時刻から何分先までに開始する予約をリマインド対象とするかの閾値(分) |

### 3. 出力

| 出力項目 | データ型 | 説明 |
|---|---|---|
| 送信結果 | Object | 送信済に更新した件数／失敗に更新した件数 |
| - 送信済件数 | Integer | 送信済に更新した件数 |
| - 失敗件数 | Integer | 失敗に更新した件数 |

### 4. 例外

| エラーID | 説明 |
|---|---|
| なし | 例外は送出しない(送信失敗・予約者取得不可は当該予約の リマインドステータス=3(失敗)に更新して次を処理) |

### 5. 処理フロー

```mermaid
flowchart TD
  Start([開始])
  P1["(1) リマインド対象抽出"]
  D1{"(2) 対象有無判定"}
  P2["(3) 予約者取得"]
  P3["(4) リマインドメール送信(Resend)"]
  P4["(5) 送信状態更新"]
  D2{"(6) 次対象判定"}
  End([終了])

  Start --> P1 --> D1
  D1 -- "対象なし" --> End
  D1 -- "対象あり" --> P2 --> P3 --> P4 --> D2
  D2 -- "次の対象あり" --> P2
  D2 -- "次の対象なし" --> End
```

### 6. 処理詳細

#### (1) リマインド対象取得処理

リマインド送信の対象となる、開始時刻が近い未送信の予約を抽出する。該当が無い場合は 0件を返す。

| SQL-ID | クエリ名 |
|---|---|
| SQL-024 | リマインド対象予約抽出 |

| 引数項目 | 値 |
|---|---|
| 基準時刻 | 現在時刻 |
| 閾値時刻 | 現在時刻 ＋ 引数.事前通知分数 |

#### (2) 対象有無判定処理

リマインド対象の予約が存在するかどうかを判定する。

条件定義:

| No | 判定対象 | 条件 |
|---|---|---|
| 条件 | (1) リマインド対象抽出の結果 | 件数 ＞ 0 |

条件分岐マトリクス:

| 条件・処理 | #1 対象あり | #2 対象なし |
|---|---|---|
| 条件 | ◯ | × |
| 処理 |  |  |
| (3) 予約者取得へ進む | ◯ | - |
| 終了する | - | ◯ |

| 項目名 | データ型 | 設定値 |
|---|---|---|
| なし | - | - |
#### (3) 予約者取得処理

メール送信のため、対象予約の予約者(氏名・メールアドレス)を取得する。該当が無い場合は NULL を返し、当該予約はメール送信をスキップして (5) で失敗として更新する。

| SQL-ID | クエリ名 |
|---|---|
| SQL-004 | ユーザー取得 |

| 引数項目 | 値 |
|---|---|
| ユーザーID | (1) リマインド対象抽出の結果.対象予約.ユーザーID |

#### (4) リマインドメール送信処理

予約者へ、外部サービス Resend を用いてリマインドメールを送信し、送信の成否を得る。

| MOD-ID | 処理名 |
|---|---|
| なし | - |

| 引数項目 | 値 |
|---|---|
| 宛先メールアドレス | (3) 予約者取得の結果.メールアドレス |
| 宛先ユーザー名 | (3) 予約者取得の結果.名称 |
| 対象予約 | (1) リマインド対象抽出の結果.対象予約(タイトル・会議室ID・開始日時) |

#### (5) 送信状態更新処理

メール送信の結果に応じ、対象予約のリマインド送信状態を更新する。1件ごとに更新をコミットし、1件の失敗が他の対象に波及しないようにする。

条件定義:

| No | 判定対象 | 条件 |
|---|---|---|
| 条件 | (4) リマインドメール送信の結果 | 送信成功 |

条件分岐マトリクス:

| 条件・処理 | #1 送信成功 | #2 送信失敗 |
|---|---|---|
| 条件 | ◯ | × |
| 処理 |  |  |
| リマインドステータス=2(送信済) に更新する | ◯ | - |
| リマインドステータス=3(失敗) に更新する | - | ◯ |

| 項目名 | データ型 | 設定値 |
|---|---|---|
| 対象予約(TBL-003) | - | 送信成功=2(送信済) / 送信失敗=3(失敗) |
#### (6) 次対象判定処理

現在処理中の予約を終えた後、まだ処理すべき予約が残っているか（ループを継続するか）を判定する。

条件定義:

| No | 判定対象 | 条件 |
|---|---|---|
| 条件 | (1) リマインド対象抽出の結果 | 未処理の対象予約が残っている |

条件分岐マトリクス:

| 条件・処理 | #1 次あり | #2 次なし |
|---|---|---|
| 条件 | ◯ | × |
| 処理 |  |  |
| (3) 予約者取得へ戻る | ◯ | - |
| 終了する | - | ◯ |

| 項目名 | データ型 | 設定値 |
|---|---|---|
| 送信結果 | Object | (5) でカウントした送信用変数の最終結果(送信済件数 / 失敗件数) |
| - 送信済件数 | Integer | 送信に成功(STATUS=2)した件数 |
| - 失敗件数 | Integer | 送信に失敗(STATUS=3)した件数 |
# 4. トランザクション・排他制御

| 項目 | 内容 |
|---|---|
| トランザクション境界 | 対象予約1件ごとに (5) 送信状態更新のみを短いトランザクションでコミットする(全体を1トランザクションにしない) |
| 排他制御 | なし |

| テーブル | C | R | U | D | 用途 |
|---|---|---|---|---|---|
| TBL-003 |  | ✓ | ✓ |  | リマインド対象の抽出・送信状態の更新 |
| TBL-001 |  | ✓ |  |  | 予約者の氏名・メールアドレスの取得 |

# 6. エラー・例外

| 条件 | エラー | 対応 |
|---|---|---|
| メール送信失敗・予約者取得不可 | - | 例外を送出せず、当該予約の リマインドステータス を 3(失敗) に更新して次の対象を処理する |
