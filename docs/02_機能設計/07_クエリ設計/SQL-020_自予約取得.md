# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-020 |
| クエリ名 | 自予約取得 |
| 概要 | 予約ID・ユーザーID一致かつ論理削除されていない自予約1件を、会議室名付きで取得する。他者の予約・不存在は 0件。 |
| 用途 | 自予約の取得、および予約変更・キャンセルの前提取得(予約者本人・状態の判定)。 |
| 使用元 | MOD-003 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-003 | T_RESERVATIONS | 抽出対象(自予約の取得) |
| TBL-002 | M_ROOMS | 会議室名(NAME)の取得 |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :id | 予約ID | INTEGER | Yes | 取得対象の予約 ID |
| :user_id | ユーザーID | INTEGER | Yes | 予約者本人の判定に使用 |

# 4. クエリ

予約ID(ID)一致・ユーザーID(USER_ID)一致・有効(DELETED_AT IS NULL)の予約を、ROOM_ID で M_ROOMS に結合して会議室名付きで1件取得する。予約者本人でない、または不存在の場合は 0件。

```sql
SELECT
  r.ID            AS ID,
  r.ROOM_ID       AS ROOM_ID,
  m.NAME          AS ROOM_NAME,
  r.USER_ID       AS USER_ID,
  r.TITLE         AS TITLE,
  r.START_AT      AS START_AT,
  r.END_AT        AS END_AT,
  r.STATUS        AS STATUS,
  r.REMIND_STATUS AS REMIND_STATUS
FROM T_RESERVATIONS r
INNER JOIN M_ROOMS m
  ON m.ID = r.ROOM_ID
WHERE r.ID = :id
  AND r.USER_ID = :user_id
  AND r.DELETED_AT IS NULL
LIMIT 1;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 予約ID | INTEGER | 予約の ID |
| ROOM_ID | 会議室ID | INTEGER | 予約対象の会議室ID |
| ROOM_NAME | 会議室名 | TEXT | 会議室名 |
| USER_ID | ユーザーID | INTEGER | 予約者の ID |
| TITLE | 予約タイトル | TEXT | 予約タイトル |
| START_AT | 利用開始日時 | TEXT | ISO8601(UTC) |
| END_AT | 利用終了日時 | TEXT | ISO8601(UTC) |
| STATUS | 予約ステータス | INTEGER | DEF-001/CODE-004 |
| REMIND_STATUS | リマインド通知ステータス | INTEGER | DEF-001/CODE-005 |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | 主キー(ID)による一意検索。USER_ID で本人限定 |
| 想定件数 | 最大1件 |
| 注意点 | ID+USER_ID で予約者本人の予約に限定する(他者の予約・不存在は 0件で、呼び出し側は NULL 相当として扱う)。変更・キャンセルの前提(状態・未開始)判定は呼び出しモジュール(MOD-003)で行う |
