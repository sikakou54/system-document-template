# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-024 |
| クエリ名 | リマインド対象予約抽出 |
| 概要 | 予約済(STATUS=1)・リマインド未送信(REMIND_STATUS=1)で、開始時刻が基準時刻から閾値以内の予約を抽出する。 |
| 用途 | リマインド送信(NotificationService)の対象予約抽出。 |
| 使用元 | MOD-006 / JOB-001 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-003 | T_RESERVATIONS | 抽出対象(リマインド対象予約) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :now | 基準時刻 | TEXT | Yes | ISO8601(UTC)。抽出の基準となる現在時刻 |
| :until | 閾値時刻 | TEXT | Yes | ISO8601(UTC)。基準時刻＋リマインド閾値分(呼び出し側で算出) |

# 4. クエリ

予約済(STATUS=1)・リマインド未送信(REMIND_STATUS=1)・有効(DELETED_AT IS NULL)で、開始時刻が `:now ＜＝ START_AT ＜＝ :until` の範囲にある予約を抽出する。利用開始日時昇順で並べる。

```sql
SELECT
  r.ID       AS ID,
  r.ROOM_ID  AS ROOM_ID,
  r.USER_ID  AS USER_ID,
  r.TITLE    AS TITLE,
  r.START_AT AS START_AT,
  r.END_AT   AS END_AT
FROM T_RESERVATIONS r
WHERE r.STATUS = 1                        -- 予約済(TBL-003/ENM-1)
  AND r.REMIND_STATUS = 1                 -- 未送信(TBL-003/ENM-2)
  AND r.DELETED_AT IS NULL
  AND r.START_AT >= :now
  AND r.START_AT <= :until
ORDER BY r.START_AT ASC, r.ID ASC;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 予約ID | INTEGER | 予約の ID |
| ROOM_ID | 会議室ID | INTEGER | 予約対象の会議室ID |
| USER_ID | ユーザーID | INTEGER | 予約者の ID(予約者取得に使用) |
| TITLE | 予約タイトル | TEXT | 予約タイトル(メール本文に使用) |
| START_AT | 利用開始日時 | TEXT | ISO8601(UTC) |
| END_AT | 利用終了日時 | TEXT | ISO8601(UTC) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | TBL-003/IDX-3(IDX_RESERVATIONS_REMIND: REMIND_STATUS, START_AT) |
| 想定件数 | 実行間隔・閾値分に依存(通常少数) |
| 注意点 | :until = 基準時刻 + リマインド閾値分(leadMinutes)は呼び出し側で算出する。抽出後、予約者取得(SQL-004)・メール送信・送信状態更新(SQL-028)を1件ずつ処理する |
