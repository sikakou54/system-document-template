# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-004 |
| クエリ名 | ユーザー取得 |
| 概要 | ユーザーID一致かつ論理削除されていない有効なユーザーを1件取得する。該当が無い場合は 0件。 |
| 用途 | 予約者情報の取得(通知)、課金契約状態・Stripe識別子の取得(課金)。 |
| 使用元 | MOD-006 / MOD-007 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-001 | M_USERS | 抽出対象(IDによるユーザー取得) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :id | ユーザーID | INTEGER | Yes | 取得対象ユーザーの ID(主キー) |

# 4. クエリ

ユーザーID(ID)一致かつ有効(DELETED_AT IS NULL)のユーザーを1件取得する。予約者の氏名・メール、課金契約状態、Stripe 識別子を返す。

```sql
SELECT
  u.ID                     AS ID,
  u.NAME                   AS NAME,
  u.EMAIL                  AS EMAIL,
  u.ROLE                   AS ROLE,
  u.STRIPE_CUSTOMER_ID     AS STRIPE_CUSTOMER_ID,
  u.STRIPE_SUBSCRIPTION_ID AS STRIPE_SUBSCRIPTION_ID,
  u.BILLING_STATUS         AS BILLING_STATUS
FROM M_USERS u
WHERE u.ID = :id
  AND u.DELETED_AT IS NULL
LIMIT 1;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | ユーザーID | INTEGER | ユーザーの ID |
| NAME | ユーザー名 | TEXT | ユーザー名 |
| EMAIL | メールアドレス | TEXT | メールアドレス |
| ROLE | ユーザーロール | INTEGER | 1=一般 / 2=管理者(TBL-001/ENM-1) |
| STRIPE_CUSTOMER_ID | Stripe顧客ID | TEXT | Stripe Customer。未登録時 NULL |
| STRIPE_SUBSCRIPTION_ID | StripeサブスクリプションID | TEXT | 従量課金サブスク。未契約時 NULL |
| BILLING_STATUS | 課金契約状態 | INTEGER | 1=未契約 / 2=有効 / 3=停止(TBL-001/ENM-2) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | 主キー(ID)による一意検索 |
| 想定件数 | 主キー一致のため最大1件 |
| 注意点 | 呼び出し元は必要な列のみ参照する(通知は NAME・EMAIL、課金は BILLING_STATUS・STRIPE_CUSTOMER_ID・STRIPE_SUBSCRIPTION_ID) |
