# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-014 |
| クエリ名 | 設備存在確認 |
| 概要 | 指定した設備IDリストのうち、有効な設備マスタに存在する件数を返す。 |
| 用途 | 会議室登録・編集時の設備ID妥当性確認(全IDが存在するかの検証)。 |
| 使用元 | MOD-004 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-004 | M_EQUIPMENTS | 存在確認対象(設備IDの実在判定) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :equipment_ids | 設備IDリスト | INTEGER | Yes | 確認対象の設備ID。IN 句のプレースホルダはアプリ側で件数分展開する |

# 4. クエリ

指定した設備IDリストのうち、有効(DELETED_AT IS NULL)な設備の件数を数える。呼び出し側で「返却件数 = 指定件数」なら全IDが存在すると判定する。

```sql
SELECT
  COUNT(DISTINCT e.ID) AS FOUND_COUNT
FROM M_EQUIPMENTS e
WHERE e.ID IN (:equipment_ids)
  AND e.DELETED_AT IS NULL;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| FOUND_COUNT | 存在件数 | INTEGER | 指定した設備IDのうち実在する件数(重複排除) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | 主キー(ID)による IN 検索 |
| 想定件数 | 1件(件数を1行で返す) |
| 注意点 | :equipment_ids はアプリ側でプレースホルダを件数分展開する。空リストのときは呼び出し側で本クエリを実行しない(設備条件なし)。FOUND_COUNT が指定件数と一致しなければ不正な設備IDが含まれると判定する |
