# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-028 |
| クエリ名 | 予約リマインド送信状態更新 |
| 概要 | 予約1件のリマインド通知ステータス(REMIND_STATUS)を更新する。 |
| 用途 | リマインド送信後の送信状態更新(送信済 / 失敗)。 |
| 使用元 | MOD-006 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-003 | T_RESERVATIONS | 更新対象(リマインド通知ステータスの UPDATE) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :id | 予約ID | INTEGER | Yes | 更新対象の予約 ID |
| :remind_status | リマインド通知ステータス | INTEGER | Yes | DEF-001/CODE-005 |
| :updated_at | 更新日時 | TEXT | Yes | ISO8601(UTC)。アプリで現在時刻を設定(共通カラム.md) |

# 4. クエリ

有効(DELETED_AT IS NULL)な予約の REMIND_STATUS を更新する。1件ごとに更新する。

```sql
UPDATE T_RESERVATIONS
SET REMIND_STATUS = :remind_status,       -- DEF-001/CODE-005
    UPDATED_AT    = :updated_at
WHERE ID = :id
  AND DELETED_AT IS NULL
RETURNING ID, REMIND_STATUS;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 予約ID | INTEGER | 予約の ID |
| REMIND_STATUS | リマインド通知ステータス | INTEGER | 更新後のリマインド通知ステータス(DEF-001/CODE-005) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | 主キー(ID)による一意更新 |
| 想定件数 | 最大1件(対象予約ごとに実行) |
| 注意点 | 対象予約1件ごとに短いトランザクションでコミットする(1件の失敗が他に波及しないようにする。MOD-006 §6)。送信結果に応じた DEF-001/CODE-005。UPDATED_AT はアプリで現在時刻を設定する |
