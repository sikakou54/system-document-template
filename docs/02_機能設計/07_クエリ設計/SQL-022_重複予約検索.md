# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-022 |
| クエリ名 | 重複予約検索 |
| 概要 | 指定会議室・時間帯に重複するDEF-001/SET-005の予約を検索する。変更時は対象予約自身を除外する。 |
| 用途 | 予約登録・変更時の時間帯重複(二重予約)判定。 |
| 使用元 | MOD-003 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-003 | T_RESERVATIONS | 重複判定(予約済の時間帯重複) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :room_id | 会議室ID | INTEGER | Yes | 判定対象の会議室ID |
| :start_at | 利用開始日時 | TEXT | Yes | ISO8601(UTC)。判定する利用開始日時 |
| :end_at | 利用終了日時 | TEXT | Yes | ISO8601(UTC)。判定する利用終了日時 |
| :exclude_reservation_id | 除外予約ID | INTEGER | No | 変更時に自予約を除外する ID。登録時は NULL(除外なし) |

# 4. クエリ

同一会議室(ROOM_ID)でDEF-001/SET-005の有効な予約が `既存予約.START_AT ＜ :end_at AND 既存予約.END_AT ＞ :start_at` を満たすとき時間帯が重複するとみなす。変更時は :exclude_reservation_id で対象予約自身を除外する(登録時は NULL)。

```sql
SELECT
  r.ID AS ID
FROM T_RESERVATIONS r
WHERE r.ROOM_ID = :room_id
  AND r.STATUS = 1                       -- 重複対象となる予約ステータス(DEF-001/CODE-004)のみ対象
  AND r.DELETED_AT IS NULL
  AND r.START_AT < :end_at
  AND r.END_AT   > :start_at
  AND (:exclude_reservation_id IS NULL OR r.ID != :exclude_reservation_id);
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 予約ID | INTEGER | 時間帯が重複する予約の ID(件数で重複有無を判定する) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | TBL-003/IDX-1(IDX_RESERVATIONS_ROOM_TIME: ROOM_ID, START_AT, END_AT) |
| 想定件数 | 通常0件(重複なし)。重複時のみ該当行を返す |
| 注意点 | DEF-001/SET-005のみ重複対象。日時は TEXT(ISO8601, UTC)の辞書順比較で判定するため保存形式を統一する。呼び出し側は返却件数=0 を重複なしと判定する。変更時は自予約を :exclude_reservation_id で除外する |
