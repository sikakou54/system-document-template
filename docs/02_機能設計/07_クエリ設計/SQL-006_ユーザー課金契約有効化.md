# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-006 |
| クエリ名 | ユーザー課金契約有効化 |
| 概要 | Stripe顧客ID一致のユーザーの課金契約を DEF-001/SET-001 にし、サブスクリプションIDを保存する。 |
| 用途 | Webhook(checkout.session.completed)による契約有効化。 |
| 使用元 | MOD-007 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-001 | M_USERS | 更新対象(課金契約状態・サブスクリプションIDの反映) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :stripe_customer_id | Stripe顧客ID | TEXT | Yes | 対象ユーザーの Stripe Customer ID |
| :stripe_subscription_id | StripeサブスクリプションID | TEXT | Yes | 有効化した従量課金サブスクの ID |
| :updated_at | 更新日時 | TEXT | Yes | ISO8601(UTC)。アプリで現在時刻を設定(共通カラム.md) |

# 4. クエリ

Stripe顧客ID(STRIPE_CUSTOMER_ID)一致かつ有効(DELETED_AT IS NULL)のユーザーの BILLING_STATUS を 2(有効)に更新し、STRIPE_SUBSCRIPTION_ID を保存する。

```sql
UPDATE M_USERS
SET BILLING_STATUS         = 2,                       -- DEF-001/CODE-002
    STRIPE_SUBSCRIPTION_ID = :stripe_subscription_id,
    UPDATED_AT             = :updated_at
WHERE STRIPE_CUSTOMER_ID = :stripe_customer_id
  AND DELETED_AT IS NULL
RETURNING ID, BILLING_STATUS, STRIPE_SUBSCRIPTION_ID;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | ユーザーID | INTEGER | 更新したユーザーの ID |
| BILLING_STATUS | 課金契約状態 | INTEGER | DEF-001/CODE-002 |
| STRIPE_SUBSCRIPTION_ID | StripeサブスクリプションID | TEXT | 保存したサブスクリプションID |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | STRIPE_CUSTOMER_ID による絞り込み(Stripe 側で顧客一意) |
| 想定件数 | 最大1件 |
| 注意点 | Webhook は冪等に処理する(同一顧客の再送でも同じ結果に収束)。BILLING_STATUS の区分値は DEF-001/CODE-002 |
