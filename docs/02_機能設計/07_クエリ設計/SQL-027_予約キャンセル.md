# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-027 |
| クエリ名 | 予約キャンセル |
| 概要 | 予約1件のステータスをキャンセル(STATUS=2)に更新し、更新後の予約を返す。 |
| 用途 | 予約のキャンセル(ReservationService)。 |
| 使用元 | MOD-003 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-003 | T_RESERVATIONS | 更新対象(予約ステータスの UPDATE) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :id | 予約ID | INTEGER | Yes | キャンセル対象の予約 ID |
| :updated_at | 更新日時 | TEXT | Yes | ISO8601(UTC)。アプリで現在時刻を設定(共通カラム.md) |

# 4. クエリ

有効(DELETED_AT IS NULL)な予約の STATUS を 2(キャンセル)に更新する。更新後の予約を返す。

```sql
UPDATE T_RESERVATIONS
SET STATUS     = 2,                       -- キャンセル(TBL-003/ENM-1)
    UPDATED_AT = :updated_at
WHERE ID = :id
  AND DELETED_AT IS NULL
RETURNING ID, ROOM_ID, USER_ID, TITLE, START_AT, END_AT, STATUS, REMIND_STATUS;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| ID | 予約ID | INTEGER | 予約の ID |
| ROOM_ID | 会議室ID | INTEGER | 予約対象の会議室ID |
| USER_ID | ユーザーID | INTEGER | 予約者の ID |
| TITLE | 予約タイトル | TEXT | 予約タイトル |
| START_AT | 利用開始日時 | TEXT | ISO8601(UTC) |
| END_AT | 利用終了日時 | TEXT | ISO8601(UTC) |
| STATUS | 予約ステータス | INTEGER | 2=キャンセル(TBL-003/ENM-1) |
| REMIND_STATUS | リマインド通知ステータス | INTEGER | 1=未送信 / 2=送信済 / 3=失敗(TBL-003/ENM-2) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | 主キー(ID)による一意更新 |
| 想定件数 | 最大1件 |
| 注意点 | 予約者本人・キャンセル可能状態(予約済・未開始)の判定は呼び出しモジュール(MOD-003)で事前に行う。UPDATED_AT はアプリで現在時刻を設定する |
