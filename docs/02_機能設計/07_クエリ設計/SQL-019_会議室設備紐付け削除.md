# 1. 基本情報

| 項目 | 内容 |
|---|---|
| クエリID | SQL-019 |
| クエリ名 | 会議室設備紐付け削除 |
| 概要 | 指定会議室の設備紐付けを全件物理削除する(付け替え用)。 |
| 用途 | 会議室編集時の設備紐付け付け替え(既存削除 → 再登録)。 |
| 使用元 | MOD-004 |

# 2. 対象テーブル

| テーブル | 物理名 | 用途 |
|---|---|---|
| TBL-005 | M_ROOM_EQUIPMENTS | 削除対象(会議室の設備紐付け) |

# 3. 入力パラメータ

| パラメータ | 論理名 | 型 | 必須 | 説明・制約 |
|---|---|---|---|---|
| :room_id | 会議室ID | INTEGER | Yes | 設備紐付けを削除する会議室の ID |

# 4. クエリ

指定会議室(ROOM_ID)の設備紐付けを物理削除する。付け替えのため物理削除とし、直後に SQL-018 で登録し直す。

```sql
DELETE FROM M_ROOM_EQUIPMENTS
WHERE ROOM_ID = :room_id;
```

# 5. 出力(列)

| 列名 | 論理名 | 型 | 説明 |
|---|---|---|---|
| なし | - | - | 戻り値なし(削除のみ) |

# 6. 補足(性能・インデックス)

| 項目 | 内容 |
|---|---|
| 利用インデックス | UX_ROOM_EQUIPMENTS_ROOM_EQUIP(ROOM_ID を先頭に含む) |
| 想定件数 | 1会議室あたり数件程度 |
| 注意点 | 中間テーブルの付け替えは物理削除で行う(論理削除だと UX_ROOM_EQUIPMENTS_ROOM_EQUIP により同一組み合わせの再登録が制約違反となるため)。削除→再登録(SQL-018)は同一トランザクション内で行う(MOD-004 §6) |
