## 1. プロダクト概要

MeetRoom は、社内の会議室を探す・予約する・利用状況を把握する業務を、ひとつの流れで扱えるようにする会議室予約プロダクトである。
会議室の空き状況が分かりにくい、予約が重なる、利用実態が見えないといった問題を減らし、社員と管理部門の双方が会議室を扱いやすい状態をつくる。

| 項目 | 内容 |
|---|---|
| プロダクト名 | MeetRoom |
| 種別 | 社内向け会議室予約プロダクト |
| 主な利用者 | 会議室を利用する社員、会議室を管理する総務・管理部門 |
| 一言説明 | 社内会議室の空き状況、予約、利用状況を分かりやすく管理するプロダクト |
| 技術スタック | Cloudflare(Pages / Workers / D1 / KV / Queues / Cron Triggers) ＋ Stripe(従量課金) ＋ Resend(メール送信) |
| 課金モデル | 有料会議室の利用時間に応じた従量課金(月次請求) |

## 2. 現在の課題

会議室の利用・管理において、現在発生している課題を定義する。

| 課題 | 課題の内容 | 影響を受ける人 |
|---|---|---|
| 空き状況が分かりにくい | 会議室ごとの空き時間、収容人数、設備を確認する手間が大きい | 社員 |
| 予約が重なる | 同じ会議室に同じ時間帯の予約が入り、会議直前の調整が発生する | 社員、総務・管理部門 |
| 管理が人に依存する | 会議室や設備の情報が担当者の把握に依存し、問い合わせが増える | 総務・管理部門 |
| 利用実態が見えない | 会議室の利用頻度や偏りが分からず、改善判断がしづらい | 総務・管理部門、組織の管理者 |

## 3. どう解決するか

MeetRoom が現在の課題をどのように解決するかを定義する。

| 解決方法 | 内容 | 対応する課題 |
|---|---|---|
| 空き会議室を検索できるようにする | 日時、人数、設備などの条件から、利用できる会議室を探せるようにする | 空き状況が分かりにくい |
| 重複予約を防ぐ | 同じ会議室に同じ時間帯の予約が入らないようにする | 予約が重なる |
| 会議室と設備の情報を管理する | 会議室、収容人数、設備などの情報を一元管理する | 管理が人に依存する |
| 利用状況を見える化する | 会議室ごとの利用実績を確認できるようにする | 利用実態が見えない |
| 必要な通知を行う | 予約開始前にリマインド通知を行い、利用忘れを減らす | 会議室の利用に関する確認漏れ |

## 4. 解決した結果どのようになるか

MeetRoom によって課題が解決された結果、利用者と組織がどのような状態になるかを定義する。

| 対象 | 期待する状態 |
|---|---|
| 社員 | 条件に合う会議室を短時間で見つけ、安心して予約できる |
| 総務・管理部門 | 会議室や設備の情報を管理しやすくなり、問い合わせや個別調整が減る |
| 組織の管理者 | 利用状況をもとに、会議室数や設備の見直しを判断できる |
| 組織全体 | 会議室に関する確認・調整の時間が減り、会議準備に集中できる |

## 5. このドキュメント群の目的

対象システム「MeetRoom」(会議室予約システム)の要件定義・機能設計ドキュメント一式である。
以下の開発プロセスのうち「要件定義」「機能設計」の成果物を docs/ 配下で管理する。

```text
要件定義 ＞ 機能設計 ＞ コーディング ＞ テスト ＞ リリース
```
