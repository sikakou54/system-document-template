# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | BR-003 |
| 要件名 | 予約を見直せるようにする |
| 概要 | 予約者が自分の予約を後から変更・取消できるようにする |
| 実現したい状態 | 予約者が自分の予約を変更・取消できる |
| 背景 | 予定変更や中止があっても予約を直せず、使われない予約が会議室を占有している |

# 2. 業務要件

この業務要件で実現したいことを定義する。

| No | 実現したいこと |
|---|---|
| 1 | 予約者が自分の予約の日時・タイトルを変更できるようにする |
| 2 | 予約者が自分の予約を取り消せるようにする |
| 3 | 変更後も同一会議室・同一時間帯の二重予約が発生しないようにする |
| 4 | 他人の予約は変更・取消できないようにする |

# 3. 成功指標・期待効果

この業務要件の達成度を測る指標と期待効果を定義する。

| No | 区分 | 内容 |
|---|---|---|
| 1 | 成功指標 | 予定変更・中止時に予約者自身が予約を修正・取消できる |
| 2 | 期待効果 | 使われない予約が減り、会議室が他の利用者に開放される |
| 3 | 期待効果 | 予約の管理者への修正依頼が不要になり運用負荷が下がる |

# 4. 関連(実現する機能要件)

この業務要件を実現する機能要件(FR)・共通機能要件(CFR)を定義する。

| ID | 名称 | 実現内容 |
|---|---|---|
| FR-003 | 予約変更・キャンセル | 予約者が自分の予約の日時・タイトル変更、またはキャンセルを行う |

# 5. 前提・制約

この業務要件の前提と制約を定義する。

| No | 区分 | 内容 |
|---|---|---|
| 1 | 前提 | 変更・取消の対象となる予約が登録されている |
| 2 | 制約 | 変更・取消できるのは自分の予約に限る |
| 3 | 制約 | 開始済み・キャンセル済みの予約は変更・取消の対象外とする |
