# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | BR-006 |
| 要件名 | 利用状況を見える化する |
| 概要 | 管理者が会議室ごとの利用実績(件数・利用時間)を月単位で把握できるようにする |
| 実現したい状態 | 管理者が会議室ごとの利用実績を把握できる |
| 背景 | 会議室ごとの利用状況が把握できず、増設・統廃合や運用改善の判断材料が不足している |

# 2. 業務要件

この業務要件で実現したいことを定義する。

| No | 実現したいこと |
|---|---|
| 1 | 管理者が月を指定して会議室ごとの利用実績を確認できるようにする |
| 2 | 会議室ごとの利用件数・利用時間を把握できるようにする |
| 3 | 集計対象は実際に利用が完了した予約とする |

# 3. 成功指標・期待効果

この業務要件の達成度を測る指標と期待効果を定義する。

| No | 区分 | 内容 |
|---|---|---|
| 1 | 成功指標 | 指定した月の会議室別の利用件数・利用時間を確認できる |
| 2 | 期待効果 | 利用の多い・少ない会議室が把握でき、増設・統廃合の判断に活用できる |
| 3 | 期待効果 | 利用実績に基づき会議室運用を継続的に改善できる |

# 4. 関連(実現する機能要件)

この業務要件を実現する機能要件(FR)・共通機能要件(CFR)を定義する。

| ID | 名称 | 実現内容 |
|---|---|---|
| FR-006 | 利用実績レポート | 管理者が会議室ごとの月次利用実績(件数・利用時間)を確認する |

# 5. 前提・制約

この業務要件の前提と制約を定義する。

| No | 区分 | 内容 |
|---|---|---|
| 1 | 前提 | 操作する利用者が管理者である |
| 2 | 前提 | 集計対象となる予約の利用実績が蓄積されている |
| 3 | 制約 | 集計対象は利用が完了した予約に限る |
