# 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | BR-007 |
| 要件名 | 登録者だけが使える |
| 概要 | 登録された利用者のみが本人確認を経てシステムを利用できるようにする |
| 実現したい状態 | 登録利用者のみが本人確認の上で利用する |
| 背景 | 誰でも利用できる状態では予約者の特定や不正利用の防止ができない |

# 2. 業務要件

この業務要件で実現したいことを定義する。

| No | 実現したいこと |
|---|---|
| 1 | 登録された利用者のみがシステムを利用できるようにする |
| 2 | 利用者が本人確認を経て利用可能な状態になるようにする |
| 3 | 各操作を行った利用者を識別できるようにする |

# 3. 成功指標・期待効果

この業務要件の達成度を測る指標と期待効果を定義する。

| No | 区分 | 内容 |
|---|---|---|
| 1 | 成功指標 | 未登録・本人確認未完了の状態では機能を利用できない |
| 2 | 期待効果 | 予約者・操作者が特定でき、不正利用や無責任な利用を抑止できる |
| 3 | 期待効果 | 利用者ごとの権限管理・課金の前提が整う |

# 4. 関連(実現する機能要件)

この業務要件を実現する機能要件(FR)・共通機能要件(CFR)を定義する。

| ID | 名称 | 実現内容 |
|---|---|---|
| CFR-001 | 認証 | 登録利用者が本人確認(ログイン)を経てシステムを利用可能な状態にする |

# 5. 前提・制約

この業務要件の前提と制約を定義する。

| No | 区分 | 内容 |
|---|---|---|
| 1 | 前提 | 利用者があらかじめ登録されている |
| 2 | 制約 | 認証方式などの実現手段は本業務要件では扱わない(CFR-001 で定義) |
