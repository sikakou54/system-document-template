## 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | FR-003 |
| 要件名 | 予約変更・キャンセル |
| 概要 | 自分の予約を変更・キャンセルできる |
| 目的 | 予定変更・中止に応じて予約者自身が予約を見直せるようにし、使われない予約を減らす |
| 対象ロール | 利用者 / 管理者 |

## 2. 機能要件

利用者の操作とシステムの処理を定義する。

| 項目 | 内容 |
|---|---|
| 利用者の操作 | 自分の予約を選び、日時・タイトルを変更、または予約をキャンセルする |
| システムの処理 | 予約者本人の予約に限り変更・キャンセルを反映する。変更時も同一会議室・同一時間帯の二重予約を防止する |

## 3. 入力情報

この機能が処理のために必要とする入力情報を業務レベルで定義する。

| 情報 | 要否 | 内容 |
|---|---|---|
| 対象の予約 | 必須 | 変更またはキャンセルする自分の予約 |
| 処理区分 | 必須 | 変更かキャンセルかの区分 |
| 変更後の利用開始日時 | 任意 | 変更時に指定する新しい利用開始日時 |
| 変更後の利用終了日時 | 任意 | 変更時に指定する新しい利用終了日時 |
| 変更後の予約タイトル | 任意 | 変更時に指定する新しい予約の用件・名称 |

## 4. 出力情報

この機能が結果として提供する出力情報を業務レベルで定義する。

| 情報 | 内容 |
|---|---|
| 変更後の予約内容 | 変更が反映された予約(会議室・利用日時・タイトル) |
| 処理結果 | 変更またはキャンセルが成立したことが分かる情報 |

## 5. 業務ルール

この機能の判断基準となる業務上のルールを定義する。

| No | ルール |
|---|---|
| 1 | 変更・キャンセルできるのは予約者本人の予約に限る(RULE-COM-002) |
| 2 | 変更後も同一会議室・同一時間帯で重複する予約は登録できない |
| 3 | 利用開始日時を過ぎた(開始済みの)予約は変更・キャンセルできない |
| 4 | キャンセル済みの予約は変更・キャンセルできない |
| 5 | 変更後の予約は開始時刻＜終了時刻であること |
| 6 | 過去日時への変更はできない |

## 6. 受入条件

この要件を満たしていることを確認する条件を定義する。

| No | 条件 |
|---|---|
| 1 | 自分の予約を選び日時・タイトルを変更すると変更が反映される |
| 2 | 自分の予約をキャンセルするとキャンセル済みになる |
| 3 | 他人の予約は変更・キャンセルできない |
| 4 | 変更後の日時が既存予約と重複する場合は変更されずエラーになる |
| 5 | 開始済み・キャンセル済みの予約は変更・キャンセルできない |

## 8. 前提条件

この要件が成り立つために、先に満たしておく条件を定義する。

| No | 条件 |
|---|---|
| 1 | 利用者がログイン済みである |
| 2 | 変更・キャンセルの対象となる予約が登録済みである |

## 9. 制限事項

この要件での制限や上限などを定義する。

| No | 制限 |
|---|---|
| 1 | 変更できるのは予約の日時・タイトルに限り、予約する会議室そのものの変更は扱わない |
| 2 | 予約の新規登録はこの要件では扱わない(FR-002) |
