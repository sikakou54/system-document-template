## 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | FR-008 |
| 要件名 | 会議室利用の従量課金 |
| 概要 | 有料会議室の利用時間に応じて従量課金し、月次で請求する。利用者は事前に支払い方法を登録する |
| 目的 | 有料会議室の費用を利用量に応じて公平に負担できるようにする |
| 対象ロール | 利用者 / 管理者 |

## 2. 機能要件

利用者の操作とシステムの処理を定義する。

| 項目 | 内容 |
|---|---|
| 利用者の操作 | 利用者が支払い方法を登録し、自分の利用量・請求を確認する |
| システムの処理 | 有料会議室の予約時に支払い方法の登録を要求する。利用完了時に利用時間に応じた利用量を計上し、月次でまとめて請求する |

## 3. 入力情報

この機能が処理のために必要とする入力情報を業務レベルで定義する。

| 情報 | 要否 | 内容 |
|---|---|---|
| 支払い方法 | 必須 | 有料会議室を利用するために登録する支払い方法 |
| 確認対象月 | 任意 | 利用量・請求を確認する対象の月 |

## 4. 出力情報

この機能が結果として提供する出力情報を業務レベルで定義する。

| 情報 | 内容 |
|---|---|
| 支払い方法の登録結果 | 支払い方法が登録され、有料会議室を利用可能になったことが分かる情報 |
| 当月の利用量 | 当月に計上された有料会議室の利用時間 |
| 請求見込み金額 | 当月の利用量に基づく請求見込みの金額 |
| 請求履歴 | 月次の請求金額と請求状態 |

## 5. 業務ルール

この機能の判断基準となる業務上のルールを定義する。

| No | ルール |
|---|---|
| 1 | 課金対象は有料(利用単価が0より大きい)会議室の予約とする |
| 2 | 無料(利用単価0)会議室は課金対象外とする |
| 3 | 利用が完了した予約に対して、利用時間に応じた利用量を計上する(従量課金) |
| 4 | 適用する単価は利用時点の会議室の利用単価とする |
| 5 | 有料会議室の予約には支払い方法の登録が必要とし、未登録の場合は予約できない |
| 6 | 計上した利用量は月次でまとめて請求する |

## 6. 受入条件

この要件を満たしていることを確認する条件を定義する。

| No | 条件 |
|---|---|
| 1 | 利用者が支払い方法を登録すると有料会議室を予約できるようになる |
| 2 | 支払い方法未登録の利用者は有料会議室を予約できない |
| 3 | 有料会議室の利用が完了すると利用時間に応じた利用量が計上される |
| 4 | 無料会議室の利用は課金されない |
| 5 | 利用者が自分の当月の利用量・請求見込みと請求履歴を確認できる |

## 8. 前提条件

この要件が成り立つために、先に満たしておく条件を定義する。

| No | 条件 |
|---|---|
| 1 | 利用者がログイン済みである |
| 2 | 会議室に利用単価が設定されている |

## 9. 制限事項

この要件での制限や上限などを定義する。

| No | 制限 |
|---|---|
| 1 | 課金対象は有料会議室の利用が完了した予約に限る |
| 2 | 課金・決済・請求の具体的な実現方式(決済サービス等)は機能設計で定義し、本要件では扱わない |
