## 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | FR-002 |
| 要件名 | 会議室予約 |
| 概要 | 空き会議室を予約できる。同一会議室・同一時間帯の二重予約は不可 |
| 目的 | 会議室のダブルブッキングを防ぎ、確実に会議室を確保できるようにする |
| 対象ロール | 利用者 |

## 2. 機能要件

利用者の操作とシステムの処理を定義する。

| 項目 | 内容 |
|---|---|
| 利用者の操作 | 空き会議室を選択して予約を登録する |
| システムの処理 | 予約を登録し、同一会議室・同一時間帯の二重予約(ダブルブッキング)を防止する |

## 3. 入力情報

この機能が処理のために必要とする入力情報を業務レベルで定義する。

| 情報 | 要否 | 内容 |
|---|---|---|
| 予約対象の会議室 | 必須 | 予約する会議室(検索結果で空きとして表示された会議室) |
| 利用開始日時 | 必須 | 利用を開始する日時 |
| 利用終了日時 | 必須 | 利用を終了する日時 |
| 予約タイトル | 必須 | 予約の用件・名称 |

## 4. 出力情報

この機能が結果として提供する出力情報を業務レベルで定義する。

| 情報 | 内容 |
|---|---|
| 予約内容 | 登録された予約(会議室・利用日時・タイトル) |
| 登録結果 | 予約が成立したことが分かる情報 |

## 5. 業務ルール

この機能の判断基準となる業務上のルールを定義する。

| No | ルール |
|---|---|
| 1 | 同一会議室で時間帯が重複する予約は登録できない |
| 2 | 過去日時の予約は登録できない |
| 3 | 予約は開始時刻＜終了時刻であること |

## 6. 受入条件

この要件を満たしていることを確認する条件を定義する。

| No | 条件 |
|---|---|
| 1 | 空き会議室を選んで予約すると予約が登録され、マイ予約一覧に表示される |
| 2 | 既存予約と時間帯が重複する場合は登録されずエラーになる |
| 3 | 過去日時を指定すると登録されずエラーになる |

この要件が成り立つために、先に満たしておく条件を定義する。

| No | 条件 |
|---|---|

この要件での制限や上限などを定義する。

| No | 制限 |
|---|---|
