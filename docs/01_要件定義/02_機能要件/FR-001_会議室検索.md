## 1. 基本情報

| 項目 | 内容 |
|---|---|
| 要件ID | FR-001 |
| 要件名 | 会議室検索 |
| 概要 | 日時・人数・設備を条件に空き会議室を検索できる |
| 目的 | 会議室の空き状況の不透明さを解消し、条件に合う会議室を迅速に見つけられるようにする |
| 対象ロール | 利用者 |

## 2. 機能要件

利用者の操作とシステムの処理を定義する。

| 項目 | 内容 |
|---|---|
| 利用者の操作 | 日時・人数・設備を条件に指定して、空き会議室を検索する |
| システムの処理 | 指定された条件に合致する空き会議室の一覧を提示する |

## 3. 入力情報

この機能が処理のために必要とする入力情報を業務レベルで定義する。

| 情報 | 要否 | 内容 |
|---|---|---|
| 利用開始日時 | 必須 | 会議室を使い始める日時 |
| 利用終了日時 | 必須 | 会議室を使い終える日時 |
| 利用人数 | 任意 | 会議の参加人数。収容人数による絞り込みに用いる |
| 設備 | 任意 | 必要とする設備。設備の有無による絞り込みに用いる |

## 4. 出力情報

この機能が結果として提供する出力情報を業務レベルで定義する。

| 情報 | 内容 |
|---|---|
| 会議室名 | 空き会議室の名称 |
| 収容人数 | 会議室に収容できる人数 |
| 場所 | 会議室の場所 |
| 設備 | 会議室が備えている設備 |
| 空き状況 | 指定した時間帯に利用できること |

## 5. 業務ルール

この機能の判断基準となる業務上のルールを定義する。

| No | ルール |
|---|---|
| 1 | 指定した時間帯に予約が入っていない会議室のみを検索結果に表示する |
| 2 | 指定人数が会議室の収容人数を超える会議室は検索結果に表示しない |
| 3 | 指定設備を持たない会議室は検索結果に表示しない |
| 4 | 開始日時が終了日時以降の場合は検索できない |
| 5 | 利用不可状態の会議室は検索結果に表示しない |

## 6. 受入条件

この要件を満たしていることを確認する条件を定義する。

| No | 条件 |
|---|---|
| 1 | 開始日時・終了日時を指定して検索すると、指定時間帯に空きがある会議室のみ表示される |
| 2 | 人数を指定して検索すると、指定人数を収容できる会議室のみ表示される |
| 3 | 設備を指定して検索すると、指定設備を持つ会議室のみ表示される |
| 4 | 条件に合致する空き会議室が存在しない場合は、結果が0件であることがわかる |
| 5 | 開始日時が終了日時以降の場合は、検索できないことがわかる |

## 8. 前提条件

この要件が成り立つために、先に満たしておく条件を定義する。

| No | 条件 |
|---|---|
| 1 | 利用者が会議室検索機能を利用できる権限を持っている |
| 2 | 会議室、設備、既存予約の情報が登録されている |

## 9. 制限事項

この要件での制限や上限などを定義する。

| No | 制限 |
|---|---|
| 1 | 予約登録はこの要件では扱わない |
| 2 | 会議室情報の登録・編集・削除はこの要件では扱わない |
| 3 | 設備情報の登録・編集・削除はこの要件では扱わない |
