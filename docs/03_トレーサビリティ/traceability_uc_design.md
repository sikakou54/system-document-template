# traceability_uc_design

# 1. 概要

業務ユースケース(UC)を実現する画面(SCR)・API の対応を追跡する。
画面SCR は各 SCR のトレース元(02_画面設計)、API は各 API のトレース元(03_API設計)からの転記で構成する。

# 2. 対応表

| UC ID | 画面SCR | API | 備考 |
|---|---|---|---|
| UC-001 空き会議室を検索する | SCR-002 | API-002, API-009 | API-009(設備一覧取得)は UC-005 と共有 |
| UC-002 会議室を予約する | SCR-002, SCR-003 | API-002, API-003 | SCR-002/API-002(検索)は UC-001 と共有 |
| UC-003 予約を変更する | SCR-004 | API-004, API-006 | - |
| UC-004 予約をキャンセルする | SCR-004 | API-005, API-006 | - |
| UC-005 会議室を登録・編集する | SCR-005 | API-007, API-009, API-013, API-014 | API-009(設備一覧取得)は UC-001 と共有 |
| UC-006 利用実績を確認する | SCR-006 | API-008 | - |
| UC-007 ログインする | SCR-001 | API-001 | - |
| UC-008 支払い方法を登録する | SCR-007 | API-010, API-011 | API-011(Stripe Webhook受信)は画面から直接呼ばない |
| UC-009 利用量・請求を確認する | SCR-007 | API-012 | - |

# 3. 更新ルール

| 更新契機 | 更新内容 |
|---|---|
| SCR の新規作成・トレース元変更 | 対応する UC 行の画面SCR を SCR のトレース元から転記する |
| API の新規作成・トレース元変更 | 対応する UC 行の API を API のトレース元から転記する |
| ID廃止 | 行は削除せず、備考に「廃止」と記す |

- 本対応表の更新は関連文書の作成・更新と同一作業内で行う(正本: docs/CLAUDE.md §6・§10-7)。
