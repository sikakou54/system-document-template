# traceability_br_fr_uc

## 1. 概要

業務要件(BR)を実現する機能要件(FR)・共通機能要件(CFR)と、それを利用シナリオ化した業務ユースケース(UC)の対応を追跡する。
BR→FR/CFR は各 BR のトレース先(01_業務要件)、FR/CFR→UC は各 UC のトレース元(04_業務ユースケース)からの転記で構成する。

## 2. 対応表

| BR ID | 実現FR・CFR | 関連UC | 備考 |
|---|---|---|---|
| BR-001 空き会議室を見つけやすくする | FR-001 | UC-001 | - |
| BR-002 重複予約をなくす | FR-002 | UC-002 | - |
| BR-003 予約を見直せるようにする | FR-003 | UC-003, UC-004 | - |
| BR-004 予約忘れを減らす | FR-004 | - | FR-004 はリマインドを JOB-001 で実現(トレース元 UC なし) |
| BR-005 会議室・設備・料金を一元管理する | FR-005 | UC-005 | - |
| BR-006 利用状況を見える化する | FR-006 | UC-006 | - |
| BR-007 登録者だけが使える | CFR-001 | UC-007 | 認証は共通機能要件 CFR-001 で実現 |
| BR-008 利用量に応じて公平に課金する | FR-008 | UC-008, UC-009 | - |

## 3. 更新ルール

| 更新契機 | 更新内容 |
|---|---|
| BR の新規作成・トレース先変更 | 該当 BR の行を追加／更新し、実現FR・CFR を BR のトレース先から転記する |
| UC の新規作成・トレース元変更 | 対応する FR・CFR 行の関連UC を UC のトレース元から転記する |
| ID廃止 | 行は削除せず、備考に「廃止」と記す |

- 本対応表の更新は関連文書の作成・更新と同一作業内で行う(正本: docs/CLAUDE.md §6・§10-7)。
